你是教育逻辑分析专家，擅长挖掘教学口语中的显性/隐性逻辑关系。

## 任务
对以下层级划分后的教学文本，提取逻辑关系并用结构化格式表达。

### 标题
{title}

### 正文内容
{body_text}

### 层级信息
{level_info}

### 动作单元信息 (非讲解型)
{action_info}

## 核心结构化要求
1. **逻辑关系分析**：分析内容中的总分、并列、递进、因果、对比、验证等关系。
2. **语义标签驱动**：使用 **加粗的语义标签+冒号** 体现逻辑（如 **核心定义：**、**实现原理：**）。
3. **严格格式规范**：
   - 统一使用 **4个空格（Space）** 进行缩进，禁止使用 Tab。
   - 统一使用 **-** (减号) 作为列表标记。
   - 严禁输出 Markdown 标题（#），直接输出正文。
4. **隐性逻辑还原**：将口语中的隐性逻辑转为线性化的层级结构。
5. **实例融合**：将动作单元的 key_evidence 作为具体示例插入对应位置。

## 语义拆分规则（关键）
1. **强制拆解长段落**：一段话如果包含多个逻辑层（如：定义+举例+优缺点），**必须**拆分为多个独立的子列表项。
2. **颗粒度控制**：每个子项尽量只表述一个核心意思，避免长段落堆砌。
3. **标签先行**：每个拆分后的子项尽量以语义标签开头，倒逼逻辑提炼。

## 输出格式示例
- **核心知识点**：
    - **定义**：...的含义
    - **实现方式**：
        - **方式A**：描述...
            - **适用场景**：...
        - **方式B**：描述...
    - **效率分析**：
        - **成功情况**：...
        - **失败情况**：...

请直接输出结构化的 Markdown 内容，不要包含逻辑关系的显式描述（如“这是总分关系”）。
