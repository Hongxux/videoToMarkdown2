# 知识讲解视频分析 Prompt

## 任务目标

将视频讲解内容整理为结构化学习笔记，**严格跟随讲者的讲解逻辑**。

---

## 核心原则

### 1. 跟随讲者逻辑
按照讲者的实际讲解顺序组织内容，不重新排列、不重构逻辑。

### 2. 保留讲者风格
- 保留讲者的原话、措辞、比喻
- 保留讲者的强调方式和重复
- 保留讲者的类比、故事、例子

### 3. 不删减、不添加
- 完整记录讲者表达的所有知识点
- 不添加讲者没有讲到的内容
- 不省略讲者认为重要的内容

### 4. 自适应结构
- 不强制套用"概念→公式→示例"的固定模板
- 根据讲者实际讲了什么来组织内容
- 如果讲者先举例再讲概念，就按这个顺序记录

---

## 输出格式

```markdown
# [视频标题/主题]

## 概述
简要说明本节内容的主题和覆盖的知识点范围（1-2句话）

## 正文内容

（按照讲者的讲解顺序自然分段，使用合适的标题层级）

### [第一个知识块标题]
（按讲者讲解方式自由组织，可以是：）
- 讲者的引入方式
- 核心概念解释
- 公式推导（如有）
- 例子说明
- 重点强调

### [第二个知识块标题]
...

## 视觉辅助素材识别

### 视觉辅助素材清单

#### 素材1：[名称]
- **时间范围**：HH:MM:SS - HH:MM:SS
- **类型**：PPT/板书/动画/代码/示意图
- **内容说明**：描述该素材展示的具体内容
- **插入位置**：正文 → [对应标题/知识点] 之后
- **提取方式**：关键帧/视频片段
- **帧数建议**：X张
- **选帧指导**：具体选取什么样的帧
```

## 示例对比

### ✅ 正确示例（跟随讲者逻辑）

```markdown
### 顺序查找的直观理解

讲者首先用一个生活例子引入：

> "就像你在一排抽屉里找东西，只能一个一个打开看，这就是顺序查找。"

然后提出核心问题：

> "那平均要找多少次呢？这就是我们要计算的平均查找长度。"

接着，讲者展示了推导过程：

1. 查找第1个元素需要1次比较
2. 查找第2个元素需要2次比较
3. ...以此类推

讲者特别强调：

> "注意这是一个等差数列，所以我们可以用求和公式。"

最终得出公式：**ASL = (n+1)/2**
```

### ❌ 错误示例（强制套模板）

```markdown
### 顺序查找

**概念定义**：顺序查找是一种线性查找算法...

**核心公式**：ASL = (n+1)/2

**推导过程**：
1. 设查找表有n个元素...

**示例**：假设n=10，则ASL=5.5

**注意事项**：时间复杂度为O(n)
```

## 额外指导

### 数学公式
- 使用 LaTeX 格式：`$公式$` 或 `$$公式$$`
- 保留讲者的推导步骤

### 代码
- 使用代码块，标注语言
- 保留讲者的注释和说明

### 引用
- 使用 `>` 引用讲者的原话
- 保留重要的口头表达

### 强调
- 使用 **加粗** 标记讲者强调的重点
- 使用列表整理步骤性内容
