## 语境化解释
1. 【概念解构（破除陌生感）】：'GPT 系列对 Markdown 友好'在此指 GPT 模型因训练数据中富含 Markdown 格式内容（如 GitHub 文档、技术博客），使其能更自然地解析 Markdown 的标题、列表、代码块等元素，从而在提示词中使用 Markdown 能引导模型输出更结构化、清晰的响应。
2. 【点石成金（回应好奇心）】：作者精准选用此术语，是因为它作为'已知模型偏好'的典型案例，将调优从依赖未知训练数据的迷雾中拉出，提供了基于格式的确定性杠杆——Markdown 作为通用标记语言，其友好性成为调优中的可靠工具。

## 深度
1. 【本质剥离】：这揭示了大模型学习的统计本质：Markdown 友好性源于训练语料中 Markdown 文本的丰富分布，模型学习到其语法模式后，在推理时能更高效地处理相关 token，降低生成噪音，提升任务性能。
2. 【第一性原理】：从计算语言学角度，这触及'形式与功能耦合'的原理——Markdown 的轻量级标记为模型提供了清晰的指令框架，类似于编程中的 API 接口，减少了 prompt 的模糊性，直接优化信息传递效率。

## 广度
1. 【跨界同构】：类比人类沟通，这像使用标准化术语（如医学术语）提升专业对话效率——Markdown 作为技术社区的'通用语'，其友好性使模型能更精准地融入该语境，在设计中类似用户界面遵循设计系统以提升可用性。
2. 【高维演进/极端延伸】：扩展到系统论，这类似协议在计算机网络中的角色——Markdown 作为模型与用户间的'通信协议'，其友好性优化了交互流，可延伸至经济学中标准化合约降低交易成本的现象。
