# Skill: 并列关系精修（logic_parallel）

当 section 中存在多个同级并列的实体、分类、优缺点时，适用本 skill。

---

## 精修规则

### 1. 强制排序 + 说明排序依据
并列项**不允许随意罗列**。必须按照以下维度之一进行有序排列，并在父节点或首项处**显式说明排序依据**：
- **时间维度**：按事件发生的先后时间顺序排列（如：先发生的在前）
- **空间维度**：按从宏观到微观、从外到内、从上到下的空间层次排列
- **重要性/递进维度**：按影响力、严重程度、优先级从高到低（或从低到高）排列
- **因果维度**：按因→果的逻辑链条排列（原因在前，结果在后）

**格式要求**：
- 在父节点或列表开头标注排序说明，如：`- **三大核心模块（按数据流先后排序）**：`
- 如果并列项之间确实没有任何自然排序关系（纯粹并列），允许保持原文顺序但需标注 `（无优先级区分）`

### 2. 去冗余与合并
- 检查并列项是否存在**语义重叠**：如果两项说的其实是同一件事的不同表述，合并为一项
- 检查并列项是否存在**层级混乱**：如果某项其实是另一项的子集，应将其降级为子节点而非并列

### 3. 对仗感增强
- 尽量保持并列项的**结构对称**：项与项之间的句式结构、标签命名风格应统一
- 例如：如果前两项是 `**XX的优势**：...`，第三项不应该突然变成 `优点是...`

### 4. 忠实原意红线
- 并列项的内容必须**严格来源于原文**
- 禁止为了凑齐排序或对仗而编造原文没有的项目、数据或论据
- 排序依据应从原文的上下文中推断，如果原文没有暗示任何排序关系，如实标注 `（无优先级区分，保持原文顺序）`

### 5. 递归深拆
当某个并列项内部仍然包含复句或多层含义时，必须继续递归拆解为更深的子节点——通过改写重组保证每个最终叶子节点都是一个**单义短句**。

### 正向示例

**原文输入：**
> 这个架构有三个优点：性能高、易于扩展、成本低。

**❌ 错误精修（添加了原文没有的内容）：**
```markdown
- **三大核心优势（按架构决策优先权重排序）**：
    - **性能优势**：得益于异步非阻塞设计，吞吐量提升数倍。  ← ❌ 原文没说"异步非阻塞"和"数倍"
    - **扩展优势**：模块化架构允许水平扩展。  ← ❌ 原文没说"模块化"和"水平扩展"
    - **成本优势**：同等负载下服务器开销显著降低。  ← ❌ 原文没说"同等负载"和"显著降低"
```

**✅ 正确精修（仅重组原文已有内容）：**
```markdown
- **三个优点（无优先级区分，保持原文顺序）**：
    - **性能高**
    - **易于扩展**
    - **成本低**
```
