# Skill: 因果/推导论证精修（logic_causal）

当 section 中存在"因为→所以"、"导致→结果"、论证推导链条时，适用本 skill。

---

## 精修规则

### 1. 因果链结构化（挖掘"为什么对"）
每一条因果关系必须拆解为清晰的链条，**严格禁止将因和果混写在同一个节点里**。

拆解时遵循以下两条核心要求：
- **标签由你自由命名**：根据原文语境，用最贴切的语言概括每个节点在因果链中扮演的角色。标签应该自然、准确，直接反映该节点在原文中的逻辑地位，不要套用固定词汇。
- **必须挖掘底层原理**：不仅要展示"做了什么 → 导致了什么"，还要寻找并显性化支撑因果成立的根本原因——即"为什么 A 能导致 B"。如果原文隐含了这个道理，必须提取为独立节点。

如果存在多级因果链（A→B→C），必须逐级展开，不得跳跃。

### 2. 忠实原意红线
- 因果链的每一环必须**严格来源于原文**
- 原文未明说的中间推导步骤，**直接省略**，禁止替原文补全论证
- 底层原理只能从原文已有的论述中归纳提取，禁止编造

### 3. 递归深拆
当因果链中的某个环节仍然包含复句或多层含义时，必须继续递归拆解为更深的子节点——通过改写重组保证每个最终叶子节点都是一个**单义短句**。

### 正向示例

**原文输入：**
> 虚拟线程不能把底层的物理线程也阻塞了。JVM 会剥离虚拟线程的上下文，释放载体线程去跑其他虚拟线程。这是它实现高并发的核心。

**❌ 错误精修（套用固定标签 + 编造内容）：**
```markdown
- **虚拟线程高并发因果链**：
    - **底层原理**：操作系统级线程切换开销巨大。 ← ❌ 原文没说
    - **关键动作**：JVM 将上下文保存至堆内存。 ← ❌ 原文没说堆内存
```

**✅ 正确精修（语义标签为精炼名词短语 + 挖掘底层约束）：**
```markdown
- **虚拟线程实现高并发的因果链**：
    - **根本约束（物理线程不可阻塞）**：虚拟线程不能把底层的物理线程也阻塞了。
    - **核心机制（JVM 上下文剥离）**：JVM 剥离虚拟线程的上下文，释放载体线程。
    - **载体线程复用**：被释放的载体线程去跑其他虚拟线程。
    - **最终效果（高并发实现）**：这是虚拟线程实现高并发的核心。
```
