# Skill: 条件分支关系精修（logic_conditional）

当 section 中存在"如果A则B，否则C"、"当...时...否则..."、"满足X条件才..."等条件分支逻辑时，适用本 skill。

---

## 精修规则

### 1. 条件表达式独立为父节点
- 条件判断语句必须**单独成为一个父节点**，不得与任何分支结果混写在同一行
- 格式：`- **条件判断（{一句话描述判断的对象}）**：`

### 2. 分支路径平行展开
- 各分支路径作为条件节点的**平行子节点**展开
- 每个分支必须标注**触发条件**和**结果/动作**
- 推荐标签：
  - `**满足条件时（{具体条件}）**`
  - `**不满足条件时（{具体条件}）**`
  - 或 `**路径A（{条件描述}）**` / `**路径B（{条件描述}）**`

### 3. 嵌套条件展开
- 如果分支内部还有进一步的条件判断（嵌套 if-else），必须**继续缩进展开**为更深的条件-分支结构
- 禁止在一个子节点里堆积多层条件逻辑

### 4. 默认路径/兜底路径标注
- 如果条件逻辑中存在"否则"、"默认"、"兜底"路径，必须显式标注
- 格式：`- **兜底路径（以上条件均不满足时）**：{兜底处理}`

### 5. 忠实原意红线
- 分支路径的每一条内容必须**严格来源于原文**
- 禁止为了让决策树更完整而编造原文没有提到的分支条件或处理逻辑
- 如果原文只提到了"如果A则B"但没有说"否则怎样"，不得编造兜底路径，保持原文的信息边界

### 6. 递归深拆
当某个分支内部仍然包含复句或多层含义时，必须继续递归拆解为更深的子节点——通过改写重组保证每个最终叶子节点都是一个**单义短句**。

### 正向示例

**原文输入：**
> 如果用户已登录就展示主页，否则跳转到登录页面。

**❌ 错误精修（编造原文没有的嵌套分支）：**
```markdown
- **入口条件判断（用户登录状态检查）**：
    - **已登录（session 有效）**：直接展示主页。
    - **未登录（session 无效或过期）**：跳转至登录页面。
        - **登录成功**：创建新 session，重定向至主页。 ← ❌ 原文没说
        - **登录失败超过3次**：锁定账号。 ← ❌ 原文没说
```

**✅ 正确精修（仅重组原文已有内容）：**
```markdown
- **入口条件判断（用户登录状态）**：
    - **已登录**：
        - **动作**：展示主页。
    - **未登录**：
        - **动作**：跳转到登录页面。
```
