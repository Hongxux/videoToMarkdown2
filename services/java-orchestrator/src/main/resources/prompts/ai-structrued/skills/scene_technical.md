# Skill: 技术原理 / 学术论证场景（scene_technical）

当 section 涉及代码、算法、协议、框架机制、系统架构等硬核技术内容时，适用本 skill。

---

## 精修规则

### 1. 机制描述拆分
遇到"机制描述"（如：当...时，系统会做A然后做B），必须拆分为：
- **触发时机（When）**：什么条件/场景下触发
- **独立行为/步骤（What/How）**：具体发生的行为序列，每个行为独立成子节点

### 2. 对比段落张力呈现
遇到"对比段落"（旧版 vs 新版、传统 vs 现代），鼓励采用对仗句式呈现张力：
- 使用成对的标签：`**旧版（同步阻塞）的瓶颈**：...` vs `**新版（异步非阻塞）的破局**：...`
- 对比维度保持一致：如果旧版从"性能"角度描述，新版也必须从"性能"角度对比

### 3. 代码块精确嵌入
- 代码片段必须作为对应机制/步骤的**子节点**嵌入，绝不脱离上下文独立罗列
- 必须标注代码语言（如 ` ```java `）
- 如有参数，将参数解释作为代码块的同级或下级子节点

### 正向示例

**输入：**
> 虚拟线程的暂停（Park）机制非常巧妙。当虚拟线程执行到阻塞操作（比如发起IO请求或者调用sleep）时，它不能把底层的物理线程（载体线程）也阻塞了。这时候JVM就会马上剥离它，把当前虚拟线程的整个执行上下文，包括程序计数器啊、栈帧啊、局部变量这些东西，全部保存到堆内存里。保存完之后，就会立刻释放那个载体线程，让载体线程去跑其他的虚拟线程。
> 这是它实现高并发的核心。比如下面这行代码 `Thread.sleep(1000);` 就能触发这个机制。

**期望精修输出：**
- **隐式机制运作流程：暂停（Park）降级动作**
    - **前置触发时机（遇到明确阻塞动作）**：
        - **触发条件**：当执行到明确的阻塞动作时（如发起网络 I/O 请求或主动等待）。
        - **代码示例验证**：
          ```java
          Thread.sleep(1000);
          ```
    - **宏观核心响应（JVM 的极速卸载策略）**：
        - **总览**：底层的 JVM 会立即针对该虚拟线程触发两步卸载动作。
        - **第一步（卸载并转移）**：
            - **动作**：将当前虚拟线程的状态完整卸载。
            - **目的地**：持久化转移保存至堆内存安全区中。
            - **底层关键要素支撑**：覆盖程序计数器、栈帧信息、局部变量值等所有执行期间的关键上下文。
        - **第二步（迅捷释放）**：
            - **动作**：立即果断释放原先绑定的底层运载物理线程。
            - **后续流转**：任由其去自由调度并执行其它正处于就绪态的任务。
- **外延架构级价值点（高并发基底）**：
    - **结果导向**：彻底突破硬件级别的并发天花板瓶颈。
    - **对比传统模型（消灭痛点）**：完美规避了物理实体线程因 I/O 等待而被==漫长无期地死死挂起==的致命痛点现象。
