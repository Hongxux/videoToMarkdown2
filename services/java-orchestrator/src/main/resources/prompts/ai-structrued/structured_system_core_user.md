## 输出要求

你必须完成以下任务：

### 任务一：将原文按内容主题划分为多个 section
- 按照原文自然的主题/段落边界进行划分
- 每个 section 应该是一个相对独立的内容单元（一个核心观点、一个步骤流程、一段叙事等）
- section 数量根据原文长度和内容复杂度自然确定，不要强行拆分也不要堆积

### 任务二：为每个 section 标注元数据
为每个 section 标注：
- `logic_tags`：该 section 的主导逻辑关系类型（可多选）：`parallel` / `hierarchical` / `causal` / `progressive` / `contrast` / `conditional`
- `scene_tags`：该 section 的内容场景类型（可多选）：`technical` / `procedure` / `reading` / `narrative`
- `title`：该 section 的一句话概括标题

### 任务三：对每个 section 进行骨架重构
- 按照 system prompt 中的父子逻辑、拆分四步法、长句打碎规则，将原文重构为带缩进层级的 Markdown
- 用语义标签展现逻辑关系
- 保留所有术语、代码块、图片标识

---

## 输出格式（严格遵守）

输出分为两部分，用 `---` 分隔：

**第一部分**：JSON 元数据块（用 ```json 代码块包裹）

```json
{
  "sections": [
    {
      "id": "s1",
      "logic_tags": ["causal", "progressive"],
      "scene_tags": ["technical"],
      "title": "一句话概括"
    },
    {
      "id": "s2",
      "logic_tags": ["parallel", "hierarchical"],
      "scene_tags": ["reading"],
      "title": "一句话概括"
    }
  ]
}
```

**第二部分**：各 section 的初稿 Markdown，每个 section 用 `## sN: {title}` 作为二级标题起始

示例结构：
```
## s1: 虚拟线程的暂停机制
- **因果链**：当虚拟线程执行到阻塞操作时...
    - JVM 剥离执行上下文...

## s2: 高并发架构的价值维度
- **浅层收益**：...
- **深层红利**：...
```

**图片与占位符去重红线提醒：**
🔴 **无论是原文自带的图片标识还是类似 `[KEYFRAME_{N}]` 等占位符，同一个图/链接在你的全文结构化输出中只能出现一次！严禁重复挂载同一张图配多个句子。若逻辑紧密需同图，请仅保留在其首次出现的逻辑最顶层根节点句尾！**

**结构与层级长句拆解深缩进最后提醒：**
1. **多层深重嵌套与大胆下钻**：遇到【概念定义/论证/具体示例/原因】等展开性内容时，**强烈要求你毫不吝啬地进行第三级、第四级甚至更深的缩进嵌套来体现立体感**！特别是**概念定义或长篇大论的长句**，必须打碎其属性特征，将其斩碎为几句短句并向下多层缩进展开，**坚决杜绝把多重含义揉杂死挤在同一段漫长的句子里！**
2. **重构散乱并列**：遇到多项并列（优势一、二、三等）时，请务必主动寻找其内在规律进行分类降维（提取父节点）或进行逻辑串联，拒绝散乱机械的平铺。**当提取高维度分类时，一定要在父节点补充说明该更高维度的来源依据是什么，以及为什么要用这个视角统摄这些底层点。**
3. **机制过程拆解**：遇到"当执行到某处时，会发生A，再发生B"这类过程机制长句时，必须拆解出**时机（When）**与具体的**含义/步骤（What/How）**作为超短的深层子节点，坚决杜绝把它们糊在一个冗长的复句里。
