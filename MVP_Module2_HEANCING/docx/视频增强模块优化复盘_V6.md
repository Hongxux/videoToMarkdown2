# 视频增强模块优化复盘 (V2, V3 & V4)

已完成视频截取逻辑的全面迭代，从 ASR 语义对齐 (V2) 进化到**语流完整性保护 (V3)**，并同步实现了**实证优先的需求决策树 (V4)**。

## 1. 视频截取 V2 (语义优先)
针对“开始晚、结束早”的初级问题，通过 `_get_complete_semantic_baseline` 引入了 8s 话题搜索窗，初步解决了片段范围覆盖不足的问题。

## 2. 视频截取 V3 (语流完整性) 核心实现
V3 架构回归口语第一性原理，彻底解决“老师讲到一半被切断”的痛点。

| 优化项 | 实现内容 | 解决问题 |
| :--- | :--- | :--- |
| **Pause Detection** | 基于 0.3s 语音间隙自动识别口语分句边界。 | 解决无标点字幕场景下的语义基准线锚定。 |
| **终止点并集逻辑** | 锚定“句尾”与“物理翻页”的并集（最晚点，上限+3s）。 | 解决视觉翻页延迟导致的结论丢失。 |
| **边界互斥校验** | 维护已确定片段缓存，3s 搜索窗自动收缩。 | 解决相邻片段逻辑重叠与“重叠夺取”。 |
| **语流自然缓冲** | 增加 0.2s-0.3s 双向缓冲，严格不跨越相邻片段。 | 消除机器剪辑感，保留自然语气。 |

## 3. 素材增强决策树 V4 (实证优先)
回归“所见即所得”的实证原则，修正语义分类的滞后性。

| 层级 | 优化逻辑 | 效果 |
| :--- | :--- | :--- |
| **Level 1** | **通用静态结构直连**：检测到架构图/流程图/表格直接锁定 SCREENSHOT。 | 解决“复杂图示被降级为文本”的问题。 |
| **Level 2** | **认知需求修正**：视觉特征强行修正语义分类偏误（如静态背景+文本）。 | 确保素材载体与感官事实对齐。 |
| **Level 4** | **Cognitive Match Priority**：如果素材匹配度 >= 0.5 且符合需求类型，优先选择。 | 减少不必要的文字兜底，优先回归多模态。 |

## 4. V3-V4 跨模块联动
在 `VIDEO_AND_SCREENSHOT` 模式下，截图锚点现在**动态锁定在 V3 的 `refined_e` (语流结束点)**，确保截图抓取的是老师讲完结论后的最稳定画面。

## 5. 环境与日志增强
- **whisper_env 修复**：已完成 `pytesseract` 环境适配，OCR 探测通过。
- **探测日志追踪**：在 `multimodal_fusion.py` 中增加了对公式识别和决策路径的详细追踪。

## 6. 自适应资源编排 V6 (性能与稳定性平衡)
针对 15GB 内存占用高峰及 `os error 1455` 屏障，实施了 V6 动态编排方案。

| 优化项 | 实现内容 | 效果 |
| :--- | :--- | :--- |
| **Dynamic Concurrency** | 基于 `psutil` 实时监控可用内存，动态调节并发深度 (2-12)。 | 在 22GB RAM 环境下自动拉升至 12 并发，资源受限时自动收缩。 |
| **Analysis Proxying** | 所有特征分析强制使用 360p 代理帧，而非 1080p 原始帧。 | 内存峰值显著从 15GB 降低至 < 4GB。 |
| **Adaptive Caching** | `SharedFrameRegistry` 容量根据系统空闲内存动态伸缩。 | 兼顾低资源稳定性与高资源命中率。 |
| **High-Res Re-extraction** | 仅对最终获胜的截图帧，重新回溯视频进行 1:1 原画质提取。 | 彻底解决因 360p 代理分析导致的“截图模糊”问题 (V6.2)。 |
