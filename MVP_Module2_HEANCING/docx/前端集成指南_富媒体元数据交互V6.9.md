# 前端集成指南：富媒体元数据交互 (V6.9.7)

## 1. 核心理念 (First Principles)
传统的视频切片往往导致体验碎片化。V6.9.7 架构升级后，后端不再输出孤立的视频文件，而是输出**结构化的富媒体元数据 (Rich Media Metadata)**。
前端应遵循 **“认知减负”** 原则：
1.  **默认静止**：优先展示高信噪比的海报 (Poster)，避免自动播放干扰用户。
2.  **按需动态**：用户感兴趣时，通过折叠面板 (Accordion) 访问高光动态片段 (Clips)。
3.  **结构化**：利用元数据中的 `type` 和 `description` 构建语义化的播放列表，而非无意义的时间轴。

---

## 2. 数据结构 (JSON Schema)

在 `e2e_result.json` 的 `video_and_screenshot` 类型的条目中，新增 `rich_media` 字段。

```typescript
interface EnhancementResult {
  id: string; // e.g. "ENH_P012_F01"
  type: "video_and_screenshot" | "screenshot"; 
  rich_media?: RichMediaMetadata; // V6.9.7 新增
  // ... 其他现有字段
}

interface RichMediaMetadata {
  // 布局类型：决定前端的渲染模式
  // "poster_with_clips": 海报+动态切片列表 (标准模式)
  // "poster_only": 仅显示海报 (LLM 判定动态不具备认知价值时降级)
  layout_type: "poster_with_clips" | "poster_only";

  // 核心海报 (Default View)
  poster_path: string;       // 相对路径或URL，e.g. "poster_ENH_P012_F01_30.00s.png"
  poster_timestamp: number;  // 海报对应的时间戳 (秒)

  // 动态切片列表 (Action Windows)
  // 仅在 layout_type="poster_with_clips" 时非空
  clips: DynamicClip[];

  // 原始字幕文本 (用于上下文展示)
  transcript: string; 
}

interface DynamicClip {
  id: string;        // e.g. "clip_0"
  start: number;     // 相对视频起始的时间 (秒)
  end: number;       
  duration: number;  // 持续时长 (秒)
  
  // 动作类型 (用于 UI 图标/标签)
  // "knowledge": 知识生产 (书写/绘图)
  // "smooth_flow": 平滑移动 (光标/图表拖动)
  // "transition": 容器切换 (PPT翻页)
  type: string;      

  description: string; // e.g. "Dynamic segment 1" (未来可由 LLM 生成更具体的描述)
}
```

## 3. 交互规范 (Interaction Spec)

请参照 **Coursera / Udemy** 等顶级教学平台的素材交互规范。

### 状态 A: 默认状态 (Collapsed)
*   **组件外观**: 展示 `poster_path` 图片。
*   **覆盖层 (Overlay)**: 
    *   显示播放按钮图标。
    *   **关键**: 显示 `clips.length` 提示标签 (例如: `⚡ 3个动态演示`).
    *   如果 `layout_type == "poster_only"`, 则**不显示**动态提示，仅作为静态图展示（可点击查看大图）。

### 状态 B: 展开状态 (Expanded)
*   **触发**: 用户点击默认海报区域。
*   **组件外观**:
    *   海报缩半或上移。
    *   下方滑出 **动态片段列表**。
*   **列表项**:
    *   展示每个 Clip 的 `type` 图标 (✏️ 书写, 🖱️ 移动)。
    *   展示时长 (e.g. `2.5s`).
    *   点击列表项 -> 在海报区域加载并播放对应时间段的 **源视频** (利用 HTML5 Video Fragment 播放技术，或请求独立 Clip URL)。
    *   *注：当前 V6.9.7 版本暂未切分物理 mp4 小文件，建议前端基于主视频 URL + `#t=start,end` 进行片段播放，或等待后端 V7.0 提供独立 mp4 URL*。

### 状态 C: 认知降级 (Poster Only)
*   当 `layout_type == "poster_only"` 时：
    *   后端虽然检测到了微弱动态，但 LLM 认为“不看也不影响理解”。
    *   前端应**完全隐藏**播放入口，仅展示图片。这是为了防止用户点击后看到“没什么信息量”的几秒钟移动，产生挫败感。

## 5. 开发建议
*   **播放器复用**: 建议前端只维护一个全局 Video 播放器实例，点击不同 Clip 时只是 seek 到不同时间点并播放指定时长，体验流畅且节省流量。
*   **预加载**: 鼠标 Hover 海报时，可预加载 `clips[0]` 的首帧或前1秒视频，提供“Gif 动图”般的预览体验。
