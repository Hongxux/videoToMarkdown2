# 一、先锚定：第一性原理——截图/视频片段的本质边界
这是所有选择的**唯一根依据**，不看画面、不看时长，只看**信息的存在形式**：

## 1.1 两种载体的本质定义
| 载体 | 第一性原理本质 | 承载的信息核心 | 信息是否依赖**时序连续** | 认知负荷特性 |
|------|---------------|----------------|--------------------------|--------------|
| **关键截图（单张/多张）** | 静态空间结构的离散定格 | 瞬时状态、最终结果、关键节点、布局结构、公式/图表/界面外观 | **不依赖时序**：多张截图按序排列，即可还原全部知识 | 更低：可凝视、可反复看、无时间压力 |
| **视频片段** | 时序连续的状态迁移与因果链 | 操作顺序、推导链条、连续变形、阶段演进、动作衔接 | **强依赖时序**：核心信息存在于「前一帧→后一帧」的关系中，静态无法完整还原 | 更高：必须按时间观看，适合必须看过程的场景 |
| **混合（截图+视频）** | 时序核心+静态锚点互补 | 过程主体用视频保时序，关键节点用截图做记忆锚点、复盘节点 | 时序为核心，静态为辅助 | 平衡：既保过程完整性，又降低记忆与回溯负荷 |

## 1.2 模态选择的三大公理（不可违反）
1. **最小负荷公理**：能完全用截图表达的过程/动态，**一律不用视频**（优先截图，降低认知负荷）。
2. **不可替代公理**：若动态的**核心知识依赖时序连续**，截图无法等价表达，则**必须用视频**（可搭配截图做锚点）。
3. **语义唯一公理**：一个语义单元内的同一段动态，只采用**一种主模态**，禁止无意义的多模态堆砌。

## 1.3 过程性知识的核心划分（你要的底层分界）
我们把**所有有效动态（知识生产型）**按第一性原理一刀切开：
- **A类：可静态化过程动态**
  动态只是「把结果一步步亮出来」，**时序本身不产生新知识**，全部信息可拆解为若干个关键状态，用多张有序截图完整覆盖。
- **B类：不可静态化过程动态**
  动态的**时序、顺序、连续变化就是知识本身**，去掉时序就丢失核心逻辑，必须用视频保留连续关系。

这就是你要的：**A类→截图，B类→视频/混合**。

# 二、领域最佳实践：教育视频过程性知识的模态匹配标准
结合 Mayer 多媒体学习理论、教学视频结构化工业落地（Coursera/edx/知识类视频结构化工具）、人机交互与认知心理学，形成和你场景完全对齐的**最佳实践规范**：

## 2.1 适合「纯截图」的过程/动态（可静态化）
特征：
- 动态是**离散式增量**（一段一段出现、一块一块绘制），而非连续平滑变形；
- 每一步都对应一个**稳定、可定格的终态**（稳定岛）；
- 步骤之间无连续因果依赖，按顺序看截图即可理解全部逻辑；
- 学习者需要反复查看某一步的细节，静态更友好。

典型内容：
- 公式逐行浮现、分块展示；
- 代码逐行打字出现（无连续光标运行、无实时编译变化）；
- 图表柱形/线段逐条弹出、分区域点亮；
- PPT内知识点逐条淡入、分点展开。

## 2.2 适合「纯视频片段 / 截图+视频混合」的过程/动态（不可静态化）
特征：
- 动态是**连续平滑变化**，状态无明确离散断点；
- 操作/推导有**严格顺序依赖**，前一个动作直接决定后一个结果；
- 核心信息在「变化过程」，而非某几个定格帧；
- 单张/多张截图会造成逻辑断层，学习者无法脑补中间过程。

典型内容：
- 手写公式连续推导、粉笔/笔不间断书写轨迹；
- 软件连续操作（点击→拖拽→输入→确认）完整链路；
- 函数曲线连续变形、数据动态实时演变；
- 实验现象连续变化、物理模拟连续动画。

### 3.2.1 K1 离散增量型
- 定义：知识元素以**独立块、离散行**分批出现，无连续书写/变形
- 典型：公式分行淡入、代码逐行弹出、分点逐条显示
- 核心：时序仅为呈现顺序，**无知识依赖**
- 模态方案：**纯多张有序截图**（提取每一步稳定岛，按序排列）
- 工程判定（你可直接写代码）：
  ```
  知识生产型前提
  && is_stepwise = True
  && has_stable_island_in_step = True
  ```

### 3.2.2 K2 分步定格型
- 定义：动态有明确步骤，每步结束后**长时间稳定定格**，步骤间有自然切断
- 典型：图表先出X轴→稳定→出Y轴→稳定→出数据点→稳定
- 核心：每步都是完整稳定状态，截图可完全还原
- 模态方案：**纯多张有序截图**
- 工程判定：
  ```
  知识生产型前提
  && 动态可切分为≥2个子区间
  && 每个子区间结束后均存在稳定岛(≥500ms)
  && 无连续平滑变形
  ```

### 3.2.3 K3 连续推演型
- 定义：**不间断连续推演/书写/变形**，无自然定格断点，时序就是逻辑链
- 典型：手写公式连续推导、函数曲线连续绘制、几何图形连续构造
- 核心：**时序不可替代**，截图会丢失推导轨迹与因果
- 模态方案：**视频片段（主）+ 首尾/关键节点截图（辅）**
- 工程判定：
  ```
  知识生产型前提
  && is_continuous_derivation = True
  && 全程无≥500ms的稳定岛断点
  ```

### 3.2.4 K4 连续操作型
- 定义：软件/仪器**连续操作链路**，动作衔接决定结果，顺序是知识核心
- 典型：点击→拖拽→输入→确认的完整操作、仪器连续调节
- 核心：操作顺序与连续状态迁移不可静态化
- 模态方案：**纯视频片段**（极简长度），复杂操作可加「起点+终点截图」
- 工程判定：
  ```
  知识生产型前提
  && is_continuous_operation = True
  && ROI内控件/界面状态连续迁移，无分步定格
  ```

# 四、端到端决策流程：从动态区间 → 模态选择（可直接编码）
完全贴合你的 pipeline 执行顺序：语义单元 → 提取动态区间 → 物理层分类 → 有效动态子分类 → 模态决策 → 素材提取

```
输入：动态区间[t_start, t_end]、ROI-SSIM序列、时长、视觉变化特征、字幕语义
输出：剔除 / 纯截图 / 纯视频片段 / 视频+截图混合

1. 先做你原有物理层+有效性判定
   1.1 if duration < 0.3s:
       → 无效变动型，【剔除】，无素材
   1.2 elif ssim_drop_ratio > 0.5 and duration < 1.5s:
       → 容器切换型，【剔除】，无素材
   1.3 elif ssim_drop_ratio < 0.2 and duration >= 1.5s:
       → 知识生产型，进入子类型与模态判定
   1.4 else:
       → 过渡灰区，【剔除】

2. 知识生产型内部模态判定（核心）
   2.1 if (is_stepwise and has_stable_island_in_step) OR (分步+每步稳定岛):
       → K1/K2，【模态：纯多张有序截图】
         提取规则：提取每一步稳定岛的最优帧，按时序排序，无视频
   2.2 elif is_continuous_derivation and 无长稳定断点:
       → K3，【模态：视频片段 + 关键节点截图】
         提取规则：剪取完整连续过程视频；截取起点、推演中段、终点3张截图
   2.3 elif is_continuous_operation:
       → K4，【模态：纯视频片段（可选加首尾截图）】
         提取规则：最短完整操作链视频，无多余截图；复杂操作可补起点+终点图
```

## 6.2 模态决策核心伪代码
```python
def decide_modality(ssim_drop_ratio, duration, is_stepwise, has_stable_in_step,
                    is_continuous_derivation, is_continuous_operation):
    # 第一步：无效动态直接剔除
    if duration < DURATION_MAX_NOISE:
        return "discard", []
    if ssim_drop_ratio > SSIM_DROP_TRANSITION and duration < DURATION_MIN_PRODUCTION:
        return "discard", []
    
    # 第二步：有效动态（知识生产型）
    if ssim_drop_ratio < SSIM_DROP_PRODUCTION and duration >= DURATION_MIN_PRODUCTION:
        # 可静态化 → 纯截图
        if is_stepwise and has_stable_in_step:
            return "multi_screenshots", ["extract_all_step_stable_islands"]
        # 连续推演 → 视频+截图
        elif is_continuous_derivation:
            return "video+key_screenshots", ["extract_full_process_video", "extract_start_mid_end_frames"]
        # 连续操作 → 纯视频(可选首尾截图)
        elif is_continuous_operation:
            return "video_only", ["extract_minimal_operation_chain"]
        else:
            # 无法归类的有效动态，保守用截图
            return "single_screenshot", ["extract_last_stable_frame"]
    # 灰区
    else:
        return "discard", []
```
