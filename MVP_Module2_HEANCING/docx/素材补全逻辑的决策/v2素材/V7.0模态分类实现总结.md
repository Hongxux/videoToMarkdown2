# V7.2 模态分类实现总结 (含呈现型动态检测)

> 更新时间: 2026-02-01

## 一、核心改进

### V7.1 修复
1. **假阳性动画检测修复**
   - 提高边缘变化阈值: 500 → 1500
   - 提高重心位移阈值: 3.0 → 8.0
   - 新增连续2帧验证，减少噪点误判

2. **动作区间合并**
   - 间隔 < 0.5s 的相邻动作自动合并
   - 合并后重新计算 `action_type` 和 `modality`

### V7.2 新增: 呈现型动态检测
拦截淡入/渐显/弹出等**非知识生产型动态**，强制输出截图。

## 二、呈现型动态判定规则
```python
def _is_presentation_dynamic(action, frames, roi, stable_islands):
    """
    判定规则 (全部满足):
    1. spatial_spread > 0.75  # 全局均匀变化
    2. diff_ratio单调平滑     # 无创作型波动
    3. content_iou > 0.90     # 首尾内容一致
    4. 动作后有稳定岛         # 呈现完成即静止
    """
```

### 规则详解
| 规则 | 呈现型特征 | 创作型特征 |
|------|-----------|-----------|
| 空间分散度 | >75% ROI变化 | 局部集中变化 |
| diff_ratio | 单调上升/下降 | 有波动/间断 |
| 首尾内容IOU | >90% 一致 | <90% 新内容增加 |
| 后续稳定 | 立即稳定 | 可能继续变化 |

## 三、决策流程 (V7.2)
```
输入: 动作区间 + 帧序列 + ROI + 稳定岛列表
─────────────────────────────────────────
Step1: 基础分类
  → ssim_drop > 0.5 → transition → DISCARD
  → duration < 0.3s → noise → DISCARD
  → ssim_drop < 0.2 + duration >= 1.5s → knowledge
  → 其他 → mixed

Step2: knowledge子类型判定
  → 有内部稳定岛? → K1/K2 → SCREENSHOT
  → 是呈现型动态? → presentation → SCREENSHOT (V7.2新增)
  → 否则 → K3/K4 → VIDEO_SCREENSHOT

Step3: 合并相邻动作 (V7.1)
  → 间隔 < 0.5s → 合并 → 重新分类
─────────────────────────────────────────
输出: 模态 + 子类型
```
