分数线/上下标触发层优化方案（第一性原理+最佳实践）

一、 优化核心准则（锚定第一性原理）

所有优化均不偏离两条底层公理，同时对齐领域最佳实践：

1. 结构化空间关系公理：优化规则必须紧扣“符号本质特征”，不引入表面像素干扰（如手写噪声、光照波动）；

2. 分层识别效率公理：优化后计算复杂度仍维持O(1)，仅依赖轮廓、矩形框等基础数据，不侵入校验层的复杂计算（如霍夫变换、重心矩）；

3. 领域最佳实践对齐：参考MathPix的“动态阈值适配”、PaddleOCR的“上下文密度过滤”，解决固定规则的场景适配问题。

二、 分数线触发层优化（原规则正确，补充动态适配与抗干扰）

1. 原规则核心不足

① 宽长比、宽度占比为固定值，无法适配480P~4K跨分辨率场景（如480P下20px分数线达标，4K下同等比例分数线可能被误筛）；② 上下“存在像素内容”判断过宽松，易被零星噪声、手写划痕触发误判；③ 无方向约束，可能将斜向扁长矩形误判为候选。

四、 关键优化补充（工程化落地保障）

1. 动态阈值的工程实现

基于分辨率因子（res_factor=当前帧宽度/1920）统一缩放所有像素级约束，无需为不同分辨率单独配置规则，代码复用性提升：

def get_dynamic_threshold(res_factor):
    # 分数线动态阈值
    fraction_height = max(2, int(8 * res_factor))  # 最小高度2px
    fraction_min_length = int(20 * res_factor)
    # 上下标动态阈值
    subscript_spacing_ratio = 0.3  # 主字符宽度占比
    vertical_offset_ratio = 0.15  # 垂直偏移趋势阈值
    return {
        "fraction": {"height": fraction_height, "min_length": fraction_min_length},
        "subscript": {"spacing_ratio": subscript_spacing_ratio, "offset_ratio": vertical_offset_ratio}
    }

2. 与校验层的衔接逻辑

优化后触发层仍坚守“粗筛”定位，校验层职责不偏移：

- 分数线：触发层输出“水平扁长、上下有内容”的候选区，校验层用概率霍夫变换精准判断直线性、长度范围；

- 上下标：触发层输出“主附面积差、垂直偏移趋势”的候选对，校验层用重心矩计算精准偏移量（≥25%主字符高度），完成最终判定。
