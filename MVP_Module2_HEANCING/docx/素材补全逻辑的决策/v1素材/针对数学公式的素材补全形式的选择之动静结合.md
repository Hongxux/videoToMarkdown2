```python
def extract_result_screenshot(self, dynamic_video_frames: list) -> np.ndarray:
    """从动态视频片段中提取最终稳定帧作为截图"""
    # 1. 截取视频的最后1秒帧（大概率是结果稳定帧）
    fps = self.fps
    last_second_frames = dynamic_video_frames[-fps:] if len(dynamic_video_frames) >= fps else dynamic_video_frames
    if len(last_second_frames) < 3:
        return None  # 无足够帧判断稳定
    # 2. 复用岛内择优逻辑，计算这些帧的质量评分
    frame_scores = []
    for frame in last_second_frames:
        content_region = self.get_effective_content_region(frame)
        clarity = self.quality_evaluator.calculate_sharpness(content_region)
        entropy = self.quality_evaluator.calculate_entropy(content_region)
        s4 = self._calculate_S4_no_occlusion(frame)
        score = 0.4*clarity + 0.4*entropy + 0.2*s4
        frame_scores.append((score, frame))
    # 3. 选择得分最高的帧作为结果截图
    frame_scores.sort(reverse=True, key=lambda x: x[0])
    return frame_scores[0][1]
```

# 视频+截图模式下截图选取的第一性原理分析与工程化策略
在视频+截图的组合模式中，**截图的本质是动态过程的“静态锚点”**——其核心价值是承载动态过程（手写/动画）结束后**最终稳定、完整、高清的结果状态**，适配结构化笔记的OCR识别、排版展示与知识复用需求。基于第一性原理，截图选取的所有规则必须围绕**“精准匹配结果维度的本质特征”**展开，同时复用现有系统的稳定岛、质量评分等工程逻辑，确保落地性。

## 一、 截图选取的核心第一性原理公理
截图的存在不是对视频的冗余补充，而是对“过程+结果”双维度信息的精准承载。其选取策略必须遵循三条不可违背的底层公理：
| 核心公理 | 本质内涵 | 对截图选取的约束 |
|----------|----------|------------------|
| **结果锚定公理** | 截图必须对应动态过程的**最终稳定结果状态**，而非过程中的任意中间帧 | 截图候选帧必须来自**动态过程完全结束后的稳定区间**，排除过程中的抖动帧、过渡帧 |
| **质量优先公理** | 截图的核心价值是“高清可复用”，需满足笔记场景的OCR识别、结构展示需求 | 截图必须在**清晰度、完整性、无遮挡**三个维度达到最优，优先于时序先后等其他指标 |
| **工程复用公理** | 截图选取需复用现有系统的质量评估、稳定岛检测逻辑，避免新增复杂计算 | 直接调用S1-S4评分体系、岛内择优算法，不引入额外算子 |

**本质结论**：截图选取的核心不是“从视频中随机选一张清晰帧”，而是**“锁定动态过程的最终稳定区间，在区间内选取质量最优的帧”**——这是区别于“过程帧”的关键，也是第一性原理的直接体现。

## 二、 截图候选帧的范围界定：锁定最终稳定区间（第一性原理落地）
动态过程（手写/动画）的时间线可划分为 **「过程阶段」→「结束临界点」→「结果稳定阶段」** 三部分。截图候选帧必须来自**结果稳定阶段**，这是由“结果锚定公理”决定的。

### 1.  不同动态场景的结束临界点判定
| 动态场景类型 | 结束临界点的视觉特征（客观真相） | 工程检测手段 | 第一性原理依据 |
|--------------|----------------------------------|--------------|----------------|
| **手写推导（Dynamic-Handwriting）** | 笔迹完全停止新增，画面无像素增量变化 | 连续3帧的笔迹轮廓面积变化率＜1%（无新增笔迹） | 手写过程的本质是“笔迹增量”，增量停止即过程结束 |
| **公式动画（Dynamic-Animation）** | 元素位移/显隐完全停止，公式结构成型且无变化 | 连续3帧的公式元素重心位移＜1px（随分辨率缩放），像素均值变化率＜2% | 动画过程的本质是“元素时序变化”，变化停止即过程结束 |
| **混合演示（Dynamic-Mixed）** | 手写笔迹停止+动画元素静止，公式最终结构稳定 | 同时满足手写+动画的结束判定条件 | 混合过程的结束是“两种变化的同时停止” |

### 2.  结果稳定阶段的区间提取
结束临界点之后，必须存在一个**持续时长≥0.3秒的稳定区间**（即结果稳定阶段），这是截图候选帧的唯一来源。区间的判定标准复用V6.2的稳定岛规则，确保工程兼容性：
| 结果稳定区间的判定条件 | 量化指标（随分辨率动态缩放） | 排除的无效区间 |
|------------------------|------------------------------|----------------|
| 1. 帧间MSE＜动态阈值（1.2%~2.0%） | MSE值＜同场景下过程阶段MSE的1/10 | 过程结束后的短暂抖动区间（如老师轻微擦碰黑板） |
| 2. 稳定持续时长≥0.3秒 | 区间帧数≥fps×0.3（如30fps下≥9帧） | 时长过短的不稳定区间（如动画结束后立即切换PPT） |
| 3. 无瞬时干扰 | 区间内无鼠标划过、弹窗遮挡等临时干扰 | 区间内S4无遮挡得分均值≥80 |

**工程化实现**：在动态视频片段的时间轴上，先定位结束临界点，再向后截取稳定区间：
```python
def get_result_stable_interval(self, dynamic_frames: list, fps: int, content_type: str) -> list:
    """
    从动态视频帧中提取结果稳定区间的帧
    :param dynamic_frames: 动态过程的所有帧
    :param fps: 视频帧率
    :param content_type: 动态场景类型（handwriting/animation/mixed）
    :return: 结果稳定区间的帧列表
    """
    # 步骤1：定位结束临界点
    critical_point_idx = 0
    frame_count = len(dynamic_frames)
    if content_type == "handwriting":
        # 手写场景：连续3帧轮廓面积变化率＜1%
        for i in range(2, frame_count):
            area_prev = self._calculate_contour_area(dynamic_frames[i-2])
            area_curr = self._calculate_contour_area(dynamic_frames[i])
            change_rate = abs(area_curr - area_prev) / area_prev
            if change_rate < 0.01:
                critical_point_idx = i
                break
    elif content_type == "animation":
        # 动画场景：连续3帧重心位移＜1px
        for i in range(2, frame_count):
            centroid_prev = self._calculate_formula_centroid(dynamic_frames[i-2])
            centroid_curr = self._calculate_formula_centroid(dynamic_frames[i])
            dist = np.sqrt((centroid_curr[0]-centroid_prev[0])**2 + (centroid_curr[1]-centroid_prev[1])**2)
            if dist < self.res_factor * 1:
                critical_point_idx = i
                break
    
    # 步骤2：提取临界点后的稳定区间（≥0.3秒）
    min_stable_frames = int(fps * 0.3)
    if frame_count - critical_point_idx < min_stable_frames:
        return []  # 无足够稳定帧
    candidate_frames = dynamic_frames[critical_point_idx:]
    # 验证区间稳定性（MSE阈值）
    stable_frames = []
    first_frame = candidate_frames[0]
    for frame in candidate_frames:
        mse = self._calculate_mse(first_frame, frame)
        if mse < self.dynamic_mse_threshold:
            stable_frames.append(frame)
        else:
            break  # 区间中断，停止提取
    return stable_frames if len(stable_frames) >= min_stable_frames else []
```

## 三、 稳定区间内的最优帧选取：质量维度的加权择优（第一性原理+工程复用）
在锁定的结果稳定区间内，需遵循**“质量优先公理”**，选取质量最优的帧作为截图。核心是复用你现有系统的**S1-S4评分体系**，并调整权重——因为截图的核心价值是“笔记复用”，需优先保障清晰度、完整性、无遮挡，弱化稳定性（区间内帧已足够稳定）。

### 1.  截图质量评分公式（调整后权重）
$$
Screenshot\_Score = 0.1×S1 + 0.3×S2 + 0.4×S3 + 0.2×S4
$$
| 评分维度 | 权重调整 | 调整依据（第一性原理） | 量化指标 |
|----------|----------|------------------------|----------|
| **S1（稳定性）** | 10%（降低） | 结果稳定区间内的帧稳定性差异极小，无需高权重 | 区间内帧间MSE的方差 |
| **S2（信息密度）** | 30%（保持） | 信息熵高的帧包含更完整的公式结构，适配笔记的知识完整性需求 | 有效内容区的香农熵 |
| **S3（完整性）** | 40%（提升） | 公式的矩形框、箭头、嵌套结构完整性是OCR识别的关键，优先级最高 | 公式结构化元素的数量+完整性 |
| **S4（无遮挡）** | 20%（保持） | 无鼠标/水印遮挡是截图复用的基础条件 | 有效内容区的遮挡占比 |

### 2.  最优帧选取的工程化步骤
1.  **预处理**：对稳定区间内的所有帧提取**有效内容区**（排除黑边、工具栏），仅对有效内容区计算质量指标，避免无关区域干扰；
2.  **指标计算**：对每帧计算S1-S4得分，代入调整后的评分公式；
3.  **择优锁定**：选取**Screenshot_Score最高的帧**作为最终截图；若存在多帧得分相同（误差＜1%），则选取**时序最靠后的帧**（更接近过程结束的最终状态）。

**工程化实现**（复用岛内择优逻辑）：
```python
def select_best_screenshot(self, stable_frames: list) -> np.ndarray:
    """从结果稳定区间中选取最优截图帧"""
    if not stable_frames:
        return None
    best_score = -1
    best_frame = None
    for frame in stable_frames:
        content_region = self.get_effective_content_region(frame)
        # 计算S1-S4得分（复用现有函数）
        s1 = self._calculate_S1_stability(frame, stable_frames)
        s2 = self._calculate_S2_info_density(content_region)
        s3 = self._calculate_S3_completeness(content_region)
        s4 = self._calculate_S4_no_occlusion(content_region)
        # 计算截图质量总分
        total_score = 0.1*s1 + 0.3*s2 + 0.4*s3 + 0.2*s4
        if total_score > best_score:
            best_score = total_score
            best_frame = frame
    return best_frame
```

## 四、 兜底策略：无稳定区间时的退选方案（遵循鲁棒性容忍公理）
若动态过程结束后无满足条件的稳定区间（如动画结束后立即切换PPT、手写后立即擦除），需遵循**“鲁棒性容忍公理”**，执行退选策略——优先保证截图存在，而非放弃。

| 场景2：无结果稳定区间（过程结束后立即切换） | 从动态过程的**最后3帧**中选取质量最优帧（Screenshot_Score最高） | 最后3帧最接近最终结果状态，是退选的最优候选 |
| 场景3：所有帧均存在严重遮挡 | 从视频中选取**遮挡占比最低的帧**，并标注“存在遮挡” | 即使质量不完美，也要提供基础素材，避免截图缺失 |
