# OCR信息量纳入截图评分 - 最终设计

## 核心思想

**OCR的双重作用**:
1. ✅ **正面**: 内容区域的文字 = 信息量丰富 → 加分
2. ❌ **负面**: 顶部/底部的文字 = 弹幕字幕遮挡 → 扣分

---

## S4评分新公式

```
S4 = 0.2×edge_score + 0.2×color_score + 0.4×ocr_info_score - 0.2×occlusion_penalty
     ────────────────────────────────   ────────────────────   ──────────────────
            基础质量评分                    OCR信息量评分          OCR遮挡惩罚
            (40%权重)                      (40%权重-新增!)         (扣分)
```

### 权重分配

| 组件 | 权重 | 说明 |
|------|------|------|
| 边缘密度 | 20% | 内容清晰度 |
| 颜色分布 | 20% | 无大面积遮挡 |
| **OCR信息量** | **40%** | **内容文字丰富度** (新增!) |
| OCR遮挡 | -20% | 弹幕/字幕惩罚 |

---

## OCR信息量评分 (0-100)

### 1. 区域划分

```
帧高度
├─ 0-20%: 遮挡区域 (顶部弹幕)
├─ 20-80%: 内容区域 (有效信息) ← 这里的文字加分!
└─ 80-100%: 遮挡区域 (底部字幕)
```

### 2. 评分维度

**只统计内容区域(20%-80%)的文字**:

| 维度 | 权重 | 计算方式 |
|------|------|---------|
| 字符数量 | 40分 | `min(total_chars / 50 * 40, 40)` |
| 文字区域数 | 30分 | `min(region_count / 10 * 30, 30)` |
| 文字覆盖率 | 30分 | `min(area_ratio * 1000, 30)` |

**总分** = 字符分 + 区域分 + 覆盖分

### 3. 示例

**场景A: 架构图截图**
- 内容区域: 30个文字区域, 120字符, 覆盖15%
- 字符分: 40 (满分)
- 区域分: 30 (满分)
- 覆盖分: 30 (满分)
- **OCR信息量 = 100分** ✅

**场景B: 纯画面截图**
- 内容区域: 0个文字
- **OCR信息量 = 20分** (基础分)

**场景C: 代码截图**
- 内容区域: 15个文字区域, 200字符, 覆盖10%
- 字符分: 40
- 区域分: 30
- 覆盖分: 30
- **OCR信息量 = 100分** ✅

## OCR遮挡惩罚 (0-100)

### 只检测顶部/底部

```python
if y < 20% or (y + h) > 80%:
    → 认为是弹幕/字幕
    → 计算惩罚 = area_ratio × 500
```

### 惩罚逻辑

- 遮挡区域越大 → 惩罚越高
- 最终从S4中扣除: `S4 -= occlusion_penalty × 0.2`

## 优势

### 1. 更准确的信息量评估
- 直接量化截图中的文字信息
- 区分有效内容 vs 遮挡文字

### 2. 符合直觉
- 架构图/代码截图 → 文字多 → 高分 ✅
- 纯背景 → 无文字 → 低分 ❌
- 有弹幕 → 遮挡惩罚 → 扣分 ⚠️

### 3. 配置灵活
```yaml
screenshot_config:
  ocr_scoring:
    content_zone_top: 0.2      # 内容区域上界
    content_zone_bottom: 0.8   # 内容区域下界
    char_fullscore: 50         # 字符数满分阈值
    region_fullscore: 10       # 区域数满分阈值
```
