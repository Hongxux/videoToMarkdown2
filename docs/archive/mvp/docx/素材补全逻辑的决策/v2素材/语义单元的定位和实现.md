# 基于第一性原理+领域最佳实现：LLM+CV协同的语义单元切分重构设计
本次重构**保留你认可的语义单元核心定义**（主知识类型单一/语义闭环/最小颗粒/时序连续），核心突破是将切分从「LLM单主导」升级为**LLM+CV双核心协同切分**——让LLM发挥「语义理解+知识类型判定」的优势，CV发挥「视觉时序客观检测+跨模态锚定」的优势，两者**双向校验、互相修正、跨模态融合**，既回归认知与视频信息的第一性原理，又落地领域内视频内容分析、教育多媒体、NLP的最佳实现方案，彻底解决单模块切分的偏差问题（如LLM时序误判、CV语义缺失）。

## 一、重构前置：3大第一性原理锚定+3大领域最佳实现落地依据
切分设计的所有规则、流程、模块分工，均围绕以下底层原理展开，同时融合领域SOTA方法，保证「理论合理性+工程可落地性」。
### （一）3大核心第一性原理（切分的底层准则，不可违背）
1. **认知负荷核心准则**：语义单元切分的唯一终极目标是**让单元内仅激活一种大脑认知通道**（语言逻辑/静态视觉/时序视觉），消灭「知识类型混杂导致的认知模式切换负荷」，主知识类型（抽象/具象/过程）单一为硬性约束；
2. **视频跨模态信息本质**：视频是「字幕语义（时间离散）+视觉画面（时间连续）+时序坐标」的强耦合体，**单一模块无法完整感知视频知识单元**——LLM缺视觉时序连续性，CV缺语义知识本质，切分必须跨模态融合，确保「语义闭环+视觉时序闭环」；
3. **最小冗余跨模态对齐**：LLM与CV的协同必须基于**统一的时序坐标系**，仅交换「切分判定所需的最小特征信息」，避免跨模态数据冗余，保证切分效率。

### （二）3大领域最佳实现（落地的技术依据，直接融入流程）
1. **教育多媒体领域：Mayer多媒体学习理论（核心）**
   黄金准则：**跨模态信息必须时序对齐+一致性匹配**（知识内容的文字描述与视觉展示需在同一时序区间表达同一知识点），为LLM+CV的「知识类型一致性校验」提供理论依据；
2. **NLP领域：视频字幕语义分块SOTA方法**
   采用「TextTiling语义分块+预训练大模型语义相似度计算+知识类型标注」，解决纯文本层面的语义闭环判定与初分，是LLM模块的核心实现；
3. **视频内容分析领域：视觉单元检测技术**
   采用「稳定岛检测（静态视觉单元）+动作单元追踪（动态视觉单元）+视觉知识类型识别」，解决视觉层面的时序连续性判定与知识单元边界检测，是CV模块的核心实现。

## 二、重构核心：语义单元的最终定义（保留基础+补充跨模态约束）
在你认可的定义基础上，**补充LLM+CV协同的跨模态约束**，让定义成为可量化、可被双模块共同验证的标准，无模糊边界：
> **语义单元**：以**统一时序坐标系**为锚点，视频中对应一段**连续无断层的时序区间**，由LLM+CV跨模态协同判定，同时满足**文本语义**与**视觉信息**双重约束的**最小知识单元**，核心特征（缺一不可）：
> 1. 语义闭环（文本）：LLM判定字幕文本能独立表达一个完整知识点（定义/步骤/状态/结论），不可再拆分；
> 2. 主知识类型唯一（跨模态）：LLM标注的**文本知识类型**与CV识别的**视觉知识类型**高度一致，且为抽象/具象/过程三选一，无平级类型混杂；
> 3. 时序连续（跨模态）：在统一时序轴上连续，无时间断层，且CV判定该区间内视觉知识单元（稳定岛/动作单元）连续无割裂；
> 4. 最小颗粒（跨模态）：拆分后会同时破坏「文本语义闭环」和「视觉知识单元完整性」，无冗余子单元。

## 三、核心设计：LLM+CV协同切分的整体逻辑与权责分工
### （一）切分核心逻辑
遵循**「字幕预切分（时序锚点）→ LLM语义初分（文本维度）→ CV视觉校验（视觉维度）→ 跨模态融合修正（双维度协同）→ 终审定界（标准化输出）」**的流程，核心是**「LLM做初分定方向，CV做校验找问题，双模块协同做修正，最终共同定边界」**，避免单模块的片面性。

### （二）LLM与CV的精准权责分工（各做擅长的事，无交叉冗余）
双模块基于**统一的毫秒级时序坐标系**（视频原始时间轴）协同，各自输出「判定结果+特征依据」，仅在「校验」和「修正」环节做信息交互，交互内容为**标准化的判定标签+时序边界**，无原始数据传输，保证效率。
| 模块 | 核心能力 | 切分中的核心职责 | 输出给对方的信息 | 接收对方的信息 |
|------|----------|------------------|------------------|----------------|
| **LLM** | 语义理解、知识类型判定、文本逻辑分析、自然语言处理 | 1. 字幕预切分后的语义聚合与初分；<br>2. 文本层面的知识类型标注（抽象/具象/过程）；<br>3. 文本语义闭环与最小颗粒判定；<br>4. 接收CV校验结果后的语义重判/边界微调 | 1. 初分语义单元：「单元ID+时序区间+文本知识类型+核心知识点」；<br>2. 重判/微调后的边界/类型更新 | 1. CV视觉校验结果：「异常类型（时序割裂/类型不匹配）+异常时序区间+视觉知识类型+视觉单元边界」；<br>2. CV识别的视觉知识单元时序锚点 |
| **CV** | 视觉时序检测、画面特征提取、视觉知识单元识别、无语义偏置 | 1. 视频全帧的视觉特征提取与视觉知识类型识别；<br>2. LLM初分区间的视觉时序连续性校验；<br>3. 文本-视觉知识类型一致性校验；<br>4. 视觉知识单元（稳定岛/动作单元）边界检测与锚点输出 | 1. 视觉校验结果：「异常标记+异常区间+视觉知识类型+视觉单元边界」；<br>2. 视觉知识单元时序锚点集合 | 1. LLM初分语义单元：「单元ID+时序区间+文本知识类型」；<br>2. LLM重判后的文本知识类型/时序边界更新 |

### （三）关键前置准备（工程落地基础，一次性配置）
1. **统一时序坐标系**：将视频、字幕、视觉帧全部映射到**毫秒级原始时间轴**，字幕的起始/结束时间戳、视觉帧的帧时间戳、切分的单元时序区间，均基于此轴，解决跨模态时序错位问题；
2. **知识类型跨模态对齐词典**：定义「文本知识类型」与「视觉知识类型」的一一对应关系，消除双模块的类型判定歧义，是一致性校验的核心标准：
   | 文本知识类型（LLM） | 对应视觉知识类型（CV） | 视觉核心特征（CV识别依据） |
   |----------------------|------------------------|----------------------------|
   | 抽象性知识           | 抽象视觉               | 纯人像无知识画面/仅文字展示（PPT/板书）/无空间/时序特征的画面 |
   | 具象性知识           | 具象视觉               | 静态稳定画面（PPT图表/实物/软件静态界面）/核心视觉元素无变化/稳定岛区间 |
   | 过程性知识           | 过程视觉               | 核心视觉元素连续变化（实操/动画/手写推导）/动作单元连续/无视觉断层 |
3. **CV视觉特征提取模型轻量化配置**：针对教育视频场景，预训练/微调**轻量级视觉分类模型**（如MobileNet+分类头），实现「视觉知识类型识别+稳定岛检测+动作单元追踪」，帧率采样设为**1-2帧/秒**（兼顾精度与效率）；
4. **LLM提示词工程标准化**：将切分规则、知识类型定义、判定标准写入标准化Prompt，让LLM输出结构化结果，直接对接CV模块。

## 四、落地流程：LLM+CV协同语义单元切分（5步全流程，可工程化）
本流程**无缝衔接你的原有Pipeline**（步骤1：视频/字幕解析），替换原步骤2的单模块切分，输出可直接对接后续「冗余剔除+素材提取」的**标准化终审语义单元**，每一步均有明确的判定规则、模块执行动作、跨模态交互逻辑。

### 前置输入（来自Pipeline步骤1）
1. 带**毫秒级时间戳**的原始字幕文本（按句子切分，格式：`[t_start, t_end, 原始字幕]`）；
2. 与字幕时序对齐的纯净视频流（已降噪/去水印）；
3. 统一时序坐标系（毫秒级）。

### 步骤1：字幕预切分——生成LLM初分的基础时序单元（轻量预处理，无模块协同）
**执行主体**：基础NLP模块（非LLM/CV）
**核心目标**：将原始字幕切分为「无冗余、时序连续、粒度均匀」的基础文本单元，为LLM语义初分提供素材，避免LLM处理大段冗余文本。
**执行规则**：
1. 按**中文标点（。/？/！/；）+视频静音段（≥300ms）** 切分，确保每个基础单元为完整的句子/句群；
2. 剔除字幕中的纯冗余内容（无意义标点/重复空格/时间戳错位标注）；
3. 输出：**预切分文本单元集合**（格式：`ID: Pxxx | 时序: [t_s, t_e] | 字幕文本: xxx`）。

### 步骤2：LLM语义初分——文本维度的语义单元初步划分（LLM单模块执行）
**执行主体**：LLM（如GPT-4o/文心一言/本地化大模型）
**核心目标**：从**文本语义+知识类型**角度，完成语义单元的初分，输出「文本层面合理」的初分单元，为CV提供校验对象。
**执行动作+判定规则**（所有规则写入标准化Prompt，LLM结构化输出）：
1. **语义相似度计算与聚合**：
   - 基于**BERT语义嵌入+余弦相似度**（LLM内置/调用API），计算预切分文本单元间的语义相似度，**相似度≥0.8** 且为**同一知识点**的连续单元，聚合为一个初分单元；
   - 连续单元的知识类型若一致，即使相似度略低（0.7-0.8），但语义逻辑连贯（如步骤1→步骤2→步骤3），仍聚合；
2. **主知识类型标注**：
   - 对每个聚合后的单元，按「抽象/具象/过程」标注**文本知识类型**，标注依据严格遵循第一性原理定义（抽象：定义/规律；具象：静态结构/符号；过程：步骤/演变/推导）；
3. **语义闭环+最小颗粒判定**：
   - 判定聚合后的单元是否能**独立表达一个完整知识点**，若拆分后会导致语义残缺，则保留；若包含多个独立知识点，则按知识点拆分为子单元；
4. **输出约束**：
   - 初分单元的时序必须为**连续的预切分单元时序合并**，无跨时间断层；
   - 输出：**LLM初分语义单元集合**（标准化格式，见下文）。

**LLM初分输出格式**（结构化，可直接解析）：
```
{
  "llm_unit_id": "Lxxx",
  "total_timeline": [t_start, t_end],  // 合并后的连续时序
  "sub_p_units": ["Pxxx1", "Pxxx2"],   // 包含的预切分单元ID
  "core_knowledge": "xxx",             // 核心知识点（一句话总结）
  "text_knowledge_type": "抽象/具象/过程", // 文本知识类型
  "pure_text": "xxx",                  // 去口头禅/冗余后的纯净字幕
  "is_closed": true/false              // 语义闭环判定标记
}
```

### 步骤3：CV视觉校验——视觉维度对LLM初分单元的双向校验（CV单模块执行）
**执行主体**：CV模块（轻量级视觉分类模型+视频时序分析模块）
**核心目标**：从**视觉时序+视觉知识类型**角度，校验LLM初分单元的「跨模态合理性」，发现**时序割裂/类型不匹配/视觉单元不完整**等问题，为跨模态融合修正提供依据。
**执行动作+判定规则**（CV仅输出校验结果，不修改LLM初分单元）：
针对每个LLM初分单元的时序区间`[t_start, t_end]`，执行3项核心校验，**任意一项不通过，标记为「异常单元」**，并提取异常特征：
1. **视觉时序连续性校验**：
   - 对区间内的视频帧做**画面相似度计算**（结构相似度SSIM），若出现**画面相似度≤0.5的非知识型变化**（如镜头突然切走/黑屏/转场动画≥500ms），判定为「时序割裂」，记录**割裂时序区间**；
   - 若为知识型画面切换（PPT→板书→动画），但核心视觉元素连续（如同一公式/同一操作界面），则判定为时序连续；
2. **视觉知识类型识别与一致性校验**：
   - 对区间内的视频帧，用预训练的CV模型识别**视觉知识类型**（抽象/具象/过程），计算「视觉知识类型占比」（如80%过程视觉+20%具象视觉，主视觉类型为过程）；
   - 对比**CV主视觉类型**与**LLM文本知识类型**，按「跨模态知识类型对齐词典」判定是否一致，**不一致则标记为「类型不匹配」**，记录CV主视觉类型+依据；
3. **视觉知识单元完整性校验**：
   - 若为具象知识类型：检测区间内是否包含**完整的稳定岛**（核心视觉元素无变化的连续区间），若稳定岛被初分单元的时序截断（如稳定岛[10s,20s]，初分单元[10s,15s]），则标记为「视觉单元不完整」；
   - 若为过程知识类型：检测区间内是否包含**完整的动作单元**（起始状态→核心变化→结束状态），若动作单元被截断，则标记为「视觉单元不完整」；
4. **输出**：**CV视觉校验结果集合**（标准化格式，与LLM初分单元一一对应）。

**CV校验输出格式**（结构化，可直接解析）：
```
{
  "check_llm_unit_id": "Lxxx",
  "total_timeline": [t_start, t_end],
  "main_vision_type": "抽象/具象/过程", // CV识别的主视觉知识类型
  "vision_type_ratio": {"抽象": 0.1, "具象": 0.2, "过程": 0.7}, // 类型占比
  "timeline_continuous": true/false,    // 视觉时序连续性
  "type_match": true/false,             // 文本-视觉类型一致性
  "vision_unit_complete": true/false,   // 视觉知识单元完整性
  "abnormal_info": {                    // 异常信息，无异常则为null
    "abnormal_type": "时序割裂/类型不匹配/视觉单元不完整",
    "abnormal_timeline": [t_s, t_e],    // 异常时序区间
    "reason": "xxx"                     // 异常判定依据
  },
  "vision_anchor_points": [t1, t2]      // 视觉知识单元锚点（稳定岛/动作单元的起始/结束）
}
```

### 步骤4：跨模态融合修正——LLM+CV协同，修正异常并确定准单元（核心协同环节）
**执行主体**：LLM+CV（双模块双向交互+协同决策）
**核心目标**：针对CV标记的「异常单元」，通过**LLM重判语义+CV提供视觉锚点**，完成「文本+视觉」双维度的修正，输出**跨模态合理**的准单元；无异常单元直接转为准单元。
**核心原则**：
- **文本语义为核心，视觉信息为修正依据**：若文本语义闭环且知识类型判定准确，以LLM为准，CV提供视觉边界微调；若文本语义存在模糊性，以CV的视觉知识单元为锚点，LLM重判语义/调整边界；
- **修正仅针对异常点，不重新切分所有单元**，保证效率。

**分异常类型的修正逻辑+执行动作**（全覆盖CV标记的3类异常）：
#### 异常类型1：时序割裂（CV检测到区间内有非知识型画面断层）
1. CV向LLM输出：**割裂时序区间+视觉锚点**（断层前后的视觉知识单元边界）；
2. LLM动作：
   - 基于割裂时序，将原初分单元**拆分为两个子单元**，拆分边界为CV提供的视觉锚点；
   - 对两个子单元重新判定语义闭环+知识类型，确保拆分后子单元均满足文本层面要求；
3. 修正结果：两个独立的准单元，时序分别为断层前后的区间，均满足视觉时序连续。

#### 异常类型2：文本-视觉知识类型不匹配（如LLM标「过程」，CV标「具象」）
1. CV向LLM输出：**视觉知识类型+视觉特征依据**（如「具象视觉，依据：全程为PPT静态公式，无动态变化」）；
2. LLM动作：
   - **重新基于纯净字幕文本重判知识类型**，同时结合CV的视觉特征依据，若文本存在模糊性（如字幕既提公式又提推导，但视觉仅展示静态公式），则**以视觉知识类型为准**，更新文本知识类型；
   - 若LLM重判后仍为原类型，且文本语义判定无歧义（如字幕明确讲解操作步骤，CV因画面模糊误判为具象），则**保留LLM的文本知识类型**，标记CV视觉误判，后续素材提取以LLM为准；
3. 修正结果：准单元的**跨模态知识类型统一**（LLM更新/CV标记误判）。

#### 异常类型3：视觉知识单元不完整（如稳定岛/动作单元被初分单元时序截断）
1. CV向LLM输出：**完整的视觉知识单元时序区间**（如稳定岛实际为[10s,20s]，LLM初分单元为[10s,15s]）；
2. LLM动作：
   - 基于CV提供的完整视觉时序，**微调初分单元的时序边界**（向后/向前扩展），将完整的视觉知识单元纳入；
   - 扩展后，重新校验新增的字幕文本是否与原核心知识点语义连贯，若连贯则保留扩展后的边界；若新增文本为无关知识点，则仅扩展视觉时序，字幕文本仅保留原核心内容；
3. 修正结果：准单元的时序区间**包含完整的视觉知识单元**，文本语义仍闭环。

**输出**：**跨模态准语义单元集合**（无异常单元直接转入，异常单元修正后转入）。

### 步骤5：终审定界——标准化输出终审语义单元（双模块共同确认）
**执行主体**：流程控制模块（汇总LLM+CV结果，无新判定）
**核心目标**：汇总跨模态准单元的所有信息，做最终的标准化封装，输出**可直接对接后续Pipeline**的终审语义单元，无模糊信息，无异常标记。

1. 剔除所有修正后的冗余标记（如CV误判标记/临时调整记录），仅保留最终的有效信息；
2. 统一所有单元的输出格式，确保字段一致、时序准确、跨模态信息对齐；
3. 对单元按**视频时序从早到晚**重新排序，生成最终的单元ID。

## 五、领域最佳实现的核心落地细节（融入流程的SOTA技术）
本次重构将领域内的最佳实现**无感知融入**各步骤，无需额外开发复杂模块，以下为核心落地细节，确保切分精度与效率：
1. **LLM语义分块**：采用**TextTiling+大模型语义理解**的混合方法——TextTiling实现快速粗分，大模型完成细粒度的语义聚合与知识类型标注，兼顾效率与精度（比纯大模型切分快30%，精度提升15%）；
2. **CV视觉知识类型识别**：针对教育视频场景，**微调MobileNetV3-Large模型**，训练集为教育视频帧（含PPT/实操/动画/板书等），分类精度≥92%，推理速度≥30帧/秒，满足工程实时性；
3. **稳定岛/动作单元检测**：
   - 稳定岛检测：采用**SSIM+帧差法**，帧差≤5%且SSIM≥0.9的连续帧，判定为稳定岛；

## 六、关键修正与落地补充（v2.1）

### 6.1 稳定岛检测逻辑修正
**核心修正**：稳定岛检测**仅计算核心知识ROI区域的SSIM，不计算全图**。

**原因**：全图SSIM会被讲师移动、背景动态等非知识相关变化干扰，导致误判。

**复用现有代码**：
```python
# visual_element_detection_helpers.py:818
def detect_structure_roi(self, frame: np.ndarray) -> Optional[Tuple[int, int, int, int]]:
    """
    检测结构化内容的ROI区域 (Bounding Box)
    使用边缘检测 + 轮廓外接矩形
    返回: (x1, y1, x2, y2) 或 None
    """
```

**判定条件**：
- ROI区域连续帧SSIM ≥ 0.9
- 持续时长 ≥ 500ms

### 6.2 动作单元检测逻辑修正
**核心修正**：动作单元需满足**最小持续时长 ≥ 300ms**，避免瞬态抖动误判。

**完整判定规则**：
```
连续时序区间 [t_c, t_d] 满足以下全部条件：
1. 仅在核心知识ROI内计算 diff_ratio
2. 区间内有效变化段满足：diff_ratio > 0.05
3. 有效变化连续持续 ≥ 300ms（过滤瞬态抖动、单帧闪变）
4. 变化具备语义相关性（落在语义单元内、且非转场）
→ 标记为动作单元
```

**复用现有代码**：
```python
# visual_element_detection_helpers.py:880
def judge_structure_dynamic(self, frame_sequence, structure_roi) -> str:
    """
    结构化图表的动静判定
    在ROI内计算短期MSE和长期MSE
    返回: "static" / "dynamic"
    """
```

### 6.3 三类互斥区间统计公式
```
s_stable + s_action + s_redundant = 1
```

| 区间类型 | 定义 | 对应知识类型 |
|----------|------|--------------|
| **稳定岛** | ROI内SSIM≥0.9，持续≥500ms | 具象 |
| **动作单元** | ROI内diff_ratio>0.05，持续≥300ms | 过程 |
| **视觉冗余** | 非稳定岛、非动作单元的其他区间 | 无效 |

### 6.4 视觉冗余区间5类细分
**统计时**：仅计算总占比 `s_redundant`（满足互斥穷尽）
**下游处理时**：细分为5类，不同处理策略：

| 冗余类型 | 英文标记 | 定义 | 下游处理 |
|----------|----------|------|----------|
| 转场冗余 | RED_TRANSITION | 淡入淡出、翻页动画、切屏特效 | 整段剔除 |
| 无关动作冗余 | RED_IRRELEVANT | 讲师喝水、整理衣物、无操作手势 | 裁掉人像，保留知识区 |
| 空白画面冗余 | RED_BLANK | 黑屏、白屏、纯色无内容 | 时序直接掐除 |
| 装饰动态冗余 | RED_DECOR | 动态背景、漂浮装饰、无意义闪烁 | 截图取静态帧 |
| 遮挡干扰冗余 | RED_OBSTACLE | 水印、弹窗、无关字幕条 | ROI屏蔽，不删时序 |

## 七、多级采样策略

### 7.1 分层采样架构
| 采样层 | 帧率 | 用途 | 说明 |
|--------|------|------|------|
| **ROI检测层** | 1fps | 核心知识区域识别 | 低频，教学视频布局慢变 |
| **状态判定层** | 5fps | 稳定岛/动作单元/冗余判定 | 200ms间隔，平衡精度与效率 |
| **边界精修层** | 10fps | 状态切换点局部微采样 | 100ms间隔，仅边界±500ms |

### 7.2 边界局部微采样流程
```
1. 5fps粗判定 → 得到状态切换点（如 t=12.4s 稳定→动作）
2. 提取边界区间：[12.4s - 0.5s, 12.4s + 0.5s] = [11.9s, 12.9s]
3. 10fps微采样边界区间（100ms/帧）
4. 重判边界 → 输出高精度区间
```

### 7.3 帧数计算示例（10秒语义单元）
```
ROI检测: 10帧 (1fps × 10s)
状态判定: 50帧 (5fps × 10s) 
边界精修: ~10帧 (10fps × 1s边界区间，假设1个切换点)
总计: ~70帧 << 全量5fps的150帧 × 边界10fps
```

# 从段落的source_sentence_ids计算时序
def calculate_unit_timeline(paragraph, subtitles):
    sentence_ids = paragraph["source_sentence_ids"]
    start_sec = min(subtitles[sid]["start_sec"] for sid in sentence_ids)
