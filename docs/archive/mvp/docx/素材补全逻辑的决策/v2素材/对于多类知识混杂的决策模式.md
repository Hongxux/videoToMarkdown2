# 视觉知识类型混杂场景的决策规则 (v2.0)

## 一、核心前提：统计维度与判定原则

### 1.1 用时长占比统计（非帧数量）
```
p_abstract = 抽象视觉总时长 / 单元总时长
p_concrete = 具象视觉总时长 / 单元总时长  
p_process  = 过程视觉总时长 / 单元总时长
```
满足：`p_abstract + p_concrete + p_process = 1`

### 1.2 核心判定原则
1. **语义主权原则**：LLM文本知识类型为第一决策依据，视觉只做辅助佐证
2. **认知通道唯一性**：主类型必须保证单一认知通道为主
3. **最小扰动原则**：能判定就不标记混杂，减少不必要的重切分
4. **鲁棒性原则**：阈值适配教学视频正常混合场景

## 二、三级阶梯式阈值体系

### 2.1 固定阈值常量
| 阈值名称 | 符号 | 数值 | 含义 |
|----------|------|------|------|
| 绝对主导阈值 | θ_abs | **0.6** | 某类≥此值，无争议定为主类型 |
| 相对主导阈值 | θ_rel | **0.4** | 最大占比下限 |
| 相对差值阈值 | θ_diff | **0.15** | 最大与次大的最小差距 |

### 2.2 四步判定流程
```python
# Step 1: 排序
max_p, max_cls = 最大占比及类型
second_p = 次大占比

# Step 2: 绝对主导
if max_p >= 0.6:
    return max_cls

# Step 3: 相对主导
if max_p >= 0.4 and (max_p - second_p) >= 0.15:
    return max_cls

# Step 4: 混杂
return VISION_MIXED
```

## 三、稳定岛/动作单元/冗余区间统计

### 3.1 三类互斥区间定义
```
s_stable + s_action + s_redundant = 1
```

| 区间类型 | 定义 | 对应知识类型 |
|----------|------|--------------|
| **稳定岛** | ROI内SSIM≥0.9，持续≥500ms | 具象 |
| **动作单元** | ROI内diff_ratio>0.05，持续≥300ms | 过程 |
| **视觉冗余** | 非稳定岛、非动作单元的其他区间 | 无效 |

### 3.2 稳定岛检测规则
**仅计算核心知识ROI区域，不计算全图SSIM**：
- ROI检测逻辑复用 `detect_structure_roi()`
- 判定条件：`ROI区域连续帧SSIM ≥ 0.9 且持续 ≥ 500ms`

### 3.3 动作单元检测规则
**完整判定规则（最终版）**：
```
连续时序区间 [t_c, t_d] 满足以下全部条件：
1. 仅在核心知识ROI内计算 diff_ratio
2. 区间内有效变化段满足：diff_ratio > 0.05
3. 有效变化连续持续 ≥ 300ms（过滤瞬态抖动、单帧闪变）
4. 变化具备语义相关性（落在语义单元内、且非转场）
→ 标记为动作单元
```

### 3.4 视觉冗余区间分类
**统计时**：仅计算总占比 `s_redundant`
**下游处理时**：细分为5类，不同处理策略：

| 冗余类型 | 英文标记 | 定义 | 下游处理 |
|----------|----------|------|----------|
| 转场冗余 | RED_TRANSITION | 淡入淡出、翻页动画、切屏特效 | 整段剔除 |
| 无关动作冗余 | RED_IRRELEVANT | 讲师喝水、整理衣物、无操作手势 | 裁掉人像，保留知识区 |
| 空白画面冗余 | RED_BLANK | 黑屏、白屏、纯色无内容 | 时序直接掐除 |
| 装饰动态冗余 | RED_DECOR | 动态背景、漂浮装饰、无意义闪烁 | 截图取静态帧 |
| 遮挡干扰冗余 | RED_OBSTACLE | 水印、弹窗、无关字幕条 | ROI屏蔽，不删时序 |

## 四、联合裁决规则（视觉混杂时）

当 `T_vision = VISION_MIXED` 时，以 LLM 文本类型为核心：

| T_text | 条件 | 结果 |
|--------|------|------|
| 抽象 | p_abstract + p_concrete ≥ 0.7 | 对齐为抽象，兼容通过 |
| 具象 | s_stable ≥ 0.5 | 对齐为具象，兼容通过 |
| 过程 | s_action ≥ 0.4 | 对齐为过程，兼容通过 |
| 都不满足 | - | 跨模态冲突，打回重切分 |

## 七、跨模态冲突时的重切分机制

### 7.1 核心原则
1. **LLM绝对主导权**：CV提供锚点，LLM做最终决策（拆分/微调/保持）
2. **最小切分原则**：仅处理冲突区间，不重新切分整个视频
3. **锚点对齐原则**：边界贴合CV视觉锚点（偏差≤300ms）
4. **可溯源原则**：所有操作记录日志

### 7.2 重切分触发条件
双条件同时满足时才启动重切分：
1. 跨模态一致性校验判定为「不一致/无法联合裁决」
2. CV能提供清晰的可拆分视觉锚点

### 7.3 CV输出冲突包格式
```json
{
  "conflict_unit_id": "Uxxx",
  "conflict_timeline": [t_s, t_e],
  "vision_anchors": [t1, t2, t3],
  "vision_anchor_type": ["稳定岛结束", "动作单元起始", "冗余区间开始"],
  "conflict_reason": "...",
  "vision_valid_timeline": [[t_s, t1], [t2, t_e]]
}
```

### 7.4 LLM三选一决策
| 决策 | 触发条件 | 执行动作 |
|------|----------|----------|
| **强制拆分** | 字幕存在类型边界 + 锚点偏差≤300ms | 按视觉锚点拆分为2个子单元 |
| **边界微调** | 语义闭环但包含冲突区间 | 收缩/扩展时序边界 |
| **保持原判定** | 语义无歧义 or CV误判 or 冲突≤20% | 标记`cross_modal_suspected` |

### 7.5 重切分输出格式
```json
{
  "original_unit_id": "Uxxx",
  "decision_type": "split/adjust/keep",
  "new_units": [
    {
      "final_unit_id": "Uxxx-1",
      "total_timeline": [t_s1, t_e1],
      "core_knowledge": "xxx",
      "final_knowledge_type": "抽象/具象/过程",
      "pure_text": "xxx",
      "cross_modal_suspected": false,
      "suspected_reason": null
    }
  ],
  "split_anchor": t_x,
  "adjust_reason": "xxx"
}
```
