## 核心锚定：底层原则与增量更新边界

### 第一性原理（三重锚定）

1. **工程增量第一性**：**复用现有高度优化的原子能力为绝对前提**，最小化代码重构（仅重构上层调度 + 增量新增 / 改进逻辑），避免破坏底层算子的稳定性和性能优势，重构的核心是 **“解耦而非重写，嵌入而非替换”**。
2. **信息处理第一性**：所有新增 / 改进逻辑均围绕 **“最小化无效计算，保留认知有效信息”**，早融合、SBD、去重、块分类的设计均以 “剔除冗余块 / 帧，精准标记信息类型” 为目标，不偏离教学视频结构化的核心价值。
3. **防误判第一性**：**单一特征易产生伪阳 / 伪阴，多模态多特征交叉验证是唯一解**（CV 领域视频分析的底层共识），所有判定逻辑均需从 “单特征阈值判定” 升级为 “多特征协同判定”，结合上下文消除孤立判断的误差。

### 领域最佳实践参考

1. **工业级 SBD 最佳实践**：硬切变采用 **“像素差异 + 颜色直方图 + 时域连续性”**多特征判定，软切变采用**“帧间差异渐变趋势 + 时域窗口统计”** 检测，避免单一阈值的误判（YouTube / 抖音视频切分的标配方案）。
2. **EdTech 早融合实践**：视听融合前置到切分阶段的核心是 **“轻量特征融合，不引入昂贵计算”**，仅用音频静音段、视觉静态特征等轻量信息做冗余过滤（学堂在线 /edX 的视频预处理方案）。
3. **块级分类最佳实践**：放弃 “单一多数投票”，采用 **“主特征加权投票 + 次要特征标记”** 的多类型标注方案，适配教学视频 “混合信息块占比超 30%” 的特征（教育视频理解的通用方案）。
4. **感知哈希去重实践**：跨段去重需结合 **“视觉 PHash + 语义轻量特征”**，避免仅靠视觉特征误判 “同画面但新信息” 的场景（文档 / 视频去重的工业级方案）。

### 增量更新的核心边界

**保留所有底层 Extractor / 算子的代码和能力**，仅对**上层调度 Pipeline**做重构，新增模块（SBD、PHash 去重）采用 **“解耦式插件化实现”**，改进逻辑（早融合、块级分类）采用**“增量嵌入现有算子，新增分支逻辑而非修改主逻辑”** 的方式，实现 “可灰度、可回滚、可演进” 的增量改造。

## 一、当前改造方向中存在的**核心问题**（基于原子能力复用与工程落地）

所有问题均围绕 “从「基于句子找画面」转向「先切分画面再对齐语义」的核心架构切换” 产生，且直接关联**伪阳 / 伪阴的诱因**和**计算效率的潜在损耗**，需在增量更新中优先解决：

### 1. 架构层面：现有动态检测与新增 SBD 的**耦合风险**

现有`DynamicDecisionEngine`的`evaluate_action_windows`是用于 **“动作窗口软切分”**的细粒度逻辑，若直接改造为 SBD 的软切变检测，易导致**“底层动作检测” 与 “上层镜头切分” 的逻辑耦合 **，后续修改动作检测参数会直接影响 SBD 的稳定性，违背 “原子能力解耦” 的工程原则。

### 2. 逻辑层面：单一阈值判定的**泛化性缺陷**（伪阳 / 伪阴的核心诱因）

用户提出的改进逻辑中，多个判定规则仍依赖 **“单一阈值 + 单特征”**，缺乏多特征协同和上下文关联，是落地后产生伪阳 / 伪阴的核心原因：

- SBD 硬切变：仅`MSE > Threshold + ColorHistogram_Diff > Threshold`，未考虑**时域连续性**（如 PPT 上光标微小移动导致单帧 MSE 突变，会被误判为硬切，伪阳性）；
- 冗余块判定：仅`Visual_Static + Audio_Gap>2s/填充词`，未考虑**视觉微动的有效性**（如老师手写推演时的视觉微动 + 短暂音频停顿，会被误判为冗余，伪阳性），也未考虑**音频有效但视觉静态**的场景（如老师讲解静态 PPT 的有效音频 + 视觉静态，被误判为冗余，伪阴性）；
- 块级分类：仅`Majority_Vote(Frame_Types)`的简单多数投票，未考虑**帧的信息密度权重**（如 5s 块中 1s 的手写推演（高信息密度）+4s 的 PPT 静态（低信息密度），被误标为 “空间结构块”，伪阴性）。

### 3. 去重层面：纯视觉 PHash 的**片面性**（跨段去重的伪阳核心）

仅引入 Perceptual Hash 做跨段去重，未结合**语义轻量特征**，会导致 **“同视觉画面但含新语义 / 视觉信息” 的场景被误判为冗余 **（如老师切回同一页 PPT 但新增了公式标注 / 讲解了新内容，PHash 一致被误判为回溯冗余，伪阳性），违背 “保留认知有效信息” 的核心目标。

### 4. 融合层面：早融合的**计算量平衡风险**

将视听融合前置到切分阶段的核心价值是 “提前过滤冗余块，减少后续昂贵计算”，但若**融合逻辑引入过多特征计算**（如提前做 OCR/Embedding），会违背 “轻量早融合” 的原则，导致切分阶段计算量激增，抵消冗余过滤的效率收益。

### 5. 调度层面：画面切分与语义对齐的**时序同步问题**

现有`ASRExtractor`输出带时间戳的字幕片段，但若**SBD 切分的视频块时间戳与 ASR 字幕的时间戳未做精准对齐**，会导致后续 “画面块 + 语义信息” 的融合出现错位（如视频块对应错误的字幕片段），影响块级分类和信息结构化的准确性。

### 6. 类型映射层面：现有语义分类与目标信息类型的**精准映射缺失**

现有`classify_knowledge_type`的`Process/Spatial/Abstract`与目标的 **“过程 / 空间 / 描述”**主导类型存在**语义边界模糊 **（如 “公式推演” 既属于 Spatial 也属于 Process），若直接硬映射，会导致块级分类的误判，增加伪阳 / 伪阴概率。

## 二、**增量更新方案**（复用原子能力 + 最小化重构 + 插件化新增）

核心思路：**“底层算子全复用，上层 Pipeline 重构为「视频切分→轻量视听融合→冗余过滤→块级分析→语义对齐」，新增模块插件化，改进逻辑增量嵌入”**，分**四个阶段**落地，每个阶段均为增量更新，可独立测试、灰度发布，避免一次性重构的风险。

### 前置准备：原子能力**参数适配与轻量封装**（无代码修改，仅做参数和封装层）

所有现有原子能力均保持代码不变，仅做**轻量封装**（提供统一的调用接口）和**参数适配**（针对 SBD、块分类等场景调整算子的输入 / 输出格式），最大化复用：

1. **Visual Operators**：
    
    - `_numba_mse`：封装为**帧间差异计算接口**，支持单帧 / 帧窗口的 MSE 批量计算（适配 SBD 硬切变）；
    - `calculate_edge_flux`：封装为**视觉微动检测接口**，输出 “微动类型（手写 / 光标 / 无）+ 微动强度”（适配冗余块判定、块级分类）；
    - `MathFormulaVisualDetector`：封装为**公式特征检测接口**，输出 “公式存在标记 + 公式区域坐标”（适配空间块 / 过程块的辅助标记）。
    
2. **Semantic Operators**：
    
    - `classify_knowledge_type`：封装为**帧级语义分类接口**，输出`Process/Spatial/Abstract`的**置信度分值**（而非单一类型，适配块级加权投票）；
    - `ASRExtractor`：新增**音频时间戳精细化接口**，输出 “静音段时间戳 + 有效音频段时间戳 + 填充词标记 + 音频语义粗分类（讲解 / 停顿 / 操作）”（适配早融合、冗余块判定）。
    
3. 基础架构：所有算子均提供**异步调用接口**，支持切分阶段的**并行计算**（如 SBD 切分与 ASR 音频分析并行执行）。

### 阶段 1：插件化新增**核心模块**（解耦实现，不侵入现有代码）

新增两个核心模块，均采用**插件化设计**，与现有算子解耦，可独立启停、调优，复用前置准备后的原子能力：

#### 插件 1：镜头边界检测器（Shot Boundary Detector，SBD）

**核心目标**：实现硬切变 / 软切变的精准检测，复用现有原子能力，避免与`DynamicDecisionEngine`耦合。

1. **硬切变检测**：**复用`_numba_mse`** + 新增颜色直方图计算（轻量视觉特征） + 新增**时域连续性校验**（领域最佳实践）：
    
    - 基础判定：`_numba_mse(帧t,帧t-1) > MSE阈值 AND ColorHistogram_Diff(帧t,帧t-1) > 直方图阈值`；
    - 时域校验：若上述条件满足，需校验**连续 3 帧**的差异趋势，若仅单帧突变（如光标移动），则判定为 “非硬切”，排除伪阳性；
    - 输出：硬切变时间戳 + 切变强度。
    
2. **软切变检测**：**改造`evaluate_action_windows`为独立分支逻辑**（不修改原逻辑），适配软切变的渐变特征：
    
    - 核心逻辑：滑动窗口（如 10 帧）内，`_numba_mse`的结果呈 **“持续中等波动（介于硬切和静态之间）+ 逐渐上升 / 下降”** 的渐变趋势，且无硬切变特征；
    - 复用`DynamicDecisionEngine`的帧间差异统计能力，输出：软切变时间戳 + 切变起始 / 结束帧 + 切变类型（渐变 / 溶解）；
    
3. **模块解耦**：SBD 插件仅接收视频帧序列作为输入，输出镜头切分后的**视频块列表（含时间戳 + 块 ID）**，不与现有动作检测逻辑交互，避免耦合。

#### 插件 2：感知哈希去重器（PHash Deduplication）

**核心目标**：实现跨段去重，复用视觉算子，新增轻量语义校验，避免纯视觉特征的伪阳性。

1. **视觉 PHash 计算**：引入轻量 PHash/dhash 算法（如 OpenCV 的`img_hash`），对 SBD 切分后的**视频块关键帧**（取块中第一帧）计算哈希值，实现视觉特征的快速比对；
2. **跨段比对逻辑**：维护**PHash 哈希池**，记录已处理视频块的 PHash + 块 ID + 时间戳，当新块的 PHash 与哈希池中某块的相似度＞95% 时，触发跨段校验；
3. **轻量语义校验**（避免伪阳性，核心改进）：**复用 ASRExtractor 的音频语义粗分类** + 复用`MathFormulaVisualDetector`，校验 “同 PHash 块” 是否含新信息：
    
    - 若新块的**音频语义与原块无差异**且**视觉无新公式 / 标注**，则判定为**回溯冗余块**，标记为待丢弃 / 合并；
    - 若新块含**新的音频讲解**或**视觉新标注 / 公式**，则判定为**同画面但新信息块**，保留并更新哈希池；
    
4. **输出**：视频块列表 + 冗余标记（正常 / 回溯冗余）+ 相似度分值。

### 阶段 2：增量改进**核心逻辑**（嵌入现有算子，新增分支，不修改主逻辑）

对需要改进的两个逻辑（早期视听静音融合、块级分类器），采用 **“增量嵌入分支逻辑”** 的方式，保留原逻辑（用于回滚），新逻辑基于复用的原子能力实现，适配切分后的视频块：

#### 改进 1：早期视听静音融合（Early Audio-Visual Fusion）

**核心目标**：前置到切分阶段，轻量融合，提前过滤冗余块，**复用`calculate_edge_flux`+`ASRExtractor`**，优化冗余块判定公式，避免伪阳 / 伪阴。

1. **轻量特征融合**：仅融合**视觉轻量特征**（静态 / 微动类型 + 强度）和**音频轻量特征**（静音段时长 + 音频类型 + 填充词），不引入 OCR/Embedding 等昂贵计算；
2. **优化冗余块判定公式**（多特征协同，替代单一判定）：
    
    Block_Is_Redundant=(Visual_State==Static) AND (Audio_Gap>2.0s OR Audio_Semantic=="FillerW​ord") AND (Edge_Flux==0)
    
    - 新增`Edge_Flux == 0`（复用`calculate_edge_flux`）：确保视觉是**真静止**（无手写 / 光标微动），排除 “视觉微动 + 音频短暂停顿” 的伪阳性；
    - 新增**上下文校验**：若块的前 / 后块为有效过程块 / 空间块，且该块为 “过渡性静态块”（如老师操作后的短暂停顿），则即使满足上述公式，也判定为非冗余，排除伪阳性；
    
3. **执行时机**：在 SBD 切分 + PHash 去重后执行，直接过滤**冗余块**，不进入后续的块级分析和昂贵特征提取（OCR/Embedding）；
4. **输出**：过滤后的**有效视频块列表**（含块 ID + 时间戳 + 视觉 / 音频轻量特征）。

#### 改进 2：块级分类器（Block-Level Classifier）

**核心目标**：给有效视频块打标，从 “帧级单一分类” 升级为 “块级加权分类 + 多类型标记”，**复用`classify_knowledge_type`+`calculate_edge_flux`+`MathFormulaVisualDetector`**，替代简单的 Majority_Vote。

1. **帧级特征采集**：对每个有效视频块，采集**所有帧**的特征：
    
    - 视觉特征：帧级`Edge_Flux`（微动类型）+`MathFormulaVisualDetector`（公式标记）；
    - 语义特征：帧级`classify_knowledge_type`的**Process/Spatial/Abstract 置信度分值**（而非单一类型）；
    
2. **加权投票算法**（替代简单多数投票，领域最佳实践）：
    
    - 给不同帧分配**信息密度权重**：高信息密度帧（手写 / 公式 / 操作，`Edge_Flux>阈值`）权重 = 2，低信息密度帧（静态 PPT，`Edge_Flux=0`）权重 = 1；
    - 块级分值计算：对 Process/Spatial/Abstract 三类，分别计算**帧级置信度分值 × 帧权重**的总和，得到块级三类分值；
    
3. **块类型标记**（适配教学视频混合块特征）：
    
    - 主导类型：块级分值最高的类型，映射为目标 **“过程 / 空间 / 描述”**（提前定义精准的映射规则，解决类型边界模糊问题）；
    - 次要类型：若某类分值与主导类型分值的差值＜20%，则标记为次要类型（如 “过程主导 + 空间次要”）；
    - 特殊标记：若块中存在公式，新增 **“公式相关”** 标记，适配教学场景；
    
4. **输出**：有效视频块列表 +**主导类型 + 次要类型 + 特殊标记**+ 类型分值（用于后续语义对齐的权重）。

### 阶段 3：重构上层**Pipeline 调度流程**（核心架构切换，无底层算子修改）

放弃原有的`Video -> ASR -> Sentences -> Visual Check per Sentence`流程，重构为 **“视频切分→并行视听分析→轻量早融合→块级分析→语义精准对齐”** 的新流程，所有步骤均复用 / 调用上述模块，**采用异步并行调度**，提升整体效率：

plaintext

```
整教学视频 → 【并行执行】
              → 视频流：SBD插件（硬/软切分）→ PHash去重插件 → 有效视频块列表
              → 音频流：ASRExtractor（音频分析+时间戳精细化）→ 音频轻量特征列表
          → 早期视听静音融合（轻量特征融合→过滤冗余块）→ 有效视频块列表
          → 块级分类器（加权投票→打标主导/次要类型）→ 带类型标记的视频块列表
          → 语义精准对齐（视频块时间戳与ASR字幕时间戳同步→块+字幕语义绑定）
          → 后续精细化分析（OCR/Embedding/关键帧提取，仅针对有效块）
```

#### 核心调度优化：

1. **异步并行**：视频切分与音频分析并行执行，利用 CPU/GPU 的异构资源，减少整体耗时；
2. **时间戳同步**：新增**时间戳校准模块**，基于视频帧的 PTS 时间戳和 ASR 的音频时间戳，做毫秒级对齐，避免画面与语义的错位；
3. **计算量隔离**：昂贵的精细化分析（OCR/Embedding/ 关键帧提取）仅针对**过滤后的有效视频块**，且根据块的类型做**定向计算**（如空间块做关键帧提取，过程块做动作窗口检测），从根源减少无效计算。

### 阶段 4：**类型映射与后处理**（增量新增，适配目标信息类型）

新增**类型映射规则**和**轻量后处理**，解决现有语义分类与目标信息类型的边界模糊问题，同时优化块的最终标记，为后续结构化输出做准备：

1. **精准类型映射规则**（解决边界模糊，避免误判）：
    
    |现有语义分类（classify_knowledge_type）|目标信息类型|映射条件|
    |---|---|---|
    |Process|过程|视觉微动为手写 / 操作 / 动画|
    |Spatial|空间|视觉为静态 PPT / 图表 / 公式，无有效微动|
    |Abstract|描述|视觉静态 + 音频为纯讲解，无公式 / 图表|
    |Process+Spatial（混合分值）|过程主导 + 空间次要|公式推演 / 手写推导（视觉有公式 + 微动）|
    |Spatial+Abstract（混合分值）|空间主导 + 描述次要|老师讲解静态 PPT / 图表|
    
2. **轻量后处理**：对带类型标记的视频块，做**相邻块合并**（如连续的同类型小块，总时长＜3s，合并为一个块），避免块度过细导致的结构化碎片化，适配人类认知习惯；
3. **最终输出**：**结构化视频块列表**（含块 ID + 时间戳 + 主导 / 次要类型 + 特殊标记 + 绑定的 ASR 字幕 + 视觉轻量特征），可直接对接后续的关键帧提取、过程片段切割、OCR 等精细化分析。

## 三、**系统性规避伪阳性与伪阴性**（分模块 + 多特征 + 上下文，结合原子能力）

伪阳 / 伪阴的核心诱因是 **“单一特征判定 + 无上下文 + 泛化性差的固定阈值”**，结合第一性原理的 “多模态多特征交叉验证” 和领域最佳实践的 “自适应阈值 + 上下文关联”，针对**每个核心模块 ** 给出具体的防误判策略，所有策略均复用现有原子能力，无需新增昂贵计算：

### 1. SBD 模块（硬切 / 软切）：防误判策略

#### 伪阳性场景 & 解决

- 场景 1：PPT 上光标 / 激光笔微小移动，导致单帧 MSE / 直方图突变，被误判为硬切；
- 解决：**时域连续性校验**（连续 3 帧差异趋势）+ 复用`calculate_edge_flux`，若微动类型为 “光标 / 激光笔”，则直接排除硬切判定；
- 场景 2：视频噪点导致的帧间差异波动，被误判为软切；
- 解决：软切变检测前做**帧间差异平滑处理**（移动平均），过滤噪点的微小波动，仅保留持续的渐变趋势。

#### 伪阴性场景 & 解决

- 场景 1：快速渐变的软切变（如 0.5s 渐变），被漏检；
- 解决：调整软切变的滑动窗口为**动态窗口**（根据视频帧率自适应，如 25fps 则窗口为 5 帧），提升快速软切的检测灵敏度；
- 场景 2：多镜头连续切分，后一个硬切被前一个切分的差异覆盖，被漏检；
- 解决：SBD 采用 **“逐帧检测 + 块级校验”**，检测到硬切后，跳过后续 2 帧的检测，避免连续切分的相互干扰。

#### 阈值策略：**自适应阈值而非固定阈值**

基于视频的**平均帧间差异**动态调整 MSE / 直方图阈值（如视频整体偏静态（PPT），则阈值降低；视频整体偏动态（实操），则阈值提高），替代固定阈值，提升泛化性。

### 2. 早融合冗余块判定模块：防误判策略

#### 伪阳性场景 & 解决

- 场景 1：老师手写推演时的视觉微动 + 短暂音频停顿（＜2s），被误判为冗余；
- 解决：复用`calculate_edge_flux`，若微动类型为 “手写”，则直接判定为非冗余，忽略音频停顿；
- 场景 2：老师讲解静态 PPT 时，音频含少量填充词，被误判为冗余；
- 解决：将音频判定条件从 “Audio_Semantic == Filler_Word” 升级为 “填充词占比＞80%”，避免少量填充词的误判。

#### 伪阴性场景 & 解决

- 场景 1：纯静态 PPT + 音频为长时间静音（＞2s），但因未做上下文校验，被漏判为冗余；
- 解决：新增**全局静态块统计**，若某静态块无任何前后关联的有效信息，直接判定为冗余；
- 场景 2：视觉为真静止 + 音频为纯填充词，但因 Edge_Flux 检测误差，被漏判为冗余；
- 解决：对 Edge_Flux 的检测结果做**二值化校验**（多次检测确认微动强度为 0），避免检测误差。

### 3. PHash 去重模块：防误判策略

#### 伪阳性场景 & 解决

- 场景 1：切回同一页 PPT，但老师新增了公式标注 / 讲解了新内容，PHash 一致被误判为冗余；
- 解决：**轻量语义 + 视觉校验**（复用`MathFormulaVisualDetector`检测新公式，复用 ASRExtractor 检测新音频语义），只要含新信息，即保留；
- 场景 2：PPT 画面微小缩放（如老师放大图表），PHash 略有差异，被漏判为同画面；
- 解决：PHash 比对采用**相似度阈值**（如 95%），而非完全一致，适配画面的微小缩放 / 平移。

#### 伪阴性场景 & 解决

- 场景 1：切回同一页 PPT，无任何新信息，但因 PHash 计算误差，被漏判为冗余；
- 解决：对 PHash 计算结果做**哈希值归一化**，消除因图像分辨率 / 格式导致的计算误差；
- 场景 2：跨段的同一 PPT 块，因时间间隔过长，被漏检；
- 解决：哈希池保留**所有已处理块**的 PHash，不设置过期时间，仅在视频处理完成后做一次全局跨段比对。

### 4. 块级分类器模块：防误判策略

#### 伪阳性场景 & 解决

- 场景 1：混合块（如 1s 手写 + 4s 静态 PPT），被简单多数投票误标为空间块；
- 解决：**加权投票算法**，高信息密度帧权重翻倍，提升手写 / 操作等有效帧的权重；
- 场景 2：公式推演块（Process+Spatial），被硬映射为单一类型；
- 解决：**多类型标记**，标注主导 + 次要类型，避免单一类型的误判。

#### 伪阴性场景 & 解决

- 场景 1：低信息密度的过程块（如轻微的实操动作），因帧级置信度低，被漏标为过程块；
- 解决：结合`calculate_edge_flux`的微动类型，若为 “操作 / 动画”，则直接提升 Process 类型的分值，确保标记；
- 场景 2：静态公式块，被漏标为空间块；
- 解决：复用`MathFormulaVisualDetector`，只要检测到公式，即优先标记为空间块，提升 Spatial 类型的分值。

### 通用防误判策略：**全流程阈值调优 + 测试集验证**

1. **自适应阈值体系**：放弃所有固定阈值，基于**教学视频数据集**（PPT 讲解 / 实操 / 公式推演等不同场景）做阈值的动态适配，每个场景有独立的阈值区间；
2. **构建教学视频专属测试集**：收集不同学科 / 类型的教学视频，标注真实的镜头切分、冗余块、信息类型，用于所有模块的调优和伪阳 / 伪阴率统计；
3. **后验校验**：对处理后的视频块，新增**轻量后验校验模块**，对疑似误判的块（如类型分值接近的混合块、PHash 相似度 90%-95% 的块）做人工复核（小样本），持续迭代阈值和判定规则。

## 四、**落地保障与工程最佳实践**

1. **插件化与可配置**：所有新增模块和改进逻辑均采用**可配置化**（阈值、权重、窗口大小等参数可通过配置文件修改，无需改代码），支持针对不同教学视频场景（如理科公式推演 / 文科 PPT 讲解 / 工科实操）做个性化调优；
2. **灰度发布与回滚**：重构后的 Pipeline 支持**灰度发布**，可先对小部分视频做测试，对比原流程和新流程的结果（伪阳 / 伪阴率、计算效率、信息提取准确性），若出现问题，可直接回滚到原流程，避免线上故障；
3. **性能监控**：新增**全流程性能监控模块**，统计每个步骤的耗时、计算量、冗余块过滤率、伪阳 / 伪阴率，针对耗时高的步骤做进一步的并行化 / 轻量化优化；
4. **异构计算利用**：将轻量计算（SBD、早融合、PHash）分配到**CPU**，将精细化计算（OCR、Embedding、关键帧提取）分配到**GPU**，充分利用异构资源，提升整体处理效率；
5. **增量训练（可选）**：若有足够的教学视频标注数据，可对`classify_knowledge_type`做**增量微调**，优化 Process/Spatial/Abstract 的分类精度，进一步降低块级分类的伪阳 / 伪阴率。

## 五、**核心结论**

1. **当前改造的核心问题**：并非原子能力不足，而是**架构耦合风险、单一阈值的泛化性缺陷、纯视觉去重的片面性、早融合的计算量平衡、时序同步问题**，这些问题均是伪阳 / 伪阴的核心诱因，也是工程落地的关键障碍；
2. **增量更新的核心**：**全量复用现有高度优化的原子能力**，采用 “插件化新增模块 + 增量嵌入改进逻辑 + 重构上层调度 Pipeline” 的方式，最小化代码重构，实现从 “基于句子找画面” 到 “先切分画面再对齐语义” 的架构切换，同时保证架构的可演进和可回滚；
3. **伪阳 / 伪阴的规避核心**：**放弃单一特征阈值判定，采用多模态多特征交叉验证 + 上下文关联 + 自适应阈值**，所有防误判策略均复用现有原子能力，不引入昂贵计算，兼顾准确性和效率；
4. **落地关键**：教学视频的结构化处理具有**强场景性**，需针对不同学科 / 类型的教学视频做**个性化调优**，通过可配置化、灰度发布、性能监控，实现工程化落地。

整体改造方案既遵循第一性原理的 “最小化无效计算、保留认知有效信息、复用原子能力”，又贴合领域最佳实践的 “分层处理、早融合过滤、加权分类、视觉 + 语义去重”，实现了**计算效率提升 + 信息提取准确性提升 + 伪阳 / 伪阴率降低**的三重目标。
