# 基于三类信息核心定义与“噪音剔除”的素材补全决策链（重构终稿）

本逻辑链严格基于**信息类型的第一性定义**构建，并引入**“认知增益（Cognitive Gain）”**作为唯一衡量标准。任何不承载核心知识逻辑、对认知无增益的信息，均被定义为**“噪音（Noise）”**并强制剔除。

## 一、 决策的基础：三类信息的核心定义与载体映射

这是决策系统的基石。所有判断不依赖“动静”表象，而是依赖“信息旨在传递什么”。

| 信息范畴 | 第一性核心定义 | 核心功能目标 | 载体映射 | 为什么不可替代？ |
| :--- | :--- | :--- | :--- | :--- |
| **过程类信息**<br>(Process Info) | 以**时序性动作 / 状态变化**为核心载体的信息。 | 传递动态逻辑、操作流程、状态演进。 | **视频片段**<br>(Video) | 有些逻辑是“生长”出来的（如推导、化学反应），切断时序（截图）即切断因果链条，认知无法闭环。 |
| **空间类信息**<br>(Spatial Info) | 以**视觉结构化符号**为核心载体的信息。 | 传递空间关系、量化逻辑、层级结构。 | **高清截图**<br>(Screenshot) | 空间关系的理解需要“定睛观察”，视频的流动性反而干扰注视。高清静止是最高效的接收形式。 |
| **描述类信息**<br>(Description Info) | 以**语言符号**为核心载体的信息。 | 传递概念定义、逻辑关系、步骤说明等线性叙事。 | **结构化文字**<br>(Structured Text) | 抽象概念（如“自由意志”）无形无相，强行配图（如配个大脑图）是无效冗余。文字是线性逻辑的最佳载体。 |

## 二、 关键前置层：冗余信息（噪音）的定义与剔除

在进入素材选择之前，必须先经过**“噪音过滤器”**。这是本次重构的核心升级点。

**噪音的定义**：看似属于上述三类信息范畴，但**不承载核心知识逻辑、对知识传递与认知理解无增益**，且会**挤占工作记忆容量、干扰目标信息提取**的内容。

### 1. 伪装成“过程类”的噪音
*   **现象**：画面有动静，有光标移动，有老师手势。
*   **甄别边界**：
    *   **无逻辑指向的手势**：老师挥手、整理衣领、无意义的教鞭晃动。（*剔除理由：不指向知识点，分散视觉注意力*）
    *   **无信息增量的光标**：鼠标在屏幕上空转、画圈，但未进行点击或高亮操作。（*剔除理由：增加视觉追踪负荷，无认知收益*）
    *   **单纯的复述性板书**：老师转身在黑板上写了一个已经讲完的标题，且无展开。（*剔除理由：信息与口述完全重复，无视觉增量*）
*   **剔除逻辑**：若 `Action_Density > 0` 但 `Semantic_Gain == 0`，判定为**Visual Noise**，强制**不输出视频**。

### 2. 伪装成“空间类”的噪音
*   **现象**：PPT有背景图、有装饰性图标、有老师的人像。
*   **甄别边界**：
    *   **装饰性视觉元素**：PPT角落的Logo、单纯为了美观的插图（如讲“网络”配个地球图）。（*剔除理由：非知识性结构，干扰对核心图表的聚焦*）
    *   **低信息量的过渡页**：只有“Part 1”大字或“谢谢观看”的页面。（*剔除理由：属于导航信息而非知识信息*）
    *   **人像遮挡**：老师身体挡住了公式推导的关键部分。（*剔除理由：信息破损，强行展示增加因“脑补”产生的认知负荷*）
*   **剔除逻辑**：若 `Structure_Completeness < Threshold` 或 `Knowledge_Density < Low`，判定为**Statistical Noise**，强制**不输出截图**。

### 3. 伪装成“描述类”的噪音
*   **现象**：口语中的废话、重复、无效的连接词。
*   **甄别边界**：
    *   **口语冗余**：“呃...那个...我们可以看到...对吧...”（*剔除理由：降低信噪比，增加阅读时长*）
    *   **流程性废话**：“大家翻到第5页，看左下角...”（*剔除理由：与知识内涵无关的操作指令*）
*   **剔除逻辑**：LLM 摘要层级，若 `Information_Entropy`（信息熵）极低，直接删减。

## 三、 重构后的决策逻辑链 (The Decision Logic Chain)

### Step 0: 噪音预清洗 (The Noise Filter)
*   **Input**: 原始视频片段 + ASR文本
*   **Process**:
    *   检查是否存在**核心知识逻辑**（Knowledge Logic）？
    *   视觉/听觉元素是否对理解该逻辑有**正向增益**（Positive Gain）？
*   **Output**: 
    *   No -> **Discard** (直接丢弃，不作为素材补充)。
    *   Yes -> 进入 Step 1。

### Step 1: 第一性原理类型锚定 (Type Anchoring)
基于**“认知目的”**而非**“视觉表象”**进行锚定：

*   **IF 目的 = 理解动态逻辑/状态演进 (Understanding Evolution)**
    *   *判定特征*：推导过程、实验步骤、动画演示。
    *   *决策*：锁定 **视频 (Video)**。
    *   *双保险*：若该过程最终产出了复杂的空间结构（如推导出了一个大公式），则追加 **+ 截图 (Screenshot)**。

*   **IF 目的 = 理解结构关系/量化逻辑 (Understanding Structure)**
    *   *判定特征*：图表、架构图、公式展示、代码片段。
    *   *决策*：锁定 **截图 (Screenshot)**。
    *   *注意*：即使该图表有微小的光标移动（噪音），只要目的是看结构，依然强制选截图。

*   **IF 目的 = 理解概念/定义/叙事 (Understanding Concept)**
    *   *判定特征*：概念讲解、背景介绍、逻辑串联。
    *   *决策*：锁定 **文字 (Text)**。
    *   *注意*：即使此时画面有老师在说话（动态），但视觉上全是人像无知识增量，依然强制选文字。

### Step 2: 质量与认知门槛校验 (Quality & Cognitive Threshold)
*   **视频校验**：清晰度足够吗？关键动作完整吗？（若不完整，退化为 **关键帧拼图 + 文字**）。
*   **截图校验**：分辨率足够吗？无遮挡吗？（若有遮挡且不可修复，退化为 **重绘图/LaTeX公式 + 文字**）。
*   **文字校验**：是否有歧义？（若有，保留原文并增加 **Note 提示**）。

## 四、 总结

该决策链的核心在于**“做减法”**。
我们不再试图捕捉视频里的每一个动作，而是**像过滤杂质一样过滤掉所有“非知识性”的视听元素**。

最终输出的素材，必须是经过**“认知增益”**提纯后的晶体——
*   要视频，是因为**只有动作**才能解释清楚。
*   要截图，是因为**只有结构**才能看清关系。
*   要文字，是因为**只有语言**才能定义概念。
除此之外，皆为噪音。
