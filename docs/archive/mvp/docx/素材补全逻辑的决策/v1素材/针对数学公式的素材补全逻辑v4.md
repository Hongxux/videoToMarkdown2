# 图片中数学公式的精准判定方案（基于第一性原理+领域最佳实践）
要解决**“数字/线条误判为数学公式”**的核心问题，需回归**数学公式的本质特征**（第一性原理），而非依赖“存在数字/线条”的表面特征。结合**MathPix公式识别标准、AWS Textract数学内容检测方案、OpenCV结构化视觉分析最佳实践**，构建“**本质特征量化检测+非公式排除规则+多模态交叉验证**”的三层判定体系，彻底消除误判。

## 一、 第一性原理：数学公式的本质特征（与普通数字/线条的核心区别）
数学公式的**本质不是“数字+符号”的简单叠加**，而是**“特殊符号体系+空间层级结构+逻辑语义关联”的三维复合体**。这是区分公式与普通数字/线条的根本依据，违背任一特征均不构成数学公式：

| 本质特征维度 | 数学公式的核心表现 | 普通数字/线条的表现 | 第一性原理依据 |
|--------------|--------------------|----------------------|----------------|
| **特殊符号体系** | 包含数学专属符号（非自然语言符号）：<br>1. 结构符号：分式线、根号（√）、矩阵括号（()/[]/{}）、求和号（∑）、积分号（∫）；<br>2. 关系符号：等号（=）、不等号（≠/＞/＜）、箭头（→）；<br>3. 修饰符号：上下标（²/₁）、省略号（…） | 仅包含数字（0-9）、普通横线/竖线（如表格边框、手写乱画线条）、自然语言符号（如逗号、句号） | 数学公式的符号体系是**领域专属的语义载体**，普通数字/线条无此语义功能 |
| **空间层级结构** | 符号间存在明确的**非水平线性空间关系**：<br>1. 垂直层级：上下标（如 $a^2$ 中“2”在“a”上方）、分式（分子在分式线上方，分母在下方）；<br>2. 嵌套结构：根号嵌套（$\sqrt{a+b^2}$）、括号嵌套（$(a+(b×c))$）；<br>3. 对齐结构：矩阵行列对齐、方程组大括号对齐 | 符号仅为**水平线性排列**（如“123+456”的普通文本）或**无关联的孤立线条**（如表格横线、手绘直线） | 数学公式的语义高度依赖空间位置（如 $a^2$≠$a_2$），普通数字/线条无空间语义 |
| **逻辑语义关联** | 数字、字母、特殊符号构成**有序的逻辑组合**（如 $E=mc^2$ 是能量与质量的关系，$\sum_{i=1}^n i$ 是求和逻辑），符号间不可拆分 | 数字/线条是**孤立或无逻辑关联的**（如表格中的“2024”是年份，横线是边框，无数学逻辑） | 数学公式的核心是**逻辑关系的可视化表达**，普通数字/线条无数学逻辑 |

**误判的根源**：仅检测“数字/线条”的表面特征，未验证上述三维本质特征——比如将表格的横线误判为分式线，将普通文本的“2+3”误判为公式，本质是忽略了“空间结构”和“逻辑关联”两个核心维度。

## 二、 领域最佳实践参考（权威方案的共识）
主流数学公式识别方案（MathPix、AWS Textract、Google Vision）均采用**“结构特征优先+语义验证兜底”**的双阶段策略，与第一性原理完全契合：
1.  **MathPix官方技术文档**：公式识别的核心是**“检测特殊结构符号（分式线/根号/上下标）+ 验证符号的空间层级关系”**，单一数字/线条无法通过验证；
2.  **AWS Textract数学内容检测最佳实践**：先通过视觉特征定位“公式候选区域”，再通过**“符号组合逻辑+OCR语义分析”**排除非公式区域（如表格、普通文本）；
3.  **OpenCV公式检测指南**：强调**“空间结构的量化分析”**（如上下标的垂直偏移量、分式线的长宽比），而非简单的轮廓检测。

## 三、 精准判定体系（三层逻辑，可工程化落地）
基于第一性原理和权威实践，构建**“候选区域筛选→核心特征量化检测→非公式排除→多模态验证”**的四层判定逻辑，复用你现有`VisualElementDetector`的轮廓检测、坐标计算功能，无需新增复杂模型。

### 1.  第一层：候选区域筛选（快速排除非公式区域）
**核心目标**：先过滤明显不是公式的区域，减少后续计算量，避免从源头误判。
#### 判定规则（基于视觉特征，快速高效）
| 筛选条件 | 保留（候选区域） | 排除（非公式） | 工具复用 |
|----------|------------------|----------------|----------|
| **区域尺寸** | 宽度 50-1000px，高度 20-500px（公式通常是紧凑的小区域） | 宽度＞1000px（大段文本）、高度＜20px（孤立线条） | `cv2.boundingRect` 计算区域尺寸 |
| **边缘密度** | 边缘密度 0.1-0.8（公式符号密集，边缘丰富） | 边缘密度＜0.1（纯色块、空白）、＞0.8（复杂图片） | 计算 Canny 边缘图的非零像素占比 |
| **颜色分布** | 灰度方差＞50（符号与背景对比明显） | 灰度方差＜50（低对比度区域，如模糊线条） | 计算区域灰度直方图的方差 |

#### 代码示例（复用现有视觉检测器）
```python
def filter_candidate_regions(self, frame: np.ndarray) -> List[np.ndarray]:
    """筛选公式候选区域，排除明显非公式区域"""
    gray = cv2.cvtColor(frame, cv2.COLOR_BGR2GRAY)
    edges = cv2.Canny(gray, 50, 150)
    # 寻找所有连通区域（候选区域）
    contours, _ = cv2.findContours(edges, cv2.RETR_EXTERNAL, cv2.CHAIN_APPROX_SIMPLE)
    candidates = []
    for cnt in contours:
        x, y, w, h = cv2.boundingRect(cnt)
        area = w * h
        if area < 1000 or area > 500000:  # 过滤过小/过大区域
            continue
        # 计算边缘密度
        region_edges = edges[y:y+h, x:x+w]
        edge_density = np.sum(region_edges > 0) / (w * h)
        if edge_density < 0.1 or edge_density > 0.8:
            continue
        # 计算灰度方差
        region_gray = gray[y:y+h, x:x+w]
        gray_var = np.var(region_gray)
        if gray_var < 50:
            continue
        candidates.append(frame[y:y+h, x:x+w])
    return candidates
```

### 2.  第二层：核心特征量化检测（判定公式的关键，基于第一性原理）
**核心目标**：验证候选区域是否具备数学公式的**三维本质特征**，量化指标避免主观判断，这是消除误判的核心步骤。

#### 特征1：特殊符号检测（专属符号体系验证）
**检测对象**：数学公式的专属结构符号（分式线、根号、上下标、矩阵括号、积分号等），这些是普通数字/线条没有的。
| 符号类型 | 量化检测规则 | 工具复用 | 判定阈值 |
|----------|--------------|----------|----------|
| **分式线** | 检测水平直线，满足：<br>1. 长度 20-200px（短而紧凑）；<br>2. 宽度 1-3px（细线）；<br>3. 上下方存在符号（数字/字母） | `cv2.HoughLinesP` 检测直线 + 坐标校验 | 满足所有条件 → 分式线候选 |
| **根号** | 检测“√”形状的轮廓，满足：<br>1. 轮廓近似三角形+竖线；<br>2. 右侧存在嵌套区域（根号内的内容） | `cv2.matchShapes` 形状匹配 + 轮廓分析 | 形状匹配得分＜0.2 → 根号候选 |
| **上下标** | 检测符号的垂直偏移，满足：<br>1. 小符号（尺寸≤主符号的1/2）；<br>2. 垂直偏移量≥10px（主符号的上方/下方） | 轮廓坐标计算（主符号与小符号的y轴差） | 偏移量≥10px → 上下标候选 |
| **矩阵括号** | 检测“()/[]/{}”的轮廓，满足：<br>1. 括号内存在多行多列符号；<br>2. 符号行列对齐（同一行的符号y坐标偏差≤5px） | 轮廓检测 + 坐标对齐校验 | 行列对齐偏差≤5px → 矩阵括号候选 |

#### 特征2：空间结构检测（层级关系验证）
**核心逻辑**：数学公式的空间结构是**非水平线性的**，通过量化“垂直层级”和“嵌套深度”区分普通数字/线条。
| 结构类型 | 量化检测规则 | 判定阈值 |
|----------|--------------|----------|
| **垂直层级** | 计算区域内符号的y轴分布方差，方差越大，垂直层级越明显（公式方差大，普通文本方差小） | y轴方差＞50 → 存在垂直层级 |
| **嵌套深度** | 计算轮廓的嵌套层数（如根号内嵌套分式，嵌套层数≥2） | 嵌套层数≥2 → 存在嵌套结构 |

#### 特征3：符号组合逻辑验证（语义关联验证）
**核心逻辑**：数学公式是**数字、字母、特殊符号的有序组合**，普通数字/线条是孤立的，通过OCR文本分析验证组合逻辑。
| 组合类型 | 验证规则 | 排除情况 |
|----------|----------|----------|
| **公式组合** | OCR识别结果包含：<br>1. 字母（a-z,A-Z）+ 数字 + 特殊符号；<br>2. 特殊符号（=、+、-）位于字母/数字之间（如a+b=c） | 仅包含数字（如“1234”）→ 排除；<br>仅包含字母（如“abcd”）→ 排除 |

#### 核心判定规则（三层特征需同时满足）
$$
\text{公式判定} = \text{特殊符号检测≥1种} \cap \text{空间结构检测通过} \cap \text{符号组合逻辑验证通过}
$$

### 3.  第三层：非公式排除规则（解决误判的关键）
针对常见的误判场景（表格数字/线条、普通文本数字、手绘线条），制定明确的排除规则，精准过滤非公式区域：

| 误判场景 | 排除规则 | 量化指标 |
|----------|----------|----------|
| **表格中的数字+横线** | 横线是**贯穿性的长线条**（长度＞区域宽度的90%），且与单元格边界关联；数字是**水平排列**（y轴方差＜20） | 横线长度/区域宽度＞0.9 → 排除；<br>y轴方差＜20 → 排除 |
| **普通文本中的数字+符号** | 无垂直层级和嵌套结构（y轴方差＜30，嵌套层数=0）；符号是**自然语言符号**（如逗号、句号） | y轴方差＜30 且 嵌套层数=0 → 排除 |
| **手绘的孤立线条/数字** | 无特殊符号（分式线/根号等）；边缘密度＜0.3（线条稀疏） | 无特殊符号 且 边缘密度＜0.3 → 排除 |
| **logo/图标中的线条** | 颜色单一（灰度方差＜30）；轮廓形状是**规则图形**（圆形、矩形） | 灰度方差＜30 且 形状规则 → 排除 |

### 4.  第四层：多模态交叉验证（兜底策略，提升准确率）
结合**视觉特征+OCR文本语义**的多模态验证，进一步消除极端误判场景，参考AWS Textract的多模态融合方案：
1.  **视觉→文本验证**：若视觉检测到分式线，但OCR识别结果是“表格边框”→ 判定为非公式；
2.  **文本→视觉验证**：若OCR识别到“a²+b²=c²”，但视觉未检测到上下标结构→ 判定为普通文本，非公式；
3.  **领域词验证**：若OCR识别结果包含数学领域词（如“积分”“矩阵”“分式”），且视觉特征通过→ 提升公式判定置信度。

### 2.  关键子方法实现示例（以特殊符号检测为例）
```python
def _detect_special_symbols(self, region: np.ndarray) -> bool:
    """检测是否存在数学公式专属符号"""
    gray = cv2.cvtColor(region, cv2.COLOR_BGR2GRAY)
    edges = cv2.Canny(gray, 50, 150)
    
    # 检测分式线（水平短线）
    lines = cv2.HoughLinesP(edges, 1, np.pi/180, threshold=20, minLineLength=20, maxLineGap=2)
    if lines is not None:
        for line in lines:
            x1, y1, x2, y2 = line[0]
            length = np.sqrt((x2-x1)**2 + (y2-y1)**2)
            width = abs(y2 - y1)
            # 分式线：水平、短、细
            if width <= 3 and length >=20 and length <= 200:
                return True
    
    # 检测上下标（小符号+垂直偏移）
    contours, _ = cv2.findContours(edges, cv2.RETR_EXTERNAL, cv2.CHAIN_APPROX_SIMPLE)
    if len(contours) >=2:
        # 计算主符号和小符号的尺寸与位置
        contour_areas = [cv2.contourArea(cnt) for cnt in contours]
        main_idx = np.argmax(contour_areas)
        main_cnt = contours[main_idx]
        x1, y1, w1, h1 = cv2.boundingRect(main_cnt)
        for i, cnt in enumerate(contours):
            if i == main_idx:
                continue
            x2, y2, w2, h2 = cv2.boundingRect(cnt)
            # 小符号：尺寸≤主符号1/2，垂直偏移≥10px
            if w2 <= w1/2 and h2 <= h1/2 and abs(y2 - y1) >=10:
                return True
    
    return False
```

# 数学公式精准判定的**决策树+加权评分体系**设计方案（工程化可落地版）
基于数学公式的**三大本质特征**（特殊符号体系、空间层级结构、逻辑语义关联），结合MathPix公式识别标准、AWS Textract视觉检测规范，设计**“分层决策树+三维加权评分”**的双轨判定方案——决策树负责**快速精准分支（排除非公式）**，评分体系负责**量化置信度（解决模糊判定）**，两者结合彻底解决“数字/线条误判为公式”的核心问题。

## 一、 核心设计原则（第一性原理+领域最佳实践）
1.  **特征优先级原则**：视觉特征（特殊符号+空间结构）＞ 语义特征（逻辑关联）——数学公式的本质是**可视化的结构符号组合**，视觉特征是客观属性，语义特征是辅助验证；
2.  **量化判定原则**：所有决策节点均基于**可计算的量化指标**，避免“看起来像公式”的主观判断；
3.  **排除优先原则**：先通过硬规则排除明确的非公式场景（如表格、普通文本），再验证公式特征，大幅降低误判率；
4.  **置信度分层原则**：通过评分体系输出0-100的置信度，支持“高置信直接判定+低置信人工复核”的分级策略。

## 二、 第一部分：三维加权评分体系（量化判定依据）
通过计算 **「符号特征得分（S₁）、空间结构得分（S₂）、语义关联得分（S₃）」** 三个维度的加权总分，量化候选区域是数学公式的置信度。

### 1.  评分维度与权重分配（权威依据）
| 评分维度 | 权重占比 | 核心量化指标 | 第一性原理依据 | 领域最佳实践参考 |
|----------|----------|--------------|----------------|------------------|
| **符号特征得分（S₁）** | 40%（核心） | 数学专属符号的**命中类型数**+**符号完整性** | 公式的基础是“领域专属符号”，无特殊符号则不可能是公式 | MathPix：优先检测分式线、根号、上下标等强符号 |
| **空间结构得分（S₂）** | 40%（核心） | 垂直层级（上下标偏移量）、嵌套深度（根号/括号层数）、行列对齐度 | 公式的语义依赖**非水平线性的空间关系**，普通文本无此特征 | AWS Textract：空间结构是区分公式与普通文本的关键 |
| **语义关联得分（S₃）** | 20%（辅助） | OCR文本的**数学逻辑组合度**、领域词匹配度（如“积分”“矩阵”） | 公式是“符号的有序逻辑组合”，普通数字/线条无数学逻辑 | 斯坦福NLP：语义验证兜底，排除“数字+符号”的无意义组合 |

### 2.  各维度得分计算公式（量化落地）
所有指标均归一化到 **0-100分**，确保总分可直接映射置信度。

#### （1） 符号特征得分（S₁：0-100分）
**核心逻辑**：统计候选区域内命中的**数学专属符号类型数**，类型越多、符号越完整，得分越高。
$$
S_1 = 20 × N_{symbol} + 30 × C_{symbol} + 50 × \max(0, 1 - \frac{M_{missing}}{T_{total}})
$$
| 参数 | 含义 | 计算方式 | 阈值/说明 |
|------|------|----------|-----------|
| $N_{symbol}$ | 命中的符号类型数 | 统计是否包含以下类型（最多5种）：<br>1. 结构符号（分式线/根号/矩阵括号）<br>2. 关系符号（=、≠、＞、∑、∫）<br>3. 修饰符号（上下标、省略号）<br>4. 运算符号（×、÷、±）<br>5. 括号符号（()、[]、{}） | 每种类型得20分，满分100→ 乘以20 |
| $C_{symbol}$ | 符号完整性 | 符号是否完整（如根号是否有闭合、分式线是否有上下内容） | 完整=1，残缺=0 → 占30分 |
| $M_{missing}$ | 缺失符号数 | 关键符号的缺失数量（如分式线下方无内容） | $T_{total}$为总符号数 → 占50分 |

**示例**：候选区域含“根号+上下标+等号”→ $N_{symbol}=3$ → 基础分60；符号完整→ $C_{symbol}=1$ → +30；无缺失→ +50 → 总分=60+30+50=140？→ 归一化到100分：$S_1 = \min(100, 计算值)$。

#### （2） 空间结构得分（S₂：0-100分）
**核心逻辑**：量化公式的**非水平线性空间特征**，普通文本/线条的空间结构得分为0。
$$
S_2 = 30 × V_{offset} + 40 × D_{nest} + 30 × A_{align}
$$
| 参数 | 含义 | 计算方式（工程化可实现） | 阈值/说明 |
|------|------|--------------------------|-----------|
| $V_{offset}$ | 垂直层级得分 | 检测上下标/分式的垂直偏移量：<br>偏移量≥10px（主符号高度的1/3）→ 1分；<br>否则→ 0分 | 占30分，普通文本的垂直偏移量≈0 |
| $D_{nest}$ | 嵌套深度得分 | 统计根号/括号的嵌套层数：<br>层数≥1→ 层数×20分；<br>层数=0→ 0分 | 最多2层（满分40分），普通文本无嵌套 |
| $A_{align}$ | 行列对齐得分 | 矩阵/方程组的行列对齐度：<br>同一行符号的y坐标偏差≤5px → 1分；<br>否则→ 0分 | 占30分，无矩阵结构则得0分 |

**示例**：候选区域是分式（分子分母垂直偏移15px）+ 根号嵌套（1层）→ $V_{offset}=1$（30分）、$D_{nest}=1$（20分）、无矩阵→ $A_{align}=0$ → 总分=30+20+0=50分。

#### （3） 语义关联得分（S₃：0-100分）
**核心逻辑**：验证OCR文本的**数学逻辑组合**，排除“数字+符号”的无意义组合。
$$
S_3 = 40 × L_{combination} + 30 × K_{domain} + 30 × R_{ratio}
$$
| 参数 | 含义 | 计算方式 | 阈值/说明 |
|------|------|----------|-----------|
| $L_{combination}$ | 逻辑组合得分 | OCR结果是否符合数学语法：<br>如“字母+符号+数字”（a+b=1）→ 1分；<br>仅数字/仅字母→ 0分 | 占40分 |
| $K_{domain}$ | 领域词匹配得分 | OCR结果是否包含数学领域词（积分、矩阵、分式） | 命中1个→ 1分，否则→ 0分 → 占30分 |
| $R_{ratio}$ | 符号数字比得分 | 符号数:数字数的比例在 **1:2 ~ 2:1** 之间（公式的典型比例）→ 1分；否则→ 0分 | 占30分，普通文本的比例极端（如10:1） |

**示例**：OCR结果是“$a^2+b^2=c^2$”→ 逻辑组合=1（40分）、无领域词=0（0分）、符号数3:数字数3=1:1→ 1（30分）→ 总分=70分。

### 3.  综合置信度得分（Final_Score：0-100分）
加权计算三个维度得分，权重严格遵循“视觉特征优先”原则：
$$
Final\_Score = 0.4 × S_1 + 0.4 × S_2 + 0.2 × S_3
$$

#### 置信度分级策略（工程化落地）
| 置信度区间 | 判定结果 | 处理策略 |
|------------|----------|----------|
| 80-100分 | 高置信公式 | 直接判定为数学公式，无需复核 |
| 40-79分 | 中置信模糊区域 | 结合多模态验证（如上下文语义）再判定 |
| 0-39分 | 低置信非公式 | 直接判定为非公式，排除误判 |

## 三、 第二部分：分层决策树（精准分支判定）
决策树以 **“排除优先+核心特征验证”** 为核心逻辑，分**5层节点**，从“快速过滤非公式”到“精准验证公式特征”，逐步缩小判定范围，确保每一步决策都基于量化指标。

### 决策树完整结构（文字版）
```
根节点：输入候选区域（来自视觉检测器的连通域筛选）
├─ 第1层：基础过滤（硬规则，快速排除非公式）
│  ├─ 节点1.1：区域尺寸校验
│  │  ├─ 条件：宽度50-1000px 且 高度20-500px → 通过
│  │  └─ 不通过 → 判定为【非公式】（过大/过小区域不可能是公式）
│  ├─ 节点1.2：边缘密度校验
│  │  ├─ 条件：边缘密度0.1-0.8 → 通过
│  │  └─ 不通过 → 判定为【非公式】（边缘过疏/过密）
│  └─ 节点1.3：灰度方差校验
│     ├─ 条件：灰度方差＞50 → 通过（符号与背景对比明显）
│     └─ 不通过 → 判定为【非公式】（低对比度，如模糊线条）
├─ 第2层：特殊符号检测（核心过滤，无符号=非公式）
│  ├─ 节点2.1：强符号命中校验
│  │  ├─ 条件：命中至少1种强符号（分式线/根号/上下标/∑/∫）→ 通过
│  │  └─ 不通过 → 进入节点2.2（弱符号校验）
│  └─ 节点2.2：弱符号命中校验
│     ├─ 条件：命中至少2种弱符号（=、+、×、括号）且 符号数字比符合1:2~2:1 → 通过
│     └─ 不通过 → 判定为【非公式】（无数学专属符号）
├─ 第3层：空间结构验证（区分公式与普通文本/线条）
│  ├─ 节点3.1：垂直层级校验
│  │  ├─ 条件：存在上下标/分式的垂直偏移（偏移量≥10px）→ 通过
│  │  └─ 不通过 → 进入节点3.2（嵌套结构校验）
│  └─ 节点3.2：嵌套结构校验
│     ├─ 条件：存在根号/括号嵌套（层数≥1）→ 通过
│     └─ 不通过 → 判定为【非公式】（无公式空间结构）
├─ 第4层：误判场景排除（解决核心误判问题）
│  ├─ 节点4.1：表格排除校验
│  │  ├─ 条件：线条是贯穿性横线（长度＞区域宽度90%）→ 判定为【表格边框】
│  │  └─ 不通过 → 进入节点4.2
│  ├─ 节点4.2：普通文本排除校验
│  │  ├─ 条件：符号仅水平线性排列 且 无垂直层级 → 判定为【普通文本】
│  │  └─ 不通过 → 进入节点4.3
│  └─ 节点4.3：手绘线条排除校验
│     ├─ 条件：边缘密度＜0.3 且 无符号组合 → 判定为【手绘线条】
│     └─ 不通过 → 通过排除
├─ 第5层：置信度评分与最终判定
│  ├─ 节点5.1：计算Final_Score
│  │  ├─ 条件：Final_Score ≥80 → 判定为【高置信数学公式】
│  │  ├─ 条件：40 ≤ Final_Score ＜80 → 判定为【中置信模糊区域】
│  │  └─ 条件：Final_Score ＜40 → 判定为【非公式】
└─ 叶子节点：输出判定结果+置信度得分
```

### 各层节点的工程化判定逻辑（关键代码参考）
#### 1.  第1层：基础过滤（复用现有视觉检测器）
```python
def basic_filter(region: np.ndarray) -> bool:
    h, w = region.shape[:2]
    # 尺寸校验
    if not (50 <= w <= 1000 and 20 <= h <= 500):
        return False
    # 边缘密度校验
    gray = cv2.cvtColor(region, cv2.COLOR_BGR2GRAY)
    edges = cv2.Canny(gray, 50, 150)
    edge_density = np.sum(edges > 0) / (w * h)
    if not (0.1 <= edge_density <= 0.8):
        return False
    # 灰度方差校验
    if np.var(gray) < 50:
        return False
    return True
```

def detect_special_symbols(region: np.ndarray) -> tuple:
    gray = cv2.cvtColor(region, cv2.COLOR_BGR2GRAY)
    edges = cv2.Canny(gray, 50, 150)
    contours, _ = cv2.findContours(edges, cv2.RETR_EXTERNAL, cv2.CHAIN_APPROX_SIMPLE)
    
    # 检测强符号：分式线（水平短线）
    lines = cv2.HoughLinesP(edges, 1, np.pi/180, 20, minLineLength=20, maxLineGap=2)
    has_strong_symbol = False
    if lines is not None:
        for line in lines:
            x1, y1, x2, y2 = line[0]
            length = np.sqrt((x2-x1)**2 + (y2-y1)**2)
            width = abs(y2 - y1)
            if width <=3 and 20 <= length <=200:  # 分式线特征
                has_strong_symbol = True
                break
    
    # 检测上下标（强符号）
    if not has_strong_symbol and len(contours)>=2:
        areas = [cv2.contourArea(c) for c in contours]
        main_idx = np.argmax(areas)
        main_h = cv2.boundingRect(contours[main_idx])[3]
        for i, c in enumerate(contours):
            if i == main_idx: continue
            h = cv2.boundingRect(c)[3]
            y_offset = abs(cv2.boundingRect(c)[1] - cv2.boundingRect(contours[main_idx])[1])
            if h <= main_h/2 and y_offset >=10:  # 上下标特征
                has_strong_symbol = True
                break
    
    # 弱符号检测（=、+、括号）
    has_weak_symbol = False

    return has_strong_symbol, has_weak_symbol

#### 3.  第4层：误判场景排除（解决表格/文本误判）
```python
def exclude_non_formula(region: np.ndarray, ocr_result: str) -> str:
    gray = cv2.cvtColor(region, cv2.COLOR_BGR2GRAY)
    edges = cv2.Canny(gray, 50, 150)
    h, w = region.shape[:2]
    
    # 表格排除：贯穿性横线
    lines = cv2.HoughLinesP(edges, 1, np.pi/180, 20, minLineLength=int(w*0.9), maxLineGap=2)
    if lines is not None:
        return "table_border"
    
    # 普通文本排除：仅水平排列，无垂直偏移
    # 通过轮廓的y坐标分布判断
    contours, _ = cv2.findContours(edges, cv2.RETR_EXTERNAL, cv2.CHAIN_APPROX_SIMPLE)
    y_coords = [cv2.boundingRect(c)[1] for c in contours]
    y_variance = np.var(y_coords)
    if y_variance < 20:  # 方差小→ 水平排列
        return "normal_text"
    
    # 手绘线条排除：边缘密度低且无符号
    edge_density = np.sum(edges > 0) / (w * h)
    if edge_density < 0.3 and len(contours) < 2:
        return "handdrawn_line"
    
    return "pass"  # 通过排除
```

## 四、 决策树+评分体系的融合执行流程（工程化落地）
### 完整执行步骤
1.  **候选区域生成**：对输入帧进行轮廓检测，提取所有连通域作为候选区域；
2.  **决策树第一层过滤**：基础过滤（尺寸+边缘密度+灰度方差），排除90%的非公式区域；
3.  **决策树第二、三层验证**：检测特殊符号+空间结构，筛选出公式疑似区域；
4.  **决策树第四层排除**：排除表格、普通文本、手绘线条等误判场景；
5.  **评分体系量化打分**：对通过决策树的疑似区域，计算Final_Score置信度；
6.  **最终判定**：根据置信度分级输出结果（高置信直接判定，中置信复核，低置信排除）。

### 核心代码骨架（整合到现有`VisualElementDetector`）
```python
class MathFormulaDetector:
    def __init__(self, visual_extractor, ocr_engine):
        self.visual_extractor = visual_extractor
        self.ocr_engine = ocr_engine
    
    def detect_formula(self, frame: np.ndarray) -> list:
        """
        返回格式：[(is_formula, confidence, bounding_box), ...]
        """
        results = []
        # 步骤1：生成候选区域
        candidates = self._get_candidate_regions(frame)
        
        for region, bbox in candidates:
            # 步骤2：决策树第一层过滤
            if not self.basic_filter(region):
                results.append((False, 0.0, bbox))
                continue
            
            # 步骤3：决策树第二、三层验证
            has_strong, has_weak = self.detect_special_symbols(region)
            has_spatial = self.detect_spatial_structure(region)
            if not ((has_strong or has_weak) and has_spatial):
                results.append((False, 0.0, bbox))
                continue
            
            # 步骤4：决策树第四层排除
            ocr_result = self.ocr_engine.recognize(region)
            exclude_type = self.exclude_non_formula(region, ocr_result)
            if exclude_type != "pass":
                results.append((False, 0.0, bbox))
                continue
            
            # 步骤5：评分体系计算置信度
            s1 = self.calculate_symbol_score(region, ocr_result)
            s2 = self.calculate_space_score(region)
            s3 = self.calculate_semantic_score(ocr_result)
            final_score = 0.4*s1 + 0.4*s2 + 0.2*s3
            
            # 步骤6：最终判定
            is_formula = final_score >= 40  # 中置信以上视为公式
            results.append((is_formula, final_score, bbox))
        
        return results
```
