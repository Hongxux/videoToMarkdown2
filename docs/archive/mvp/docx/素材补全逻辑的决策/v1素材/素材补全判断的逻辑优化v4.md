# 补全素材决策树的第一性原理分析与“图被降级为TEXT”的优化方案
## 一、补全素材选择的核心第一性原理（分析基准）
补全素材决策的终极目标是**以“最小认知成本”满足用户对断层信息的补全需求**，拆解为两条不可违背的底层公理，所有决策逻辑的合理性都应以此为标尺：
1. **认知匹配公理**：素材类型必须匹配信息的“固有呈现形态”——
   - 抽象逻辑/指代缺失（无视觉载体）→ 仅TEXT能满足，认知成本最低；
   - 静态空间结构/视觉关系（如图表、架构图、层级关系）→ SCREENSHOT是最优解（TEXT描述会增加空间认知成本）；
   - 动态过程/操作流转→ VIDEO是最优解；
2. **表现力优先公理**：在匹配“固有呈现形态”的前提下，选择“实际表现力得分最高”的素材；若素材类型与呈现形态不匹配，即使得分高也应降级（如TEXT无法替代图的空间认知价值）。

## 二、现有决策树的合理性分析（基于第一性原理）
整体框架符合“认知需求驱动+表现力择优”的核心逻辑，但局部设计违背第一性原理，是“图被降级为TEXT”的核心原因：

### 1. 合理的部分（契合第一性原理）
- **第一层（第一性原理直连）**：完全契合“认知匹配公理”——数学公式的固有形态是“静态结构/动态推导”，直接绑定SCREENSHOT/VIDEO/混合态，跳过后续打分避免干扰，高置信度（0.95）匹配高价值场景的优先级，逻辑无问题；
- **第二层（认知需求分类）**：方向正确——通过文本分类匹配“认知需求”，本质是识别信息的“固有呈现形态”，契合认知匹配公理；
- **第三层（素材响应验证）**：契合“表现力优先公理”——对目标素材的实际表现力打分，避免“需求匹配但素材质量差”的问题；

### 2. 不合理的部分（违背第一性原理，导致图被降级为TEXT）
| 决策环节 | 违背第一性原理的具体表现 | 直接后果 |
|----------|--------------------------|----------|
| 第一层覆盖范围过窄 | 仅绑定“数学公式”，未覆盖“架构图/流程图/空间关系图”等其他静态结构场景 | 这些场景只能走第二层，而第二层易误判为TEXT |
| 第二层分类仅依赖文本 | 零样本分类仅输入“断层文本+上下文”，完全忽略画面中“是否存在图”的视觉事实 | 即使画面有图（固有形态是SCREENSHOT），若文本描述模糊（如“这个关系不对”），也会被误分为TEXT（抽象逻辑） |
| 第三层得分权重设计失衡 | 1. SCREENSHOT的“视觉结构完整性”权重（0.5）未体现图的核心价值；<br>2. TEXT保底分过高（0.6/0.85），即使截图更匹配，得分也易低于TEXT | 即使分类正确为SCREENSHOT，也因得分低被降级为TEXT |
| 第四层决策优先级模糊 | 未明确“认知匹配（素材类型）优先于得分”，仅按得分高低决策 | 即使TEXT不符合固有形态，也因得分高成为最终选择 |

## 三、“本该用图却降级为TEXT”的根源拆解（第一性原理视角）
核心是**“视觉事实（画面有图）”让位于“文本分类结果+得分博弈”**，违背了“认知匹配公理”——图的固有呈现形态是SCREENSHOT，无论文本描述是否模糊、得分是否略低，都不应被降级为TEXT（TEXT无法替代图的空间认知价值）。

举个典型场景：
> 断层文本：“这个层级关系不对”，画面中存在清晰的架构层级图。
- 第二层：BERT零样本分类误将“层级关系”判为TEXT（抽象逻辑），而非SCREENSHOT（静态空间关系）；
- 第三层：TEXT保底分0.85，截图得分因“语义相似”略低（0.7），最终得分TEXT更高；
- 第四层：直接选择TEXT，完全忽略画面有图的视觉事实，用户用TEXT补全后仍无法理解层级关系（认知成本极高）。

## 四、基于第一性原理的优化方案（修正偏差，保留原有框架）
核心优化逻辑：**强化“视觉事实”对静态结构场景的优先级，让“认知匹配”优先于“文本分类+得分博弈”**，具体分四层优化：

### 1. 第一层扩展：新增“通用静态结构直连”（最高优先级）
将第一层从“仅数学公式”扩展到“所有静态视觉结构”，直接绑定SCREENSHOT，避免进入第二层误判：
```python
def decide_enhancement_type_refined(self, fault_text, frame, context):

```
**第一性原理依据**：画面中有图（静态空间结构），其固有呈现形态是SCREENSHOT，直接绑定最高优先级，避免文本分类的干扰。

### 2. 第二层优化：文本分类+视觉特征交叉验证
不再仅依赖文本分类，而是结合“视觉特征”修正分类结果，避免误判：
```python
def classify_cognitive_demand(self, fault_text, context, frame):
    # 2.1 原有文本零样本分类（保留）
    text_based_label = self.bert_zero_shot_classifier(fault_text, context)
    # 2.2 提取视觉特征（是否有静态结构/动态动作）
    visual_features = self.extract_visual_features(frame)
    # 2.3 交叉验证：修正分类结果
    corrected_label = text_based_label
    if text_based_label == "TEXT":
        # 若文本判为TEXT，但画面有静态结构→修正为SCREENSHOT
        if visual_features.has_static_visual_structure:
            corrected_label = "SCREENSHOT"
        # 若文本判为TEXT，但画面有动态动作→修正为VIDEO
        elif visual_features.has_dynamic_action:
            corrected_label = "VIDEO"
    return corrected_label
```
**第一性原理依据**：认知需求的判定应结合“文本需求+视觉事实”，而非仅文本，确保分类结果匹配信息的固有呈现形态。

### 3. 第三层优化：重构得分权重，降低TEXT保底分
修正得分权重，突出SCREENSHOT的核心价值（视觉结构完整性），并降低TEXT保底分，避免得分博弈导致的降级：
```python
def calculate_match_scores(self, target_label, fault_text, frame, audio):
    # 3.1 视频得分（保留原有逻辑，权重微调）
    score_video = 0.3 * semantic_similarity + 0.5 * visual_action_strength + 0.2 * speech_rate
    # 3.2 截图得分：提升视觉结构完整性权重（0.5→0.7），降低语义相似权重（0.3→0.1）
    score_screenshot = 0.1 * semantic_similarity + 0.7 * visual_structure_completeness + 0.2 * context_environment
    # 3.3 文字得分：降低保底分（0.6/0.85→0.5/0.7），且仅当target_label=TEXT时生效
    if target_label == "TEXT":
        score_text = 0.7  # 仅匹配时保底分高
    else:
        score_text = 0.5  # 不匹配时保底分降低
    return score_video, score_screenshot, score_text
```
**第一性原理依据**：SCREENSHOT的核心价值是“视觉结构完整性”，而非语义相似；TEXT仅在匹配抽象逻辑需求时才应获得高保底分，否则不应具备竞争优势。

### 4. 第四层优化：认知匹配优先的决策逻辑
明确“素材类型匹配目标标签”优先于得分，仅在类型匹配的前提下比较得分：
```python
def final_decision(self, target_label, score_video, score_screenshot, score_text):
    # 4.1 先筛选出与目标标签匹配的素材（认知匹配优先）
    candidate_scores = {}
    if target_label == "VIDEO":
        candidate_scores["VIDEO"] = score_video
    elif target_label == "SCREENSHOT":
        candidate_scores["SCREENSHOT"] = score_screenshot
    elif target_label == "TEXT":
        candidate_scores["TEXT"] = score_text
    # 4.2 若匹配素材得分≥0.5，直接选择（避免低分但匹配的素材被淘汰）
    if max(candidate_scores.values()) >= 0.5:
        final_choice = max(candidate_scores, key=candidate_scores.get)
    # 4.3 若匹配素材得分过低，才考虑其他类型（兜底）
    else:
        all_scores = {"VIDEO": score_video, "SCREENSHOT": score_screenshot, "TEXT": score_text}
        final_choice = max(all_scores, key=all_scores.get)
    return final_choice
```
**第一性原理依据**：即使匹配的素材得分略低，也比不匹配的素材认知成本更低（比如SCREENSHOT得分0.6，TEXT得分0.7，仍选SCREENSHOT），仅当匹配素材得分极低（<0.5，如画面模糊）时才兜底选择其他类型。

## 五、核心优化总结（第一性原理回归）
现有决策树的核心问题是“认知匹配让位于文本分类和得分博弈”，优化后重新锚定两条核心公理：
1. **视觉事实优先**：画面中有图（静态结构）时，直接绑定SCREENSHOT，避免文本误判；
2. **认知匹配优先**：先确保素材类型匹配信息的固有呈现形态，再比较得分，而非仅按得分高低决策；
3. **权重适配价值**：截图的得分权重向“视觉结构完整性”倾斜，文字保底分仅在匹配抽象逻辑时生效。

这套优化方案保留了原有“四层决策”的框架，仅通过“扩展第一层、交叉验证第二层、重构第三层权重、明确第四层优先级”，就能彻底解决“本该用图却降级为TEXT”的问题，同时符合“最小认知成本补全信息”的第一性原理。
