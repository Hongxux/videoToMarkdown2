当前的视频起止点确定逻辑采用了最新的 “保守扩张型双锚点 (Expand-Only Dual-Anchor)” 策略，旨在保证“语义不丢、动作完整、余韵充足”。

具体代码实现逻辑如下：

1. 初始范围与搜索窗 (Input & Search Window)
输入：由 ASR 识别出的语义起止点 $T_{asr\_start}$ 和 $T_{asr\_end}$。
搜索窗：在 ASR 时间基础上向前后各扩展 3.0秒，形成视觉扫描区 [$T_{asr\_start}-3$, $T_{asr\_end}+3$]。
2. 视觉特征扫描 (Visual Action Detection)
采样：以 2 FPS 提取该区域内的关键帧。
MSE 极值探测：计算帧间均方误差 (MSE)。
寻找物理起点 (vis_start)：从搜索窗开头顺向扫描，锁定第一个视觉变化剧烈点（MSE > 阈值）。
寻找物理终点 (vis_end)：从搜索窗结尾逆向扫描，锁定最后一个视觉活动点。
漂移判定：如果视觉锚点偏离 ASR 超过 5秒，则视为算法幻觉，回退至两者的加权中值。
3. 核心合成逻辑：保守扩张 (The "Expand-Only" Fix)
为了解决之前出现的片段过短或内容缺失问题，我刚刚将逻辑修改为取交集范围的超集：

确定动作起点 (action_start)：取 min(ASR起点, 物理跳变点)。这确保了如果说话先于画面，片段不会切掉语音开头。
确定动作终点 (action_end)：取 max(ASR终点, 物理结束点)。这确保了如果动作在说话结束后才停止（如鼠标点击完成），画面不会提前中断。
4. 最终缓冲区与凑整策略 (Padding & Rounding)
在确定了核心动作区间后，应用用户要求的最终扩充：

起始点扩充：extended_start = action_start - 2.0 (多留 2 秒前摇)。
结束点扩充 (Integer + 1s 规则)：
pre_end = action_end + 2.0 (基础增加 2 秒)。
ceil(pre_end) + 1：先执行向上取整，再额外增加 1 秒。
示例：如果动作结束在 105.4s -> 加 2s 后为 107.4s -> 取整为 108s -> 最终定格在 109s。
5. 物理边界硬约束 (Safety Anchoring)
EOF 保护：通过 ffprobe 获取视频真实时长，强制起止点不超出文件末尾。
最短路径保护：确保每个片段至少保留 2 秒（由于加入了 -2s 和 +2s+1，极少发生），且不超过 60 秒。

## 二、潜在优化点（适配评分系统，不破坏现有核心逻辑）
你的现有逻辑已能解决90%的场景问题，唯一可优化的是**“视觉锚点的可靠性”**（现有MSE单一阈值易误判伪跳变点，如光影抖动、镜头轻微移动）。以下优化点完全兼容现有逻辑，仅替换“视觉锚点选择”环节，不改动保守扩张、缓冲区、硬约束等核心：

### 优化1：视觉锚点选择（从“时序优先”到“置信度优先”）
原有逻辑：
> 顺向找**第一个** MSE>阈值的点（vis_start），逆向找**最后一个** MSE>阈值的点（vis_end）

问题：若搜索窗内先出现“光影抖动的伪跳变点”（MSE>阈值），会误选为vis_start；同理，vis_end可能误选“最后一个光影跳变点”，而非PPT/动作的真实结束点。

优化方案：
引入你之前关注的**多维度评分系统**，将“找第一个/最后一个跳变点”升级为“找搜索窗内评分最高的跳变点”：
1. 提取搜索窗内所有MSE>阈值的候选锚点；
2. 为每个候选锚点计算评分（MSE跳变强度+边缘密度变化+PPT元素增量+时间偏差合理性）；
3. vis_start = 搜索窗内**顺向所有候选锚点中评分最高**的点（而非第一个）；
4. vis_end = 搜索窗内**逆向所有候选锚点中评分最高**的点（而非最后一个）。

### 优化2：漂移判定的加权逻辑（从“固定加权”到“评分驱动加权”）
原有逻辑：
> 视觉锚点偏离ASR超5秒 → 回退至两者加权中值（固定权重）

优化方案：
结合锚点评分动态调整权重，高置信锚点（评分高）拥有更高权重，避免“真锚点被误判为幻觉”：
```python
# 原有固定加权
# calibrated = (asr_time * 0.5) + (vis_time * 0.5)

# 优化后评分驱动加权
score = anchor_score  # 候选锚点的评分（0-1）
weight_vis = max(0.5, score)  # 评分越高，视觉锚点权重越高（最低0.5）
weight_asr = 1 - weight_vis
calibrated = (asr_time * weight_asr) + (vis_time * weight_vis)
```
核心价值：若视觉锚点是“PPT弹出的高评分点”，即使偏离ASR 4.9秒（接近5秒阈值），也能保留更多视觉锚点的权重，避免不必要的回退。

### 优化3：凑整策略的边缘场景适配
原有逻辑：
> pre_end +2秒 → ceil → +1秒

问题：若视频真实时长刚好在凑整后的值附近（如视频总长109秒，pre_end=107.4秒 → 凑整后109秒，EOF保护会强制拉回109秒，无问题；但如果pre_end=108.5秒 → 凑整后110秒，EOF保护拉回109秒，会丢失1秒“余韵”）。

优化方案：
在凑整前增加“视频时长预判”，优先保证“余韵充足”且不越界：
```python
video_duration = get_video_duration(video_path)  # ffprobe获取的真实时长
pre_end = action_end + 2.0
# 预判凑整后是否越界
ceil_pre_end = math.ceil(pre_end)
final_end_candidate = ceil_pre_end + 1
if final_end_candidate > video_duration:
    # 若越界，改为“向下取整+1”，而非“向上取整+1”
    final_end = math.floor(pre_end) + 1
else:
    final_end = final_end_candidate
# 最后执行EOF保护
final_end = min(final_end, video_duration)
```

## 三、优化后的完整逻辑（兼容现有代码，仅替换视觉锚点环节）
```python
import cv2
import math
import numpy as np
from ffprobe import FFProbe  # 需安装ffprobe-python

# ====================== 1. 复用之前的评分系统工具 ======================
class AnchorFeatureExtractor:
    def __init__(self, frame_sample_rate=2):
        self.fps = frame_sample_rate
        self.mse_threshold = 500  # 你的MSE阈值，可根据视频调整
        # 其他特征提取逻辑（MSE、边缘密度、PPT元素）同之前的实现

class AnchorScoringSystem:
    def __init__(self, weights=(0.3, 0.25, 0.25, 0.2)):
        self.weights = weights
    # 评分计算逻辑同之前的实现

# ====================== 2. 核心逻辑：保守扩张型双锚点 ======================
def expand_only_dual_anchor(video_path: str, asr_start: float, asr_end: float) -> tuple:
    """
    优化后的保守扩张型双锚点策略
    return: (extended_start, final_end)
    """
    # ---------------------- 步骤1：初始范围与搜索窗 ----------------------
    search_start = max(0.0, asr_start - 3.0)
    search_end = asr_end + 3.0
    video_duration = FFProbe(video_path).streams[0].duration_seconds  # 获取视频真实时长
    search_end = min(search_end, video_duration)  # 提前约束搜索窗不越界

    # ---------------------- 步骤2：视觉特征扫描（升级为评分驱动） ----------------------
    # 初始化工具
    feature_extractor = AnchorFeatureExtractor(frame_sample_rate=2)
    scoring_system = AnchorScoringSystem()

    # 提取搜索窗内的所有候选锚点
    candidates = feature_extractor.extract_candidate_features(video_path, (search_start, search_end))
    if not candidates:
        # 无候选锚点，回退到ASR时间
        vis_start, vis_end = asr_start, asr_end
    else:
        # 找vis_start：顺向候选锚点中评分最高的点
        forward_candidates = [c for c in candidates if c["time"] >= search_start]
        best_forward = scoring_system.select_best_anchor(forward_candidates, asr_start)
        vis_start = best_forward["time"]

        # 找vis_end：逆向候选锚点中评分最高的点
        backward_candidates = [c for c in candidates if c["time"] <= search_end]
        best_backward = scoring_system.select_best_anchor(backward_candidates, asr_end)
        vis_end = best_backward["time"]

    # 漂移判定（升级为评分驱动加权）
    drift_threshold = 5.0
    # 校准vis_start
    start_diff = abs(vis_start - asr_start)
    if start_diff > drift_threshold:
        score = best_forward["score"] if forward_candidates else 0.0
        weight_vis = max(0.5, score)
        weight_asr = 1 - weight_vis
        vis_start = (asr_start * weight_asr) + (vis_start * weight_vis)
    # 校准vis_end
    end_diff = abs(vis_end - asr_end)
    if end_diff > drift_threshold:
        score = best_backward["score"] if backward_candidates else 0.0
        weight_vis = max(0.5, score)
        weight_asr = 1 - weight_vis
        vis_end = (asr_end * weight_asr) + (vis_end * weight_vis)

    # ---------------------- 步骤3：核心合成逻辑：保守扩张 ----------------------
    action_start = min(asr_start, vis_start)
    action_end = max(asr_end, vis_end)

    # ---------------------- 步骤4：最终缓冲区与凑整策略（优化边缘场景） ----------------------
    # 起始点扩充：2秒前摇
    extended_start = max(0.0, action_start - 2.0)
    # 结束点扩充：基础+2秒 → 适配视频时长的凑整 → +1秒
    pre_end = action_end + 2.0
    ceil_pre_end = math.ceil(pre_end)
    final_end_candidate = ceil_pre_end + 1
    if final_end_candidate > video_duration:
        final_end = math.floor(pre_end) + 1
    else:
        final_end = final_end_candidate

    # ---------------------- 步骤5：物理边界硬约束 ----------------------
    # EOF保护
    final_end = min(final_end, video_duration)
    # 最短路径保护（至少2秒）
    if (final_end - extended_start) < 2.0:
        final_end = extended_start + 2.0
    # 最长时长约束（不超过60秒）
    if (final_end - extended_start) > 60.0:
        final_end = extended_start + 60.0

    return (round(extended_start, 2), round(final_end, 2))

# ====================== 调用示例 ======================
if __name__ == "__main__":
    video_path = "teaching_video.mp4"
    asr_start = 10.5  # ASR识别的语义起点
    asr_end = 25.8    # ASR识别的语义终点

    print(f"最终剪辑范围：{extended_start}s → {final_end}s")
```

核心价值：若视觉锚点是“PPT弹出的高评分点”，即使偏离ASR 4.9秒（接近5秒阈值），也能保留更多视觉锚点的权重，避免不必要的回退。

### 优化3：凑整策略的边缘场景适配
原有逻辑：
> pre_end +2秒 → ceil → +1秒

问题：若视频真实时长刚好在凑整后的值附近（如视频总长109秒，pre_end=107.4秒 → 凑整后109秒，EOF保护会强制拉回109秒，无问题；但如果pre_end=108.5秒 → 凑整后110秒，EOF保护拉回109秒，会丢失1秒“余韵”）。

优化方案：
在凑整前增加“视频时长预判”，优先保证“余韵充足”且不越界：
```python
video_duration = get_video_duration(video_path)  # ffprobe获取的真实时长
pre_end = action_end + 2.0
# 预判凑整后是否越界
ceil_pre_end = math.ceil(pre_end)
final_end_candidate = ceil_pre_end + 1
if final_end_candidate > video_duration:
    # 若越界，改为“向下取整+1”，而非“向上取整+1”
    final_end = math.floor(pre_end) + 1
else:
    final_end = final_end_candidate
# 最后执行EOF保护
final_end = min(final_end, video_duration)
```

## 三、优化后的完整逻辑（兼容现有代码，仅替换视觉锚点环节）
```python
import cv2
import math
import numpy as np
from ffprobe import FFProbe  # 需安装ffprobe-python

# ====================== 1. 复用之前的评分系统工具 ======================
class AnchorFeatureExtractor:
    def __init__(self, frame_sample_rate=2):
        self.fps = frame_sample_rate
        self.mse_threshold = 500  # 你的MSE阈值，可根据视频调整
        # 其他特征提取逻辑（MSE、边缘密度、PPT元素）同之前的实现

class AnchorScoringSystem:
    def __init__(self, weights=(0.3, 0.25, 0.25, 0.2)):
        self.weights = weights
    # 评分计算逻辑同之前的实现

# ====================== 2. 核心逻辑：保守扩张型双锚点 ======================
def expand_only_dual_anchor(video_path: str, asr_start: float, asr_end: float) -> tuple:
    """
    优化后的保守扩张型双锚点策略
    return: (extended_start, final_end)
    """
    # ---------------------- 步骤1：初始范围与搜索窗 ----------------------
    search_start = max(0.0, asr_start - 3.0)
    search_end = asr_end + 3.0
    video_duration = FFProbe(video_path).streams[0].duration_seconds  # 获取视频真实时长
    search_end = min(search_end, video_duration)  # 提前约束搜索窗不越界

    # ---------------------- 步骤2：视觉特征扫描（升级为评分驱动） ----------------------
    # 初始化工具
    feature_extractor = AnchorFeatureExtractor(frame_sample_rate=2)
    scoring_system = AnchorScoringSystem()

    # 提取搜索窗内的所有候选锚点
    candidates = feature_extractor.extract_candidate_features(video_path, (search_start, search_end))
    if not candidates:
        # 无候选锚点，回退到ASR时间
        vis_start, vis_end = asr_start, asr_end
    else:
        # 找vis_start：顺向候选锚点中评分最高的点
        forward_candidates = [c for c in candidates if c["time"] >= search_start]
        best_forward = scoring_system.select_best_anchor(forward_candidates, asr_start)
        vis_start = best_forward["time"]

        # 找vis_end：逆向候选锚点中评分最高的点
        backward_candidates = [c for c in candidates if c["time"] <= search_end]
        best_backward = scoring_system.select_best_anchor(backward_candidates, asr_end)
        vis_end = best_backward["time"]

    # 漂移判定（升级为评分驱动加权）
    drift_threshold = 5.0
    # 校准vis_start
    start_diff = abs(vis_start - asr_start)
    if start_diff > drift_threshold:
        score = best_forward["score"] if forward_candidates else 0.0
        weight_vis = max(0.5, score)
        weight_asr = 1 - weight_vis
        vis_start = (asr_start * weight_asr) + (vis_start * weight_vis)
    # 校准vis_end
    end_diff = abs(vis_end - asr_end)
    if end_diff > drift_threshold:
        score = best_backward["score"] if backward_candidates else 0.0
        weight_vis = max(0.5, score)
        weight_asr = 1 - weight_vis
        vis_end = (asr_end * weight_asr) + (vis_end * weight_vis)

    # ---------------------- 步骤3：核心合成逻辑：保守扩张 ----------------------
    action_start = min(asr_start, vis_start)
    action_end = max(asr_end, vis_end)

    # ---------------------- 步骤4：最终缓冲区与凑整策略（优化边缘场景） ----------------------
    # 起始点扩充：2秒前摇
    extended_start = max(0.0, action_start - 2.0)
    # 结束点扩充：基础+2秒 → 适配视频时长的凑整 → +1秒
    pre_end = action_end + 2.0
    ceil_pre_end = math.ceil(pre_end)
    final_end_candidate = ceil_pre_end + 1
    if final_end_candidate > video_duration:
        final_end = math.floor(pre_end) + 1
    else:
        final_end = final_end_candidate

    # ---------------------- 步骤5：物理边界硬约束 ----------------------
    # EOF保护
    final_end = min(final_end, video_duration)
    # 最短路径保护（至少2秒）
    if (final_end - extended_start) < 2.0:
        final_end = extended_start + 2.0
    # 最长时长约束（不超过60秒）
    if (final_end - extended_start) > 60.0:
        final_end = extended_start + 60.0

    return (round(extended_start, 2), round(final_end, 2))

# ====================== 调用示例 ======================
if __name__ == "__main__":

    asr_start = 10.5  # ASR识别的语义起点
    asr_end = 25.8    # ASR识别的语义终点

    print(f"最终剪辑范围：{extended_start}s → {final_end}s")
```

## 四、核心总结
### 关键点回顾
1. **现有逻辑的核心价值**：“Expand-Only”保守扩张（min/max取超集）是教学视频剪辑的“黄金逻辑”，彻底解决了语音-画面不同步导致的内容缺失问题；
2. **核心优化方向**：引入评分系统优化视觉锚点选择，从“时序优先”升级为“置信度优先”，过滤光影抖动等伪跳变点，不破坏现有核心逻辑；
3. **工程化兜底**：漂移判定的动态加权、凑整策略的边缘适配，进一步提升了极端场景下的鲁棒性，保证“语义不丢、动作完整、余韵充足”的目标落地。
