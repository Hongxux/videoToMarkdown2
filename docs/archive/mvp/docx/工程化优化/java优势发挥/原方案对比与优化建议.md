## 三、分阶段优化建议

### Phase 1: 多级缓存（最高优先级）⭐⭐⭐⭐⭐

**投入**: 1-2 天  
**收益**: 处理时间 15s → 8s (47% 提升)  
**风险**: 低

#### 3.1 为什么优先做缓存？

1. **投入产出比最高**: 代码改动小，收益大
2. **测试/调试场景收益明显**: 相同视频重复处理，缓存命中率 70-80%
3. **降低 Python Worker 负载**: 减少 60-70% 的计算请求
4. **无需 Python 端配合**: 纯 Java 端实现

#### 3.2 实现方案

```java
@Service
public class CachedFusionService {
    
    @Autowired
    private FusionDecisionService fusionService;
    
    // L1: 本地缓存 (Caffeine) - 热数据
    private final Cache<String, FeatureResponse> localCache = Caffeine.newBuilder()
        .maximumSize(1000)
        .expireAfterWrite(10, TimeUnit.MINUTES)
        .recordStats()
        .build();
    
    // L2: Redis 缓存 (可选，跨实例共享)
    @Autowired(required = false)
    private RedisTemplate<String, byte[]> redisTemplate;
    
    /**
     * 带缓存的特征提取
     */
    public CompletableFuture<FeatureResponse> extractFeaturesWithCache(
        String videoPath, double start, double end, String text
    ) {
        String cacheKey = generateKey(videoPath, start, end);
        
        // L1 查询
        FeatureResponse cached = localCache.getIfPresent(cacheKey);
        if (cached != null) {
            logger.debug("Cache HIT (L1): {}", cacheKey);
            return CompletableFuture.completedFuture(cached);
        }
        
        // L2 查询 (如果配置了 Redis)
        if (redisTemplate != null) {
            byte[] redisValue = redisTemplate.opsForValue().get(cacheKey);
            if (redisValue != null) {
                logger.debug("Cache HIT (L2): {}", cacheKey);
                FeatureResponse response = deserialize(redisValue);
                localCache.put(cacheKey, response); // 回填 L1
                return CompletableFuture.completedFuture(response);
            }
        }
        
        // Cache MISS: 调用原有逻辑
        logger.debug("Cache MISS: {}", cacheKey);
        return fusionService.processVideo(videoPath, text, start, end, ...)
            .thenApply(result -> {
                // 解析结果并缓存
                FeatureResponse response = parseResponse(result);
                
                // 写入缓存
                localCache.put(cacheKey, response);
                if (redisTemplate != null) {
                    redisTemplate.opsForValue().set(
                        cacheKey, 
                        serialize(response), 
                        1, 
                        TimeUnit.HOURS
                    );
                }
                
                return response;
            });
    }
    
    private String generateKey(String videoPath, double start, double end) {
        // 使用视频名 + 时间范围生成 key
        String videoName = new File(videoPath).getName();
        return String.format("%s:%.1f:%.1f", 
            videoName, 
            Math.floor(start), 
            Math.floor(end)
        );
    }
}
```

### Phase 2: 批处理优化（高优先级）⭐⭐⭐⭐

**投入**: 2-3 天  
**收益**: gRPC 调用 57 次 → 2-3 次 (95% 减少)  
**风险**: 中（需要 Python 端配合）

#### 3.6 为什么做批处理？

1. **网络开销巨大**: 57 次 gRPC 调用，累计延迟 1-2 秒
2. **GPU 利用率低**: 单个 segment 无法充分利用 GPU 批量推理
3. **序列化开销**: 每次调用都要序列化/反序列化

#### 3.7 实现方案

```java
@Service
public class BatchProcessor {
    
    private final List<BatchItem> batch = new CopyOnWriteArrayList<>();
    private final ScheduledExecutorService scheduler = 
        Executors.newScheduledThreadPool(1);
    
    private static final int MAX_BATCH_SIZE = 10;
    private static final long MAX_WAIT_MS = 100;
    
    @PostConstruct
    public void init() {
        // 定时刷新批次
        scheduler.scheduleAtFixedRate(
            this::processBatch, 
            MAX_WAIT_MS, 
            MAX_WAIT_MS, 
            TimeUnit.MILLISECONDS
        );
    }
    
    /**
     * 提交 segment 到批处理队列
     */
    public CompletableFuture<FeatureResponse> submitSegment(
        String videoPath, double start, double end, String text
    ) {
        CompletableFuture<FeatureResponse> future = new CompletableFuture<>();
        BatchItem item = new BatchItem(videoPath, start, end, text, future);
        batch.add(item);
        
        // 达到批次大小，立即处理
        if (batch.size() >= MAX_BATCH_SIZE) {
            processBatch();
        }
        
        return future;
    }
    
    private void processBatch() {
        if (batch.isEmpty()) return;
        
        List<BatchItem> items = new ArrayList<>(batch);
        batch.clear();
        
        try {
            // 🔑 关键：一次 gRPC 调用处理多个 segment
            BatchFeatureRequest request = buildBatchRequest(items);
            BatchFeatureResponse response = pythonClient.extractFeaturesBatch(request);
            
            // 分发结果
            for (int i = 0; i < items.size(); i++) {
                items.get(i).future.complete(response.getResponses(i));
            }
            
            logger.info("Batch processed: {} segments in one call", items.size());
            
        } catch (Exception e) {
            // 失败时逐个降级
            items.forEach(item -> item.future.completeExceptionally(e));
        }
    }
}
```

async def ExtractFeaturesBatch(
    self, 
    request: BatchFeatureRequest, 
    context
) -> BatchFeatureResponse:
    """批量特征提取 - 优化 GPU 利用率"""
    
    # 批量提取视觉特征 (GPU 并行)
    visual_features_batch = await self.visual_extractor.extract_batch([
        (seg.video_path, seg.start_sec, seg.end_sec) 
        for seg in request.segments
    ])
    
    # 批量提取语义特征 (BERT 批量推理)

        seg.segment_text for seg in request.segments
    ])

        FeatureResponse(
            success=True,
            visual_features=visual_features_batch[i],
            semantic_features=semantic_features_batch[i]
        )
        for i in range(len(request.segments))
    ]
    
    return BatchFeatureResponse(responses=responses)

### Phase 3: 容错增强（中优先级）⭐⭐⭐

**投入**: 1 天  
**收益**: 可靠性 95% → 99%

#### 3.11 为什么做容错？

1. **单点故障影响大**: 一个 segment 失败导致整个视频失败
2. **无超时控制**: gRPC 调用可能无限等待
3. **无降级策略**: 失败后无法提供基础服务

#### 3.12 实现方案

```java
@Service
public class ResilientFusionService {
    
    @Autowired
    private PythonComputeClient pythonClient;
    
    @Autowired
    private CachedFusionService cachedService;
    
    /**
     * 带容错的特征提取
     */
    public CompletableFuture<FeatureResponse> extractFeaturesResilient(
        String videoPath, double start, double end, String text
    ) {
        return cachedService.extractFeaturesWithCache(videoPath, start, end, text)
            // 🔑 超时控制：3 秒
            .orTimeout(3, TimeUnit.SECONDS)
            .exceptionally(ex -> {
                if (ex instanceof TimeoutException) {
                    logger.warn("Timeout for segment {}:{}-{}", videoPath, start, end);
                } else {
                    logger.error("Failed to extract features: {}", ex.getMessage());
                }
                
                // 降级策略：返回简化特征
                return createFallbackResponse(videoPath, start, end, text);
            });
    }
    
    private FeatureResponse createFallbackResponse(
        String videoPath, double start, double end, String text
    ) {
        // 降级：仅使用文本特征
        return FeatureResponse.newBuilder()
            .setSuccess(true)
            .setSemanticFeatures(SemanticFeatures.newBuilder()
                .setKnowledgeType("abstract") // 保守策略
                .setConfidence(0.3) // 低置信度
                .build())
            .setVisualFeatures(VisualFeatures.newBuilder()
                .setVisualConfidence(0.1) // 极低置信度
                .setIsPotentialDynamic(false)
                .build())
            .build();
    }
}
```

```java
@Service
public class CircuitBreakerFusionService {
    
    @CircuitBreaker(name = "pythonWorker", fallbackMethod = "fallbackExtractFeatures")
    @Retry(name = "pythonWorker", maxAttempts = 3)
    @Bulkhead(name = "pythonWorker", type = Bulkhead.Type.THREADPOOL)
    public CompletableFuture<FeatureResponse> extractFeatures(
        String videoPath, double start, double end, String text
    ) {
        return pythonClient.extractFeatures(...);
    }
    
    private CompletableFuture<FeatureResponse> fallbackExtractFeatures(
        String videoPath, double start, double end, String text, Throwable ex
    ) {
        logger.warn("Circuit breaker OPEN, using fallback");
        
        // 尝试从缓存获取
        FeatureResponse cached = cachedService.getFromCache(videoPath, start, end);
        if (cached != null) {
            return CompletableFuture.completedFuture(cached);
        }
        
        // 返回降级结果
        return CompletableFuture.completedFuture(
            createFallbackResponse(videoPath, start, end, text)
        );
    }
}
```

## 四、综合优化方案

### 4.1 最终架构

```java
@Service
public class OptimizedVideoProcessor {
    
    @Autowired
    private CachedFusionService cachedService;
    
    @Autowired
    private BatchProcessor batchProcessor;
    
    @Autowired
    private ResilientFusionService resilientService;
    
    @Autowired
    private AdaptiveResourceOrchestrator orchestrator;
    
    /**
     * 综合优化版本：缓存 + 批处理 + 容错 + 限流
     */
    public CompletableFuture<VideoProcessingResult> processVideoOptimized(
        String videoPath,
        List<Segment> segments
    ) {
        List<CompletableFuture<SegmentResult>> futures = segments.stream()
            .map(seg -> CompletableFuture.supplyAsync(
                () -> {
                    try {
                        // 限流
                        orchestrator.getSemaphore().acquire();
                        
                        // 缓存 + 批处理 + 容错
                        FeatureResponse response = resilientService
                            .extractFeaturesResilient(
                                videoPath, 
                                seg.getStart(), 
                                seg.getEnd(), 
                                seg.getText()
                            )
                            .get();
                        
                        return SegmentResult.success(seg.getId(), response);
                        
                    } catch (Exception e) {
                        return SegmentResult.failure(seg.getId(), e.getMessage());
                    } finally {
                        orchestrator.getSemaphore().release();
                    }
                }
            ))
            .collect(Collectors.toList());
        
        return CompletableFuture.allOf(futures.toArray(new CompletableFuture[0]))
            .thenApply(v -> {
                List<SegmentResult> results = futures.stream()
                    .map(CompletableFuture::join)
                    .collect(Collectors.toList());
                
                return new VideoProcessingResult(videoPath, results);
            });
    }
}
```
