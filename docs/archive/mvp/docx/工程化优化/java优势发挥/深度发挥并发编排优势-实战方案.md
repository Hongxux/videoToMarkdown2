## 一、当前架构的并发编排痛点分析

### 1.1 现状问题

```java
// 当前实现：串行处理，未发挥 Java 并发优势
public List<String> processFullVideo(String videoPath, ...) {
    List<Segment> segments = loadSegments(); // 19 个片段
    List<String> results = new ArrayList<>();
    
    // ❌ 问题：串行处理，单核利用率低
    for (Segment seg : segments) {
        String result = processVideo(seg).join(); // 阻塞等待
        results.add(result);
    }
    
    return results;
}
```

**痛点**:
1. **未利用多核**: 19 个 segment 串行处理，CPU 利用率 < 20%
2. **阻塞等待**: 每个 segment 等待 gRPC 返回，浪费线程资源
3. **无依赖优化**: 部分 segment 可并行，但未识别
4. **无容错机制**: 单个失败导致整体失败

## 三、具体实施方案

### 3.1 方案 A: CompletableFuture 并行编排（推荐）

**核心思想**: 利用 `CompletableFuture` 实现 segment 的并行处理 + 依赖编排

#### 3.1.1 基础并行处理

```java
@Service
public class ParallelVideoProcessor {
    
    @Autowired
    private FusionDecisionService fusionService;
    
    @Autowired
    private AdaptiveResourceOrchestrator orchestrator;
    
    private final ExecutorService executor = new ThreadPoolExecutor(
        4,  // corePoolSize: 4 核心线程
        16, // maximumPoolSize: 最多 16 线程
        60, TimeUnit.SECONDS,
        new LinkedBlockingQueue<>(100),
        new ThreadPoolExecutor.CallerRunsPolicy() // 拒绝策略：调用者执行
    );
    
    /**
     * 并行处理视频 - 基础版
     * 性能提升: 串行 60s → 并行 15s (4x)
     */
    public CompletableFuture<VideoProcessingResult> processVideoParallel(
        String videoPath,
        List<Segment> segments
    ) {
        // 1. 创建所有 segment 的异步任务
        List<CompletableFuture<SegmentResult>> futures = segments.stream()
            .map(seg -> CompletableFuture.supplyAsync(
                () -> processSegment(seg),
                executor // 使用自定义线程池
            ))
            .collect(Collectors.toList());
        
        // 2. 等待所有任务完成（利用 JUC 的 allOf）
        CompletableFuture<Void> allDone = CompletableFuture.allOf(
            futures.toArray(new CompletableFuture[0])
        );
        
        // 3. 聚合结果
        return allDone.thenApply(v -> {
            List<SegmentResult> results = futures.stream()
                .map(CompletableFuture::join)
                .collect(Collectors.toList());
            
            return new VideoProcessingResult(videoPath, results);
        });
    }
    
    private SegmentResult processSegment(Segment seg) {
        try {
            // 调用原有逻辑
            String result = fusionService.processVideo(
                seg.getVideoPath(),
                seg.getTranscription(),
                seg.getStart(),
                seg.getEnd(),
                // ...
            ).join();
            
            return SegmentResult.success(seg.getId(), result);
            
        } catch (Exception e) {
            logger.error("Segment {} failed: {}", seg.getId(), e.getMessage());
            return SegmentResult.failure(seg.getId(), e.getMessage());
        }
    }
}
```

**核心价值**:
- ✅ 利用 Java 1:1 线程模型，真正多核并行
- ✅ `CompletableFuture.allOf()` 是 JUC 原生工具，零开销
- ✅ 线程池复用，避免频繁创建/销毁线程

#### 3.1.2 高级编排：依赖调度 + 容错

```java
/**
 * 高级并行编排 - 支持依赖调度、容错、超时
 * 场景: 某些 segment 依赖前序结果（如字幕生成依赖前文）
 */
public class AdvancedParallelProcessor {
    
    /**
     * 依赖编排示例：
     * Segment 1-5 并行 → 生成字幕 → Segment 6-19 并行
     */
    public CompletableFuture<VideoProcessingResult> processWithDependencies(
        String videoPath,
        List<Segment> segments
    ) {
        // 阶段 1: 前 5 个 segment 并行处理
        List<CompletableFuture<SegmentResult>> phase1 = segments.subList(0, 5).stream()
            .map(seg -> CompletableFuture.supplyAsync(
                () -> processSegment(seg),
                executor
            ))
            .collect(Collectors.toList());
        
        // 阶段 2: 等待阶段 1 完成后，生成字幕
        CompletableFuture<String> subtitleFuture = CompletableFuture.allOf(
            phase1.toArray(new CompletableFuture[0])
        ).thenApplyAsync(v -> {
            // 聚合前 5 个结果生成字幕
            List<SegmentResult> phase1Results = phase1.stream()
                .map(CompletableFuture::join)
                .collect(Collectors.toList());
            
            return generateSubtitles(phase1Results);
        }, executor);
        
        // 阶段 3: 字幕生成后，处理剩余 segment（依赖字幕）
        CompletableFuture<List<SegmentResult>> phase2 = subtitleFuture.thenComposeAsync(
            subtitle -> {
                List<CompletableFuture<SegmentResult>> futures = segments.subList(5, 19).stream()
                    .map(seg -> CompletableFuture.supplyAsync(
                        () -> processSegmentWithSubtitle(seg, subtitle),
                        executor
                    ))
                    .collect(Collectors.toList());
                
                return CompletableFuture.allOf(futures.toArray(new CompletableFuture[0]))
                    .thenApply(v -> futures.stream()
                        .map(CompletableFuture::join)
                        .collect(Collectors.toList())
                    );
            },
            executor
        );
        
        // 最终聚合
        return phase2.thenCombine(
            CompletableFuture.completedFuture(phase1),
            (p2Results, p1Futures) -> {
                List<SegmentResult> allResults = new ArrayList<>();
                allResults.addAll(p1Futures.stream()
                    .map(CompletableFuture::join)
                    .collect(Collectors.toList()));
                allResults.addAll(p2Results);
                
                return new VideoProcessingResult(videoPath, allResults);
            }
        );
    }
    
    /**
     * 容错编排：单个失败不影响整体
     */
    public CompletableFuture<VideoProcessingResult> processWithFaultTolerance(
        String videoPath,
        List<Segment> segments
    ) {
        List<CompletableFuture<SegmentResult>> futures = segments.stream()
            .map(seg -> CompletableFuture.supplyAsync(
                () -> processSegment(seg),
                executor
            )
            // 🔑 关键：异常处理，失败返回降级结果
            .exceptionally(ex -> {
                logger.warn("Segment {} failed, using fallback", seg.getId());
                return SegmentResult.fallback(seg.getId(), "TEXT_ONLY");
            })
            // 🔑 关键：超时控制（3 秒）
            .orTimeout(3, TimeUnit.SECONDS)
            .exceptionally(ex -> {
                logger.warn("Segment {} timeout", seg.getId());
                return SegmentResult.timeout(seg.getId());
            }))
            .collect(Collectors.toList());
        
        return CompletableFuture.allOf(futures.toArray(new CompletableFuture[0]))
            .thenApply(v -> {
                List<SegmentResult> results = futures.stream()
                    .map(CompletableFuture::join)
                    .collect(Collectors.toList());
                
                return new VideoProcessingResult(videoPath, results);
            });
    }
}
```

**核心价值**:
- ✅ `thenCompose`/`thenCombine` 实现复杂依赖编排
- ✅ `exceptionally` 实现优雅降级
- ✅ `orTimeout` 实现超时控制
- ✅ 所有操作都是 JUC 原生，性能最优

### 3.2 方案 B: Fork/Join 框架（大视频分治）

**核心思想**: 利用 Fork/Join 框架实现递归分治，适合超大视频

#### 3.2.1 实现代码

```java
/**
 * Fork/Join 分治处理
 * 场景: 视频 > 1 小时，segment > 100 个
 */
public class VideoForkJoinTask extends RecursiveTask<List<SegmentResult>> {
    
    private static final int THRESHOLD = 10; // 阈值：10 个 segment
    
    private final List<Segment> segments;
    private final int start;
    private final int end;
    
    public VideoForkJoinTask(List<Segment> segments, int start, int end) {
        this.segments = segments;
        this.start = start;
        this.end = end;
    }
    
    @Override
    protected List<SegmentResult> compute() {
        int length = end - start;
        
        // 基准情况：segment 数量 <= 阈值，直接处理
        if (length <= THRESHOLD) {
            return processDirectly();
        }
        
        // 递归情况：分治
        int mid = start + length / 2;
        
        // Fork 左半部分
        VideoForkJoinTask leftTask = new VideoForkJoinTask(segments, start, mid);
        leftTask.fork(); // 异步执行
        
        // 当前线程处理右半部分
        VideoForkJoinTask rightTask = new VideoForkJoinTask(segments, mid, end);
        List<SegmentResult> rightResult = rightTask.compute();
        
        // Join 左半部分结果
        List<SegmentResult> leftResult = leftTask.join();
        
        // 合并结果
        List<SegmentResult> merged = new ArrayList<>();
        merged.addAll(leftResult);
        merged.addAll(rightResult);
        
        return merged;
    }
    
    private List<SegmentResult> processDirectly() {
        List<SegmentResult> results = new ArrayList<>();
        
        for (int i = start; i < end; i++) {
            Segment seg = segments.get(i);
            try {
                SegmentResult result = processSegment(seg);
                results.add(result);
            } catch (Exception e) {
                results.add(SegmentResult.failure(seg.getId(), e.getMessage()));
            }
        }
        
        return results;
    }
}

// 使用 Fork/Join 池
@Service
public class ForkJoinVideoProcessor {
    
    private final ForkJoinPool forkJoinPool = new ForkJoinPool(
        Runtime.getRuntime().availableProcessors() // 自动适配 CPU 核数
    );
    
    public List<SegmentResult> processVideo(List<Segment> segments) {
        VideoForkJoinTask task = new VideoForkJoinTask(segments, 0, segments.size());
        return forkJoinPool.invoke(task);
    }
}
```

**核心价值**:
- ✅ 工作窃取算法（Work-Stealing）：线程自动负载均衡
- ✅ 递归分治：自动适配任务规模
- ✅ JVM 原生优化：性能接近手写多线程

### 3.3 方案 C: 自适应并发控制（避免过载）

**核心思想**: 根据系统负载动态调整并发度

#### 3.3.1 实现代码

```java
@Service
public class AdaptiveConcurrencyProcessor {
    
    @Autowired
    private AdaptiveResourceOrchestrator orchestrator;
    
    private final Semaphore semaphore;
    
    public AdaptiveConcurrencyProcessor(AdaptiveResourceOrchestrator orchestrator) {
        this.orchestrator = orchestrator;
        this.semaphore = orchestrator.getSemaphore();
    }
    
    /**
     * 自适应并发处理
     * 根据 CPU/内存动态调整并发度
     */
    public CompletableFuture<VideoProcessingResult> processAdaptive(
        String videoPath,
        List<Segment> segments
    ) {
        List<CompletableFuture<SegmentResult>> futures = segments.stream()
            .map(seg -> CompletableFuture.supplyAsync(
                () -> {
                    try {
                        // 🔑 关键：获取许可（自适应限流）
                        semaphore.acquire();
                        
                        // 处理 segment
                        return processSegment(seg);
                        
                    } catch (InterruptedException e) {
                        Thread.currentThread().interrupt();
                        return SegmentResult.failure(seg.getId(), "Interrupted");
                    } finally {
                        // 释放许可
                        semaphore.release();
                    }
                }
            ))
            .collect(Collectors.toList());
        
        return CompletableFuture.allOf(futures.toArray(new CompletableFuture[0]))
            .thenApply(v -> {
                List<SegmentResult> results = futures.stream()
                    .map(CompletableFuture::join)
                    .collect(Collectors.toList());
                
                return new VideoProcessingResult(videoPath, results);
            });
    }
}

// 动态调整并发度
@Component
public class AdaptiveResourceOrchestrator {
    
    private final Semaphore semaphore = new Semaphore(8); // 初始 8 个许可
    private final OperatingSystemMXBean osBean;
    
    public AdaptiveResourceOrchestrator() {
        this.osBean = (OperatingSystemMXBean) ManagementFactory.getOperatingSystemMXBean();
        
        // 每秒调整一次并发度
        ScheduledExecutorService scheduler = Executors.newScheduledThreadPool(1);
        scheduler.scheduleAtFixedRate(this::adjustConcurrency, 0, 1, TimeUnit.SECONDS);
    }
    
    private void adjustConcurrency() {
        double cpuLoad = osBean.getCpuLoad();
        long freeMem = osBean.getFreeMemorySize() / (1024 * 1024); // MB
        
        int targetLimit;
        
        // 根据 CPU 和内存动态调整
        if (cpuLoad > 0.8 || freeMem < 1024) {
            targetLimit = 2; // 高负载：降级
        } else if (cpuLoad > 0.6 || freeMem < 2048) {
            targetLimit = 4; // 中负载：限流
        } else {
            targetLimit = 8; // 低负载：全速
        }
        
        // 调整 Semaphore 许可数
        int current = semaphore.availablePermits();
        int delta = targetLimit - current;
        
        if (delta > 0) {
            semaphore.release(delta); // 增加许可
        } else if (delta < 0) {
            semaphore.tryAcquire(-delta); // 减少许可
        }
        
        logger.debug("Adjusted concurrency: {} (CPU: {}, FreeMem: {}MB)", 
                     targetLimit, cpuLoad, freeMem);
    }
    
    public Semaphore getSemaphore() {
        return semaphore;
    }
}
```

**核心价值**:
- ✅ `Semaphore` 是 JUC 原生工具，性能极高
- ✅ 自动适配系统负载，避免过载
- ✅ 保护 Python Worker 不被压垮

### 3.4 方案 D: 批量 + 并行组合（终极优化）

**核心思想**: 批处理减少 gRPC 调用 + 并行提升吞吐

#### 3.4.1 架构图

```
19 个 Segment
    ↓
分批 (每批 10 个)
    ↓
┌─────────────────────────────────────┐
│  Batch 1 (Seg 1-10)  │  Batch 2 (Seg 11-19) │
│         ↓            │         ↓             │
│  gRPC 批量调用       │  gRPC 批量调用        │  ← 并行执行
│         ↓            │         ↓             │
│  返回 10 个结果      │  返回 9 个结果        │
└─────────────────────────────────────┘
    ↓
聚合 19 个结果
```

#### 3.4.2 实现代码

```java
@Service
public class BatchParallelProcessor {
    
    @Autowired
    private BatchProcessor batchProcessor;
    
    private static final int BATCH_SIZE = 10;
    
    /**
     * 批量 + 并行组合
     * gRPC 调用: 19 次 → 2 次
     * 并行度: 2 批并行
     */
    public CompletableFuture<VideoProcessingResult> processBatchParallel(
        String videoPath,
        List<Segment> segments
    ) {
        // 1. 分批
        List<List<Segment>> batches = partitionList(segments, BATCH_SIZE);
        
        // 2. 每批并行处理
        List<CompletableFuture<List<SegmentResult>>> batchFutures = batches.stream()
            .map(batch -> CompletableFuture.supplyAsync(
                () -> processBatch(batch),
                executor
            ))
            .collect(Collectors.toList());
        
        // 3. 等待所有批次完成
        return CompletableFuture.allOf(batchFutures.toArray(new CompletableFuture[0]))
            .thenApply(v -> {
                List<SegmentResult> allResults = batchFutures.stream()
                    .map(CompletableFuture::join)
                    .flatMap(List::stream)
                    .collect(Collectors.toList());
                
                return new VideoProcessingResult(videoPath, allResults);
            });
    }
    
    private List<SegmentResult> processBatch(List<Segment> batch) {
        // 调用批处理器（一次 gRPC 调用处理多个 segment）
        return batchProcessor.processBatch(batch);
    }
    
    private <T> List<List<T>> partitionList(List<T> list, int size) {
        List<List<T>> partitions = new ArrayList<>();
        for (int i = 0; i < list.size(); i += size) {
            partitions.add(list.subList(i, Math.min(i + size, list.size())));
        }
        return partitions;
    }
}
```

**核心价值**:
- ✅ gRPC 调用次数: 19 → 2 (90% 减少)
- ✅ 批次间并行，充分利用多核
- ✅ 综合性能提升: **8-10 倍**

## 六、代码示例：完整实现

```java
@Service
public class OptimizedVideoProcessor {
    
    @Autowired
    private FusionDecisionService fusionService;
    
    @Autowired
    private BatchProcessor batchProcessor;
    
    @Autowired
    private AdaptiveResourceOrchestrator orchestrator;
    
    private final ExecutorService executor = Executors.newFixedThreadPool(8);
    
    /**
     * 终极优化版本：批量 + 并行 + 容错 + 自适应
     */
    public CompletableFuture<VideoProcessingResult> processVideoOptimized(
        String videoPath,
        List<Segment> segments
    ) {
        // 1. 分批
        List<List<Segment>> batches = partitionList(segments, 10);
        
        // 2. 每批并行处理（带容错和自适应限流）
        List<CompletableFuture<List<SegmentResult>>> batchFutures = batches.stream()
            .map(batch -> CompletableFuture.supplyAsync(
                () -> {
                    try {
                        // 自适应限流
                        orchestrator.getSemaphore().acquire();
                        
                        // 批量处理
                        return batchProcessor.processBatch(batch);
                        
                    } catch (InterruptedException e) {
                        Thread.currentThread().interrupt();
                        return batch.stream()
                            .map(seg -> SegmentResult.failure(seg.getId(), "Interrupted"))
                            .collect(Collectors.toList());
                    } finally {
                        orchestrator.getSemaphore().release();
                    }
                },
                executor
            )
            // 容错：单批失败不影响其他批
            .exceptionally(ex -> {
                logger.error("Batch failed: {}", ex.getMessage());
                return batch.stream()
                    .map(seg -> SegmentResult.fallback(seg.getId(), "BATCH_FAILED"))
                    .collect(Collectors.toList());
            })
            // 超时控制：每批最多 10 秒
            .orTimeout(10, TimeUnit.SECONDS)
            .exceptionally(ex -> {
                logger.error("Batch timeout");
                return batch.stream()
                    .map(seg -> SegmentResult.timeout(seg.getId()))
                    .collect(Collectors.toList());
            }))
            .collect(Collectors.toList());
        
        // 3. 聚合所有批次结果
        return CompletableFuture.allOf(batchFutures.toArray(new CompletableFuture[0]))
            .thenApply(v -> {
                List<SegmentResult> allResults = batchFutures.stream()
                    .map(CompletableFuture::join)
                    .flatMap(List::stream)
                    .collect(Collectors.toList());
                
                // 统计成功/失败
                long successCount = allResults.stream()
                    .filter(SegmentResult::isSuccess)
                    .count();
                
                logger.info("Processing complete: {}/{} segments succeeded", 
                           successCount, allResults.size());
                
                return new VideoProcessingResult(videoPath, allResults);
            });
    }
}
```
