# 基于语义地形与双层画像的 Markdown 阅读器 (Kotlin/Android)

## 一、 产品愿景与核心理念
本项目旨在彻底解决移动 Web 端长文阅读时的手势冲突与沉浸感破坏问题，通过 Kotlin 原生 (Jetpack Compose) 打造一款拥有“物理直觉”的认知闭环阅读器。
阅读器不仅是对图文与富媒体的展示（媲美 Obsidian 渲染标准），更是一个**“思想的实验室”**。它通过极简的手势收集无感数据，构建用户的“十维精神画像”，进而动态渲染出千人千面的“多焦段语义地形”，在帮助用户聚焦核心价值的同时，打破信息茧房。

---

## 二、 核心架构：多焦段渲染与桥接投影

### 1. 内容载体标准
*   **富文本兼顾**：作为输入源的 `.md` 文件（如 `enhanced_output.md`）可能极为复杂。阅读器必须基于类似 Obsidian 的渲染标准，完美支持复杂的嵌套列表、内嵌视频播放模块、高清截图模块以及复杂排版。

### 2. 多焦段界面呈现 (Multi-Focal Interface)
系统不再以同等字号盲目平铺整篇文章，而是根据用户的“精神画像”判断每段文字的权重（relevance_score）。

*   **强相关（聚焦区）**：加大字号、高亮、加粗，置于视觉中心。
*   **弱相关（降噪区）**：并非粗暴剔除，而是进行视觉降噪（文字变灰、边距缩小、整体折叠）。
*   **视觉隐喻补充 - “呼吸态指示”**：当用户滚过这些“极度不相关被降噪”的内容时，文字左侧会有一个微弱呼吸着的发光小蓝点（或 ✧ 符号）。这不仅是 UI 的点缀，它在心理层面对用户暗示：“这里虽然折叠了，但有一条专门为你准备的内幕/解说”。

### 3. AI 桥接文本 (Bridge Text)
对于跨专业或难以理解的概念，后端会动态生成一段“桥接文本”。
*   **对话场域（Conversation Feel）**：桥接便签不使用冰冷的百科说明语气，而是悬浮在原文旁侧，头部甚至带有一个微小的 AI 助手头像轮廓。
*   **情感定位**：它呼应了苏格拉底式的陪伴感，扮演一个了解你十维精神画像的向导。

## 三、 闭环交互规范：认知采集与物理探针

### 1. 段落级探针交互 (Command Swipes)
抛弃传统的“点赞/长按唤出菜单”，用户通过手指如同“指挥家”般的纯粹滑动，完成对知识片段的评判。所有手势动作（Telemetry）均会被后端系统采集，用于不断校准用户的【双层十维精神画像】。

*   **左滑 (Swipe Left) -> 降权/删除**
    *   **动效**：段落向左平滑移出屏幕，伴随删除动画，下方的段落自动上移填补空白。
    *   **业务逻辑**：代表用户对这段信息**不感兴趣或点暗**。该负向信息被采集，用于调整底层画像。
*   **右滑 (Swipe Right) -> 批注/思考/桥接投影**
    *   **动效**：
        1. 针对普通段落：向右滑开，如同抽屉拉出，在段落下方/旁侧原地展开一个批注便签。
        2. 针对带 ✧ 符号的降噪段落：拉出 AI 生成的“桥接便签”。
        3. **焦点居中（Focus Management）**：伴随一声清脆的触觉马达震动，屏幕极其顺滑地自动微调滚动位置（Auto-scroll），确保被滑动的原段落和展开的便签**同时悬停在屏幕的视觉正中心**。
    *   **业务逻辑**：代表用户对该信息**很有想法和思考**。系统将读取用户的批注内容（或桥接互动），作为正向的高价值数据被采集。
*   **双击 (Double Tap) -> 收藏/共鸣**
    *   **动效**：段落产生短暂的高亮闪烁或微小缩放（Scale弹簧效果）。
    *   **业务逻辑**：代表用户觉得该段落极符合需要，产生极强共鸣，为最高权重的正向画像反馈。

### 2. 词句级手术刀交互 (Lexical Scalpel)
*   **第一次单击 (Tap) -> 智能分词与浮起**
    *   **技术实现**：在 Android 平台引入轻量级 NLP 框架（如利用 ONNX 运行的轻量分词模型），通过 JNI 供 Kotlin 调用，实现端侧极低延迟的最小语义扩展抓取。
    *   **动效**：被抓取的词汇下方产生极具标识性的下划线，同时在 z 轴上轻微浮起。必须伴随极度短促清脆的高频震动（区分于段落滑动的低沉厚重感），给予精准抓取的触觉确认。
*   **再次单击该词 (Double Click on Selected) -> 三维解析卡片展开**
    *   打破传统的浅表词典释义，直接从三个思想维度生成深层解析气泡：
        1. **语境化 (Contextualize)**：解析该词在这个特定句子、这段上下文中的确切含义。
        2. **深 (First Principle)**：从第一性原理出发，剖析该词为什么在这个语境能产生如此精妙的表达效果，梳理出作者逻辑论证的骨架。
        3. **广 (Industry Horizon)**：跨界连接，梳理当前行业顶尖人物或企业是如何认知并使用这个维度的底层思维的。

## 四、 后端 Prompt 架构雏形

后端架构放弃直接输出 Markdown，而是结合该读者的【十维精神画像】和当次【段落交互事件切片】，由 LLM 生成供 Android Compose 渲染的 JSON AST 语法树。

**生成协议示例草稿：**
```json
// LLM Response JSON Payload
[
  {
    "type": "paragraph",
    "original_markdown": "- 自修改软件的实现与意义：作者无意中构建了一个真正的自修改软件...",
    "relevance_score": 0.8, // 决定前端是加粗放大还是变灰折叠
    "bridge_text": null, // 当分数极端（如<0.3）时，依据其十维画像生成向导解说
    "reasoning": "该段落契合用户对效率的极致关注，展示为强相关。"
  }
]
```

## 五、 技术选型及落地要求
1. **原生框架**：全面采用 Kotlin + Jetpack Compose。利用 `SubcomposeLayout` 解决多焦段复杂侧边气泡相对于降噪段落的锚点对齐。
2. **手势引擎**：采用 `Modifier.pointerInput` 结合基于物理的阻尼动画（Spring Physics），保证左滑/右滑/取消时的极致跟手感与粘滞感。
3. **Markdown 渲染**：不建议自研底层解析器，推荐基于 `Markwon` 或其他健壮的 Android MD 引擎，深度开发其 `Span` 注册表，将我们的“词句级 JNI 回调”与“多媒体渲染锚点”挂载于语法树上。
4. **焦点跟踪**：在 Compose 中维护强大的 `ScrollState` 与可见区域索引坐标系，确保无论是“右滑批注”还是“展开桥接文本”，焦点必须经过精准计算滑动至屏幕垂直黄金分割线处。
